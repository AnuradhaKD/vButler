## User Story

**As a** developer building the Destinity vButler app,
**I want** a centralised `NotificationService` module that manages the creation, reading, and deletion of guest notifications,
**So that** all parts of the app (service requests, complaints, billing) can consistently trigger and manage notifications without duplicating storage logic.

---

## Background / Context

Multiple features across the Destinity vButler app — request submission, complaint status changes, billing charges, wake-up call confirmations — need to notify the guest. Without a centralised service, each feature would independently manage `vb:notifications[]` in localStorage, leading to race conditions, inconsistent schemas, and bloated code. `NotificationService` is the single source of truth for notification lifecycle management, ensuring the Alpine.js `notifications` store and localStorage stay in sync.

---

## Acceptance Criteria

- [ ] `NotificationService` is an ES module exported from `/js/services/NotificationService.js`.
- [ ] `NotificationService.create({ type, title, body, resvId, linkTo })` creates a new notification object with the following shape:
  ```json
  {
    "id": "uuid-v4-string",
    "type": "serviceRequest | complaint | billing | checkIn | wakeUp | feedback | general",
    "title": "string",
    "body": "string",
    "isRead": false,
    "resvId": "string | null",
    "linkTo": "string | null",
    "createdAt": "ISO 8601 datetime string"
  }
  ```
- [ ] `create()` appends the new notification to the `vb:notifications[]` array in localStorage.
- [ ] `create()` immediately updates `Alpine.store('notifications').list` to trigger reactive badge count updates without a page reload.
- [ ] `create()` enforces a cap of 100 entries in `vb:notifications[]`; when the cap is exceeded, the oldest entries (lowest `createdAt`) are pruned until the list is at most 100 items.
- [ ] `NotificationService.markRead(id)` finds the notification by `id`, sets `isRead: true`, persists to localStorage, and updates the Alpine store.
- [ ] `NotificationService.markAllRead()` sets `isRead: true` on every notification in the list, persists to localStorage, and updates the Alpine store.
- [ ] `NotificationService.delete(id)` removes the notification with the matching `id` from the list, persists to localStorage, and updates the Alpine store.
- [ ] `NotificationService.getUnreadCount()` returns the count of notifications where `isRead === false`.
- [ ] `NotificationService.getAll()` returns the full notifications array sorted by `createdAt` descending.
- [ ] The service is imported and used by: `RequestService`, `ComplaintService`, and `billing.js` at minimum.
- [ ] All methods handle the case where `vb:notifications` does not yet exist in localStorage (initialise to empty array).

## Technical Notes

- UUID generation: use `crypto.randomUUID()` (available in all modern browsers and Node 14.17+). Fallback: `Date.now().toString(36) + Math.random().toString(36).substr(2)`.
- Module structure:
  ```js
  // /js/services/NotificationService.js
  import { StorageService } from './StorageService.js';

  const STORAGE_KEY = 'vb:notifications';
  const MAX_NOTIFICATIONS = 100;

  function getList() {
    return StorageService.get(STORAGE_KEY) || [];
  }

  function saveList(list) {
    StorageService.set(STORAGE_KEY, list);
    // Sync Alpine store
    if (window.Alpine) {
      Alpine.store('notifications').list = [...list];
    }
  }

  export const NotificationService = {
    create({ type, title, body, resvId = null, linkTo = null }) {
      const list = getList();
      const notification = {
        id: crypto.randomUUID(),
        type,
        title,
        body,
        isRead: false,
        resvId,
        linkTo,
        createdAt: new Date().toISOString()
      };
      list.unshift(notification); // add to front (newest first)
      const pruned = list.slice(0, MAX_NOTIFICATIONS);
      saveList(pruned);
      return notification;
    },
    markRead(id) { /* ... */ },
    markAllRead() { /* ... */ },
    delete(id) { /* ... */ },
    getUnreadCount() { /* ... */ },
    getAll() { /* ... */ }
  };
  ```
- Pruning strategy: `list.slice(0, MAX_NOTIFICATIONS)` after `unshift` keeps the 100 most recent (the list is maintained newest-first).
- `saveList` must not replace the Alpine store array reference; instead mutate it: `Alpine.store('notifications').list.splice(0, Infinity, ...list)` or use assignment with a spread — test reactivity.
- Import in consuming modules:
  ```js
  import { NotificationService } from './NotificationService.js';
  // ...
  NotificationService.create({ type: 'serviceRequest', title: 'Request Submitted', body: 'Your towel request has been received.', linkTo: '#/requests' });
  ```
- JSDoc example:
  ```js
  /**
   * Creates and persists a new notification.
   * @param {Object} opts
   * @param {string} opts.type - Notification category
   * @param {string} opts.title - Short heading
   * @param {string} opts.body - Detail text
   * @param {string|null} [opts.resvId] - Associated reservation ID
   * @param {string|null} [opts.linkTo] - Hash route to navigate on tap
   * @returns {Object} The created notification object
   */

  ```

## Test Cases

### Happy Path

1. Import `NotificationService` in a test script.
2. Call `NotificationService.create({ type: 'billing', title: 'Charge Added', body: 'LKR 2,500 added for minibar.', linkTo: '#/billing' })`.
3. Verify `vb:notifications` in localStorage contains the new entry with correct shape (`id`, `type`, `title`, `body`, `isRead: false`, `resvId: null`, `linkTo`, `createdAt`).
4. Verify `Alpine.store('notifications').list` contains the new entry.
5. Verify `getUnreadCount()` returns 1.
6. Call `markRead(id)` — verify `isRead` is `true` in localStorage and Alpine store.
7. Verify `getUnreadCount()` returns 0.
8. Call `delete(id)` — verify the entry is removed from localStorage and Alpine store.
9. Call `getAll()` — verify empty array returned.

### Edge Cases

- **`vb:notifications` absent from localStorage:** `create()` initialises the array and adds the notification; no error thrown.
- **101st notification created:** List pruned to 100; oldest notification removed; 100 most recent retained.
- **`markRead` with non-existent ID:** No error; `vb:notifications[]` unchanged.
- **`delete` with non-existent ID:** No error; `vb:notifications[]` unchanged.
- **`markAllRead` on empty list:** No error; empty array persisted; `getUnreadCount()` returns 0.
- **localStorage quota exceeded:** `saveList` catches the `QuotaExceededError`; logs a console warning; does not throw; badge count may be stale (acceptable degradation).
- **Alpine not yet initialised (service called during early boot):** Guard `if (window.Alpine)` prevents undefined reference error; storage is still written correctly.
- **Concurrent calls (rapid double-submit):** `getList()` reads fresh from localStorage each time; race condition risk is low in single-threaded JS; document that calls are synchronous.
