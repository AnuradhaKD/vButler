## User Story

**As a** hotel guest,
**I want** to view a full history of my notifications in a dedicated screen,
**So that** I can review updates about my requests, billing, and stay events even if I missed them when they first appeared.

---

## Background / Context

Guests at Browns Hotels & Resorts generate numerous notifications throughout their stay — from room service status updates and housekeeping confirmations to billing charges and check-in reminders. The notification history screen acts as an audit trail and communication log, particularly valuable when guests want to reference a billing event or track a complaint status change. Clear visual differentiation between read and unread items and day-based grouping make it easy to scan.

---

## Acceptance Criteria

- [ ] The screen is accessible at the route `#/notifications` and via the bell icon in the app header.
- [ ] All notifications for the current guest are displayed, sorted by `createdAt` in descending order (newest first).
- [ ] Unread notifications are visually distinct: white (light mode) / dark-surface (dark mode) background with a 3px brand-coloured left border accent (#003c52).
- [ ] Read notifications use a grey/muted background (e.g., `bg-gray-50` light / `bg-gray-800` dark) with no border accent.
- [ ] Each notification item displays: a type-specific icon, a bold title (16px), a body text (14px, muted), and a relative timestamp (e.g., "2 minutes ago", "Yesterday at 3:45 PM") using Day.js `fromNow()`.
- [ ] Tapping a notification item marks it as `isRead: true` in `vb:notifications[]` and navigates to the screen specified by the notification's `linkTo` field (if present).
- [ ] If `linkTo` is absent or null, tapping the notification marks it as read but does not navigate away.
- [ ] A "Mark All as Read" button is displayed at the top of the screen (below the header); tapping it marks all notifications as `isRead: true`.
- [ ] After "Mark All as Read" is tapped, the button disappears (hidden when `unreadCount === 0`).
- [ ] Notifications are grouped by day with sticky section headers: "Today", "Yesterday", and date labels for earlier dates (e.g., "14 March 2026").
- [ ] Individual notifications can be deleted by swiping left (mobile) or long-pressing (mobile) to reveal a "Delete" action, OR via a swipe-to-dismiss gesture.
- [ ] Deleted notifications are removed from `vb:notifications[]` and the Alpine store; the item disappears from the list with a slide-out animation.
- [ ] The empty state (no notifications) shows a bell illustration and the text: "You're all caught up!" with a sub-label "No new notifications."
- [ ] The screen respects light/dark mode and RTL layout.
- [ ] Entering this screen does NOT automatically mark all notifications as read — only tapping individual items or the "Mark All as Read" button does.

## UI / UX Notes

- Screen header: "Notifications" title (18px semibold); back button (left) and an overflow/kebab menu (right) with a "Clear All" option.
- "Mark All as Read" button: text-only style (no fill), brand colour, right-aligned or below the header; hidden when `unreadCount === 0`.
- Notification item height: min 72px; auto-expands for long body text.
- Type-specific icons (20px, in a 40px circle background):
  - Service request: wrench/tool icon, blue background.
  - Complaint: flag icon, amber background.
  - Billing: credit card icon, green background.
  - Check-in reminder: calendar icon, purple background.
  - Wake-up call: alarm clock icon, orange background.
  - Feedback prompt: star icon, gold background.
  - General/system: bell icon, grey background.
- Unread left border: 3px solid #003c52; visually prominent.
- Timestamp: 12px, muted, right-aligned on the same row as the title (or below the body).
- Day section headers: sticky, 12px uppercase, muted, `letter-spacing: 0.05em`; light divider line below.
- Swipe-to-delete (mobile): swipe left reveals a red delete zone with trash icon; full swipe deletes; partial swipe reveals the zone and springs back on release.
- Long-press (mobile): shows a context menu with "Mark as Read" and "Delete" options.
- Empty state: centred vertically; illustration 120px; text 16px medium; sub-label 14px muted.
- Slide-out animation on delete: `x-transition:leave` with translate-x and opacity-0 over 250ms.

## Technical Notes

- Alpine.js `x-data` component reads from `Alpine.store('notifications').list`.
- Day grouping logic:
  ```js
  groupByDay(list) {
    const today = dayjs().startOf('day');
    const yesterday = today.subtract(1, 'day');
    return list.reduce((groups, n) => {
      const d = dayjs(n.createdAt).startOf('day');
      let label;
      if (d.isSame(today)) label = 'Today';
      else if (d.isSame(yesterday)) label = 'Yesterday';
      else label = d.format('D MMMM YYYY');
      (groups[label] = groups[label] || []).push(n);
      return groups;
    }, {});
  }
  ```
- Day.js: import `relativeTime` plugin for `fromNow()`. Relative time refreshes every 60 seconds via `setInterval` to keep timestamps current.
- Mark as read: `NotificationService.markRead(id)` mutates the store list entry and calls `StorageService.set('vb:notifications', list)`.
- Mark all read: `NotificationService.markAllRead()` iterates the list setting all `isRead: true`.
- Delete: `NotificationService.delete(id)` splices the item from the store list and persists.
- Swipe gesture: implement with `@touchstart`, `@touchmove`, `@touchend` handlers or use a lightweight touch library; threshold for delete = swipe > 50% of item width.
- "Clear All": `Object.assign` replacement of store list with `[]`; `StorageService.set('vb:notifications', [])`.
- Sticky section headers: CSS `position: sticky; top: 0;` with `z-index` above list items.

## Test Cases

### Happy Path

1. Navigate to `#/notifications` with 5 notifications: 3 unread (today), 1 read (yesterday), 1 unread (yesterday).
2. Verify "Today" and "Yesterday" section headers are visible and sticky on scroll.
3. Verify 3 unread items have brand left border; 1 read item has muted background.
4. Verify relative timestamps: "5 minutes ago", "32 minutes ago", "1 hour ago", "Yesterday at 2:15 PM".
5. Tap an unread item with `linkTo: '#/requests'` — verify it marks as read and navigates to requests screen.
6. Return to notifications — verify the tapped item now shows read styling (no brand border, muted background).
7. Tap "Mark All as Read" — verify all items switch to read styling and the "Mark All as Read" button disappears.
8. Verify the bell badge in the header shows 0 (hidden).

### Edge Cases

- **Empty list:** Verify bell illustration and "You're all caught up!" message displayed; "Mark All as Read" button hidden.
- **100 notifications (cap test):** List displays correctly; older items pruned by NotificationService once cap is exceeded.
- **Delete via swipe:** Swipe notification left past 50% width → verify it slides out and is removed from `vb:notifications[]`.
- **Long notification body text:** Item height expands; text does not overflow; timestamp remains visible.
- **Notification with no `linkTo`:** Tapping marks as read; user stays on notifications screen.
- **Day grouping at midnight:** Notification created at 11:59 PM today vs 12:00 AM tomorrow — correct "Today" vs new-day grouping.
- **Dark mode:** Unread brand border visible; read muted background correct; section headers legible.
- **RTL (Arabic):** Brand border moves to the right side of unread items; swipe direction reverses (swipe right to delete); timestamps align correctly.
- **Refresh page:** Notification list restored from `vb:notifications[]`; read/unread state preserved.
