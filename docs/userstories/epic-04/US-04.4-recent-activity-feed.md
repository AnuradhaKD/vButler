## User Story

**As a** hotel guest,
**I want** to see a summary of my most recent activity — service requests and billing charges — directly on my dashboard,
**So that** I can stay informed about the status of my requests and charges without navigating away from the home screen.

---

## Background / Context

During an active stay at a Browns Hotels & Resorts property, guests frequently submit service requests and accumulate folio charges. Surfacing the four most recent items from both categories on the dashboard gives guests instant situational awareness and reduces the need to open separate sections to check on pending requests or unexpected charges. The merged feed should feel cohesive and scannable, with clear visual differentiation between request statuses and billing amounts.

---

## Acceptance Criteria

- [ ] The section title "Recent Activity" is displayed with a "View All" link aligned to the right
- [ ] Tapping "View All" navigates to `#/notifications`
- [ ] Up to 4 items are displayed in the feed
- [ ] Items are a merged list of: (a) service requests from `vb:requests` filtered by the active `resvId`, and (b) billing charges from `activeReservation.folio[]`
- [ ] The merged list is sorted by `updatedAt` descending (most recent first)
- [ ] Only the first 4 items of the sorted merged list are displayed
- [ ] Each service request item displays: category icon, service name as title, status badge, and "time ago" relative timestamp (Day.js `fromNow()`)
- [ ] Each billing charge item displays: billing icon, charge description as title, charge amount formatted as "LKR X,XXX", and "time ago" relative timestamp
- [ ] Status badges for service requests use colour coding: `submitted` → blue, `acknowledged` → amber, `in_progress` → orange, `completed` → green, `cancelled` → grey
- [ ] Tapping a service request item navigates to `#/requests/:requestId`
- [ ] Tapping a billing charge item navigates to `#/bill` and scrolls to the relevant charge
- [ ] When the merged feed has zero items, the empty state message "No recent activity" is displayed in place of the feed list
- [ ] The Recent Activity section is hidden entirely when the guest has no active reservation
- [ ] The section renders within the dashboard scroll flow, below the Quick Actions Grid

## Technical Notes

- **Data fetching**:
  ```js
  const requests = StorageService.get('vb:requests') || [];
  const activeResvRequests = requests.filter(r => r.resvId === activeResvId);
  const folioCharges = activeReservation.folio || [];
  ```
- **Normalise to common shape** before merging:
  ```js
  // Request item
  { id, type: 'request', icon, title: serviceName, badge: status, updatedAt }
  // Folio item
  { id, type: 'folio', icon: 'receipt', title: charge.description, amount: charge.amount, updatedAt: charge.postedAt }
  ```
- **Sort and slice**:
  ```js
  const feed = [...activeResvRequests, ...folioCharges]
    .sort((a, b) => dayjs(b.updatedAt).valueOf() - dayjs(a.updatedAt).valueOf())
    .slice(0, 4);
  ```
- **Relative time**: `dayjs(item.updatedAt).fromNow()` — requires Day.js `relativeTime` plugin; update timestamps every 60 seconds via `setInterval` to keep "X minutes ago" values fresh
- **Alpine.js component**: `x-data="recentActivityFeed()"` in `src/js/components/recentActivityFeed.js`
- **Navigation**: `@click="navigate('#/requests/' + item.id)"` for requests; `@click="navigate('#/bill')"` for folio items
- **Reactivity**: listen for `vb:request:updated` and `vb:folio:updated` custom events to re-compute the feed without a full page reload

## Test Cases

### Happy Path

1. Guest has 3 service requests and 2 folio charges for the active reservation
2. Verify merged list is sorted by `updatedAt` descending
3. Verify only 4 items displayed (the 4 most recent)
4. Verify each request item shows status badge in correct colour
5. Verify each folio item shows amount as "LKR X,XXX"
6. Verify "time ago" text for an item updated 5 minutes ago reads "5 minutes ago"
7. Tap a request item; verify navigation to `#/requests/:requestId`
8. Tap a folio item; verify navigation to `#/bill`
9. Tap "View All"; verify navigation to `#/notifications`

### Edge Cases

- 0 items total: empty state "No recent activity" displayed, section header still visible
- Only folio charges, no requests: only folio items shown (up to 4)
- Only requests, no folio: only request items shown (up to 4)
- More than 4 total items: only 4 most recent shown; "View All" still visible
- Guest with no active reservation: entire Recent Activity section hidden from DOM
- Item updated "just now" (< 1 minute): Day.js fromNow() renders "a few seconds ago"
- Amount of LKR 0: still renders "LKR 0" without errors
