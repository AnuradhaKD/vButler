## User Story

**As a** hotel guest,
**I want** a visually striking hero section at the top of my dashboard that reflects my current stay status,
**So that** I immediately understand where I am in my journey (upcoming, in-stay, or post-stay) and feel a premium, personalised connection to the Browns Hotels & Resorts brand.

---

## Background / Context

Browns Hotels & Resorts positions itself as a luxury boutique hotel group in Sri Lanka. The hero section is the emotional centrepiece of the dashboard — it should evoke the beauty of the property through imagery while delivering personalised, contextually relevant information about the guest's reservation. The content of the hero changes depending on whether the guest has an active stay, an upcoming stay, or has recently checked out, ensuring the experience always feels relevant rather than generic.

---

## Acceptance Criteria

- [ ] The hero section displays the active property's hero image (`property.heroImages[0]`) as a full-width background image
- [ ] When no hero image is available, a brand gradient (`var(--color-brand-gradient)`) is used as the background fallback
- [ ] A dark overlay (`rgba(0,0,0,0.45)`) is applied over the background image to ensure text readability in all lighting conditions
- [ ] The property name is displayed within the hero using Playfair Display font
- [ ] The welcome message reads "Welcome back, [First Name]" using the guest's first name from `vb:profile`
- [ ] **State A — Active Reservation**: A "Day N of Total" pill is shown (e.g., "Day 2 of 4"), calculated from check-in date to check-out date using Day.js
- [ ] **State A — Active Reservation**: A "Checking out [Date]" line is shown beneath the pill (e.g., "Checking out 18 Mar 2026"), formatted `DD MMM YYYY`
- [ ] **State B — Upcoming Reservation**: A "Check-in in N days" pill is shown (e.g., "Check-in in 3 days"), calculated using `dayjs().diff()` from today to check-in date
- [ ] **State B — Upcoming Reservation**: The check-in date is displayed beneath the pill in `DD MMM YYYY` format
- [ ] **State C — Recently Checked Out (last stay within 7 days)**: The message "Thank you for your stay" is displayed
- [ ] **State C — Recently Checked Out**: A "Share your feedback" button is displayed in the hero; tapping it navigates to `#/feedback/:resvId` for the most recent past reservation
- [ ] Hero height is approximately 220px on mobile and 280px on desktop
- [ ] Background image uses `object-fit: cover` and is centred
- [ ] All text in the hero passes WCAG AA contrast ratio against the overlaid background
- [ ] Pill elements use a semi-transparent white background with brand-coloured text

## Technical Notes

- **Hero image loading**: Set `background-image` via inline style binding `:style="{ backgroundImage: 'url(' + heroImage + ')' }"` in Alpine.js; `heroImage` computed from `activeProperty.heroImages[0]` or `null`
- **Day calculation (State A)**:
  ```js
  const dayN = dayjs().diff(dayjs(resv.checkInDate), 'day') + 1;
  const totalDays = dayjs(resv.checkOutDate).diff(dayjs(resv.checkInDate), 'day');
  ```
- **Countdown (State B)**:
  ```js
  const daysUntil = dayjs(resv.checkInDate).diff(dayjs(), 'day');
  ```
- **State C detection**: Most recent reservation has `status: 'checked_out'` and `checkOutDate` is within 7 days of today: `dayjs().diff(dayjs(resv.checkOutDate), 'day') <= 7`
- **State priority**: Active > Upcoming > Checked Out > No reservations (handled by US-04.6)
- **Data sources**: `StorageService.getActiveReservation()` for active; `StorageService.getUpcomingReservation()` for upcoming; `StorageService.getRecentCheckout()` for post-stay
- **Guest first name**: `StorageService.get('vb:profile')?.firstName`
- **Image performance**: Lazy-load hero image with a low-res placeholder or CSS gradient shown until image resolves; use `loading="lazy"` or CSS `background-image` with transition

## Test Cases

### Happy Path

1. Log in as guest "Nadia" with an active reservation at "The Oaks Kandy", check-in 13 Mar 2026, check-out 17 Mar 2026; today is 15 Mar 2026
2. Verify hero background is the property's hero image with dark overlay
3. Verify property name "The Oaks Kandy" in Playfair Display
4. Verify welcome message "Welcome back, Nadia"
5. Verify pill reads "Day 3 of 4"
6. Verify supporting line "Checking out 17 Mar 2026"
7. Switch guest to upcoming reservation (check-in 20 Mar 2026, today 15 Mar 2026)
8. Verify pill reads "Check-in in 5 days"
9. Verify check-in date line "20 Mar 2026"
10. Switch guest to checked-out status (checked out 12 Mar 2026, today 15 Mar 2026)
11. Verify "Thank you for your stay" message and "Share your feedback" button
12. Tap feedback button; verify navigation to `#/feedback/:resvId`

### Edge Cases

- Day 1 of stay (checked in today): pill shows "Day 1 of N"
- Single-night stay: pill shows "Day 1 of 1", checkout line shows same date
- Check-in tomorrow: pill shows "Check-in in 1 day" (singular)
- Check-out date is today: pill shows "Day N of N", checkout line shows today's date
- Checked out exactly 7 days ago: feedback state still shown
- Checked out 8 days ago: falls through to no-active-stay empty state (US-04.6)
- No hero image available: brand gradient fallback renders, no broken image artifacts
- Very long property name: text truncates gracefully, does not overflow hero bounds
- Guest profile has no first name: welcome message falls back to "Welcome back" without a name
