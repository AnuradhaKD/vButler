## User Story

**As a** hotel guest,
**I want** a clear, branded header fixed at the top of my dashboard,
**So that** I always know which property I am at, can access notifications at a glance, and can navigate to my profile without scrolling.

---

## Background / Context

The Destinity vButler app serves guests across multiple Browns Hotels & Resorts properties in Sri Lanka. The header is the primary trust and orientation element — guests need to see the correct property name immediately on load and must be able to reach key functions (notifications, profile) with a single tap regardless of how far they have scrolled. The dark/light mode toggle is surfaced here because it is a global preference that guests may change at any time.

---

## Acceptance Criteria

- [ ] Browns Hotels & Resorts SVG logo is displayed in the top-left of the header
- [ ] The active property name is displayed directly below or beside the logo (sourced from the active reservation's `property.name`)
- [ ] A notification bell icon is displayed in the top-right area of the header
- [ ] A red badge showing the unread notification count is overlaid on the bell icon
- [ ] The red badge is hidden (not merely `0`) when the unread count is zero
- [ ] A guest avatar is displayed in the top-right area, adjacent to the notification bell
- [ ] When the guest has no profile photo, the avatar renders as a filled circle displaying the guest's initials (first + last name initial)
- [ ] When a profile photo exists, the avatar renders the photo cropped to a circle
- [ ] Tapping the avatar navigates the app to the route `#/profile`
- [ ] A dark/light mode toggle icon is present in the header
- [ ] Toggling dark/light mode applies the theme immediately and persists the preference in `localStorage` under key `vb:theme`
- [ ] The header is sticky: it remains visible and fixed at the top of the viewport as the user scrolls down the dashboard
- [ ] The header does not obscure page content (appropriate `padding-top` applied to the scroll container)
- [ ] Header renders correctly on mobile (375px+) and desktop (1024px+) viewports
- [ ] All header elements are accessible with correct ARIA labels (`aria-label="Notifications"`, `aria-label="My Profile"`, `aria-label="Toggle theme"`)

## UI / UX Notes

- Header height: 56px mobile, 64px desktop
- Background: `var(--color-surface-elevated)` with a subtle bottom border `1px solid var(--color-border)`
- Apply a `backdrop-filter: blur(8px)` with semi-transparent background when overlapping hero content for a premium feel
- Browns Hotels & Resorts logo: max-height 28px, preserves aspect ratio; use `<img>` with the SVG src
- Property name: `font-size: 0.75rem`, `font-weight: 600`, `color: var(--color-text-secondary)`, truncated with ellipsis if long
- Notification bell: 24×24px icon (Heroicons `bell`); badge is a 16px red pill absolutely positioned at top-right of the bell icon, font-size 10px, max display `99+`
- Guest avatar circle: 36px diameter, `background: var(--color-brand-primary)`, initials in white, `font-size: 0.875rem`, `font-weight: 700`
- Dark/light toggle: 24×24px icon — use `sun` icon in dark mode, `moon` icon in light mode; positioned between avatar and bell, or as leftmost right-group icon
- Spacing between right-group icons: 16px gap
- z-index: 100 to ensure header sits above all other content layers

## Technical Notes

- **Framework**: Alpine.js `x-data` component (`headerStore`) defined in `src/js/components/header.js`
- **Property name**: Read from `StorageService.getActiveReservation()` → `reservation.property.name`; fall back to `StorageService.get('vb:activeProperty')?.name`
- **Notification count**: Subscribe to `NotificationService.getUnreadCount()`; reactively update Alpine.js `unreadCount` data property via a `setInterval` poll (every 30s) or custom event `vb:notification:new`
- **Avatar initials**: Derive from `StorageService.get('vb:profile')` → `profile.firstName[0] + profile.lastName[0]`, uppercase; render profile photo if `profile.photoBase64` is present
- **Navigation**: Use the app router's `navigate('#/profile')` helper on avatar click
- **Dark mode**: Toggle adds/removes class `dark` on `<html>` element; persist choice with `localStorage.setItem('vb:theme', 'dark'|'light')`; apply on app init before first paint to prevent flash
- **Sticky behaviour**: `position: sticky; top: 0` on the `<header>` element; ensure parent containers do not have `overflow: hidden` which would break stickiness

## Test Cases

### Happy Path

1. Launch the app with an active reservation for "Cinnamon Red Colombo"
2. Verify the Browns Hotels & Resorts logo appears top-left
3. Verify "Cinnamon Red Colombo" appears beneath/beside the logo
4. Verify notification bell is visible top-right with no red badge (unread = 0)
5. Trigger a new unread notification; verify red badge appears with count "1"
6. Verify guest avatar displays initials "A B" for guest "Anuradha Bandara"
7. Tap the avatar; verify navigation to `#/profile`
8. Tap the dark/light toggle; verify theme switches and persists on page reload
9. Scroll down 300px; verify header remains fixed at top of viewport

### Edge Cases

- Guest with a single-word name (no last name): avatar displays first initial only
- Unread count of 100 or more: badge displays "99+"
- Very long property name (>30 chars): truncates with ellipsis, does not wrap header to two lines
- Profile photo set: avatar shows photo, not initials
- No active reservation: property name area is blank or shows chain name "Browns Hotels & Resorts"
- App loaded in dark mode (persisted): dark theme applied before first paint, no flash of light theme
