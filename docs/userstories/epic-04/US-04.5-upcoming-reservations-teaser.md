## User Story

**As a** hotel guest with multiple bookings across Browns Hotels & Resorts properties,
**I want** to see a compact strip of my upcoming reservations on the dashboard,
**So that** I can quickly review all my planned stays and jump to any reservation's detail without navigating to a separate reservations list.

---

## Background / Context

Loyal guests of Browns Hotels & Resorts frequently hold multiple reservations — a current stay alongside future bookings at different properties. The teaser strip provides a lightweight, glanceable view of these future stays directly on the dashboard, reinforcing the guest's broader relationship with the brand. The strip is intentionally minimal to avoid cluttering the dashboard for guests with only a single booking.

---

## Acceptance Criteria

- [ ] The teaser strip is displayed ONLY when the guest has 2 or more non-cancelled reservations in their reservation list
- [ ] The strip is hidden when the guest has 0 or 1 non-cancelled reservation
- [ ] The strip displays a horizontal scrollable row of compact reservation cards
- [ ] Each compact card shows: property name, check-in and check-out date range, and a status badge
- [ ] Status badge values and labels: `upcoming` → "Upcoming", `active` → "In Stay", `checked_out` → "Completed"
- [ ] Cancelled reservations (`status: "cancelled"`) are excluded from the strip
- [ ] The strip includes a "View All Stays" link that navigates to `#/reservations`
- [ ] The strip is horizontally scrollable and does not wrap cards to a second row
- [ ] Cards snap to position on scroll (CSS `scroll-snap-type: x mandatory`)
- [ ] Tapping a compact card navigates to `#/reservations/:resvId`
- [ ] The currently active reservation (if present) is included in the strip cards and visually distinguished with a brand-coloured left border or highlighted status badge

## UI / UX Notes

- **Section header**: "Your Stays" label (`font-size: 0.875rem`, `font-weight: 700`) left-aligned; "View All Stays" link right-aligned in brand colour
- **Compact card dimensions**: approximately 160px wide × 80px height; `border-radius: 10px`; `background: var(--color-surface-card)`; `padding: 10px 12px`
- **Card content layout**: property name at top (`font-size: 0.75rem`, `font-weight: 700`, single line with ellipsis); date range below (`font-size: 0.6875rem`, `color: var(--color-text-secondary)`, format "DD MMM – DD MMM YYYY"); status badge at bottom-left
- **Status badge**: `font-size: 0.625rem`, `font-weight: 600`, `border-radius: 999px`, `padding: 2px 7px`; "Upcoming" → blue tint, "In Stay" → brand primary tint, "Completed" → grey tint
- **Active reservation card distinction**: `border-left: 3px solid var(--color-brand-primary)`
- **Horizontal scroll container**: `display: flex`, `gap: 10px`, `overflow-x: auto`, `-webkit-overflow-scrolling: touch`, `padding: 4px 0 12px`; hide scrollbar via `::-webkit-scrollbar { display: none }`
- **Scroll snap**: `scroll-snap-type: x mandatory` on container; `scroll-snap-align: start` on each card
- **Strip margin**: 24px top margin from the section above; positioned below the Recent Activity Feed

## Technical Notes

- **Data source**: `StorageService.get('vb:reservations')` returns all reservations array; filter with:
  ```js
  const visibleReservations = reservations.filter(r => r.status !== 'cancelled');
  ```
- **Visibility condition**: `visibleReservations.length >= 2`
- **Sort order**: active reservation first, then upcoming sorted by `checkInDate` ascending, then completed sorted by `checkOutDate` descending
- **Alpine.js component**: `x-data="reservationsTeaser()"` in `src/js/components/reservationsTeaser.js`
- **Template**:
  ```html
  <template x-if="visibleReservations.length >= 2">
    <section x-data="reservationsTeaser()"> ... </section>
  </template>
  ```
- **Navigation**: `@click="navigate('#/reservations/' + resv.id)"`
- **Date formatting**: `dayjs(resv.checkInDate).format('DD MMM')` + ` – ` + `dayjs(resv.checkOutDate).format('DD MMM YYYY')`
- **Active reservation identification**: compare `resv.id === activeReservation.id`

## Test Cases

### Happy Path

1. Guest has 3 reservations: 1 active, 1 upcoming, 1 completed (none cancelled)
2. Verify the teaser strip is visible with the section header "Your Stays"
3. Verify 3 compact cards are rendered in the correct sort order
4. Verify active reservation card has brand-coloured left border
5. Verify each card shows property name, date range, and status badge
6. Scroll the strip horizontally to verify scrollability and snap behaviour
7. Tap the upcoming reservation card; verify navigation to `#/reservations/:resvId`
8. Tap "View All Stays"; verify navigation to `#/reservations`

### Edge Cases

- Guest with exactly 1 reservation: strip is entirely hidden (not rendered, no empty space)
- Guest with 0 reservations: strip is hidden
- Guest has 2 reservations but one is cancelled: only 1 visible; strip hidden
- Guest has 2 non-cancelled reservations: strip shown with 2 cards
- Very long property name (>25 chars): truncates with ellipsis on single line
- Many reservations (10+): horizontal scroll works correctly; all cards accessible
- No active reservation, only upcoming: no active card highlighted; upcoming card is first
