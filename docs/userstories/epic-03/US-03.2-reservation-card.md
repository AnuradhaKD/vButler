## User Story

**As a** guest,
**I want** to see a concise summary card for each reservation showing the key details at a glance,
**So that** I can quickly identify a stay and take the appropriate next action without opening the full detail view.

---

## Background / Context

The reservation card is the primary repeating unit of the Stays list. It needs to communicate the essentials — where, when, and what status — within a compact card footprint suitable for mobile. The CTA button varies by status, ensuring the guest is always offered the most contextually relevant action for their current booking lifecycle stage. A well-executed card design reinforces the premium Browns brand through thoughtful typography, photography, and spacing.

---

## Acceptance Criteria

- [ ] Each reservation card displays: property thumbnail image (or brand-colour placeholder if no image URL), property name, reservation/booking reference number, room type, check-in date, check-out date, number of adults and children, and a `StatusBadge` component
- [ ] Property image renders as a fixed-height strip at the top of the card (`h-36` or `h-40`); `object-cover`; if image unavailable or fails to load, show a `#003c52` background placeholder with the Destinity vButler monogram centred
- [ ] Property name is displayed in Playfair Display font, prominent (e.g., `text-lg font-semibold`)
- [ ] Reservation reference number is displayed in a monospaced or distinct small style (e.g., `text-xs text-gray-500 font-mono`)
- [ ] Room type displayed as `text-sm text-gray-600 dark:text-gray-300`
- [ ] Dates formatted as human-readable: "15 Mar 2026 → 18 Mar 2026" (or equivalent locale-friendly format); total nights may optionally be shown in parentheses "3 nights"
- [ ] Guest count displayed as e.g., "2 adults, 1 child" or "2 adults" if no children; uses appropriate singular/plural
- [ ] `StatusBadge` component rendered with the correct status variant (see US-01.7)
- [ ] CTA button text varies by status:
  - `active` → "Manage Stay"
  - `upcoming` → "View Details"
  - `checked_out` / `completed` → "View Bill"
  - `cancelled` → "View Details" (button is disabled, greyed out)
- [ ] Tapping the card body OR the CTA button navigates to `#/reservation/:id` (Reservation Detail View, US-03.3)
- [ ] Cancelled reservations: the entire card may have reduced opacity (`opacity-75`) and CTA button is visually disabled (`disabled` attribute, `cursor-not-allowed`)
- [ ] Cards in the list are sorted as defined by the parent Stays list view (US-03.1)

## UI / UX Notes

- **Card shape:** `rounded-xl shadow-sm border border-gray-100 dark:border-gray-700 bg-white dark:bg-gray-800 overflow-hidden`
- **Image area:** Top of card, full card width, height 144–160px; `object-cover object-center`; lazy-loaded with `loading="lazy"`
- **Placeholder image:** CSS `background: linear-gradient(135deg, #003c52, #00516e)` with a centred white icon or Destinity vButler initial; no broken image icon
- **Property name overlay option:** Property name may be overlaid at the bottom of the image area with a gradient scrim (`from-transparent to-black/60`) and `text-white`; alternatively name can appear below the image — choose one style and apply consistently
- **Card body:** Padding `p-4`; reservation number in `text-xs text-gray-400 font-mono` above the property name; dates row with a `→` separator or `calendar_month` icon; guest count row with `person` icon; `StatusBadge` on the right side of the property name row or in the card header
- **CTA button:** Full-width at card bottom, `h-10`, `bg-brand text-white` for active/primary actions; `bg-white border border-brand text-brand` for secondary (upcoming/past); disabled state: `bg-gray-100 text-gray-400 cursor-not-allowed`
- **Active cards:** May have a subtle brand colour top border (4px `border-t-4 border-brand`) to visually distinguish currently checked-in stays
- **Hover/press state:** Subtle `shadow-md` on hover (desktop); `scale-[0.99]` active press on mobile via `active:scale-[0.99] transition-transform`

## Technical Notes

- **File:** `js/components/reservation-card.component.js` (or template defined inline in the stays page)
- **Alpine.js usage:** The card is rendered inside `x-for="reservation in filteredReservations" :key="reservation.id"` in the stays page
- **Passing data:** Each card receives the full reservation object as its data context; access via `reservation.propertyName`, `reservation.checkIn`, etc.
- **Navigation on card tap:**
  ```js
  // Prevent double navigation when CTA button is inside the card-click zone
  // Option A: Make entire card an <a> tag with href="#/reservation/{id}"
  // Option B: @click on card wrapper calls Router.navigate, CTA @click.stop then calls same
  ```
- **Date formatting helper:**
  ```js
  function formatDate(isoString) {
    return new Date(isoString).toLocaleDateString('en-GB', {
      day: 'numeric', month: 'short', year: 'numeric'
    });
  }
  // Renders: "15 Mar 2026"
  ```
- **Plural helper:**
  ```js
  function guestCountLabel(adults, children) {
    const a = `${adults} adult${adults !== 1 ? 's' : ''}`;
    const c = children > 0 ? `, ${children} child${children !== 1 ? 'ren' : ''}` : '';
    return a + c;
  }
  ```
- **Image error fallback:** `<img @error="$event.target.style.display='none'; $event.target.nextElementSibling.style.display='flex'">` + a hidden fallback `<div>` placeholder
- **Reservation data shape (relevant fields):**

## Test Cases

### Happy Path

1. Active reservation card renders with brand teal top border accent, "Manage Stay" button in `bg-brand`, property image loaded, correct StatusBadge "Active" (green)
2. Upcoming reservation card renders with "View Details" button (outlined brand style), "Upcoming" badge (blue)
3. Past/checked-out card renders with "View Bill" button, "Checked Out" badge (grey)
4. Tap card body (not CTA) → navigates to `#/reservation/res-001`
5. Tap "Manage Stay" CTA → navigates to `#/reservation/res-001`

### Edge Cases

- Property image URL is null or blank → fallback placeholder shown (`#003c52` gradient background with monogram); no broken image icon
- Property image 404 → `@error` handler triggers; fallback placeholder renders
- Adults = 1, Children = 0 → "1 adult" (singular, no children shown)
- Adults = 2, Children = 1 → "2 adults, 1 child" (singular child)
- Cancelled card → `opacity-75`; CTA button disabled (`cursor-not-allowed`); tapping disabled CTA does not navigate
- Very long property name (> 30 chars) → truncates with `truncate` class; no card overflow
- Very long room type description → truncated with `line-clamp-1`
