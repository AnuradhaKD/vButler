## User Story

**As a** guest,
**I want** to view all details of a specific reservation in one place,
**So that** I have everything I need about my stay — room details, dates, property contact, current balance, and quick access to hotel services.

---

## Background / Context

The Reservation Detail View is the operational hub for a guest's stay. It aggregates the information a guest most frequently needs during a hotel visit — check-in and check-out times to share with a taxi driver, the property phone number for a quick call, the current folio balance to avoid billing surprises, and quick links to services and concierge requests. For Upcoming reservations, it also surfaces pre-arrival tasks like special requests. The view must be comprehensive yet scannable.

---

## Acceptance Criteria

- [ ] The detail view is accessible at route `#/reservation/:id`, where `:id` is the reservation's `id` field
- [ ] If the reservation ID from the URL does not match any reservation in `$store.reservations.list`, show an `EmptyState` with "Reservation not found" and a "Back to My Stays" CTA
- [ ] **Hero section:** Property image (full-width, `h-48` or `h-56`); property name overlaid at bottom of image in Playfair Display, white text, with a dark gradient scrim; `StatusBadge` positioned in the top-right corner of the hero image
- [ ] **Stay Details section:** displays — Check-in date and time (formatted: "Saturday, 15 March 2026 at 14:00"), Check-out date and time (formatted: "Tuesday, 18 March 2026 at 11:00"), Room Number, Room Type, Number of Adults, Number of Children (omit row if 0), Total Nights (computed from check-in and check-out dates)
- [ ] Total Nights must be **computed** from check-in and check-out dates (not stored); formula: `Math.ceil((checkOut - checkIn) / 86400000)` — display as "3 nights"
- [ ] **Property Info section:** Full property address (multi-line), property phone number (tappable `tel:` link on mobile), property email (tappable `mailto:` link)
- [ ] **Booking Info section:** Booking reference number, booking source (e.g., "Direct Booking", "Booking.com"), total number of guests (adults + children combined)
- [ ] **Special Requests section:** Displays the `specialRequests` text if present; shows "None" in muted style if the field is empty or null
- [ ] **Current Balance section:** Shows the current folio balance in the property's currency (e.g., "LKR 45,000"); if balance is 0, shows "Paid in Full" in green; if balance > 0, shows the amount in `text-amber-600` with a "Pay Now" button (Phase 1: "Pay Now" shows an info toast "Online payment coming soon")
- [ ] **Quick Links row:** Horizontally scrollable row of shortcut cards/buttons: "Services" → `#/services`, "My Bill" → `#/bill`, "Complaints" → `#/complaints`; for `upcoming` status reservations only, also show "Pre-Arrival" → `#/pre-arrival`
- [ ] For `cancelled` reservations: all action buttons and Quick Links are disabled; a prominent notice banner "This reservation has been cancelled" displayed in `bg-red-50 dark:bg-red-900/20 text-red-700 dark:text-red-400` at the top of the content below the hero
- [ ] A Back button (`arrow_back`) in the header navigates to `#/stays`

## UI / UX Notes

- **Page structure (top to bottom):** Back button header → Hero image with property name + status badge → Cancellation notice (if applicable) → Stay Details card → Property Info card → Booking Info card → Special Requests card → Current Balance card → Quick Links row → (bottom navigation)
- **Section cards:** Each section is a white/dark card with `rounded-xl p-4 shadow-sm`; section header in `text-xs font-semibold uppercase tracking-wider text-gray-400` (like a form section label); content rows use `flex justify-between items-start` with a label on the left and value on the right
- **Hero image:** `object-cover object-center`; brand gradient scrim `bg-gradient-to-t from-black/70 to-transparent` overlaid at bottom; property name in `font-display text-xl font-semibold text-white` bottom-left; StatusBadge top-right `absolute top-3 right-3`
- **Phone and email:** On mobile, `<a href="tel:+94114770000">` makes the number directly dialable; on desktop, display as styled text link
- **Quick Links:** Each shortcut is a `flex flex-col items-center gap-1` card with an icon (Material Symbol, 28px), short label (`text-xs`); rendered in a `flex gap-3 overflow-x-auto` row; each card `min-w-[72px] bg-surface-light dark:bg-gray-700 rounded-xl p-3`
- **Balance section:** Large amount display `text-2xl font-bold`; "Paid in Full" in `text-green-600 dark:text-green-400 font-semibold`; "Pay Now" button `bg-brand text-white rounded-lg px-4 py-2 text-sm font-medium`

## Technical Notes

- **File:** `js/pages/reservation-detail.js`
- **Route parameter extraction:**
  ```js
  // In Router, parse :id from hash
  const id = window.location.hash.split('/')[2]; // '#/reservation/res-001' → 'res-001'
  ```
- **Data lookup:**
  ```js
  const reservation = Alpine.store('reservations').list.find(r => r.id === id);
  if (!reservation) this.notFound = true;
  ```
- **Total nights computation:**
  ```js
  get totalNights() {
    const checkIn = new Date(this.reservation.checkIn);
    const checkOut = new Date(this.reservation.checkOut);
    return Math.ceil((checkOut - checkIn) / (1000 * 60 * 60 * 24));
  }
  ```
- **Date/time formatting:**
  ```js
  function formatDateTime(isoString) {
    return new Date(isoString).toLocaleString('en-GB', {
      weekday: 'long', day: 'numeric', month: 'long', year: 'numeric',
      hour: '2-digit', minute: '2-digit'
    });
  }
  // Renders: "Saturday, 15 March 2026 at 14:00"
  ```
- **Currency formatting:**
  ```js
  function formatCurrency(amount, currency) {
    return new Intl.NumberFormat('en-LK', { style: 'currency', currency }).format(amount);
  }
  ```
- **Reservation data shape (extended fields for this view):**

## Test Cases

### Happy Path

1. Navigate from a reservation card to `#/reservation/res-001` → hero image loads; property name in Playfair Display; status badge visible; all 7 sections render with correct data
2. Total nights computed correctly: check-in 15 Mar, check-out 18 Mar → "3 nights" displayed
3. Tap property phone number on mobile → device dialer opens with pre-filled number
4. Balance = LKR 45,000 → amount shown in amber; "Pay Now" button visible; tap "Pay Now" → info toast "Online payment coming soon"
5. Balance = 0 → "Paid in Full" shown in green; no "Pay Now" button
6. Upcoming reservation → Pre-Arrival quick link visible in Quick Links row
7. Active reservation → Pre-Arrival quick link not shown

### Edge Cases

- Navigate to `#/reservation/nonexistent-id` → "Reservation not found" EmptyState with "Back to My Stays" CTA
- Cancelled reservation → cancellation notice banner at top; Quick Links row disabled or hidden; action buttons disabled
- `children = 0` → Children row omitted from Stay Details; guest count shows "2 adults" only
- `specialRequests` is null/empty → "None" displayed in muted style; section still shows
- Very long `specialRequests` text (500+ chars) → wraps naturally within card; no overflow
- Route parameter missing (navigating to `#/reservation/` with no ID) → treated as not found; EmptyState shown
- Back button tapped → navigates to `#/stays`; correct tab active (should show the tab matching this reservation's status)
