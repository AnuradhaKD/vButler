## User Story

**As a** developer building the Destinity vButler app,
**I want** a robust i18n (internationalisation) service that loads translation files and provides translated strings to templates,
**So that** the entire guest-facing UI can be displayed in the guest's chosen language without code duplication or hardcoded strings.

---

## Background / Context

Browns Hotels & Resorts hosts guests from Sri Lanka, Europe, the Middle East, and beyond. Supporting Sinhala, Tamil, Arabic, and German alongside English is essential for making all guests feel welcomed. The i18n service is the foundational infrastructure that powers the language selector (US-18.2), RTL layout (US-18.3), and all translated UI strings across every screen. It must be efficient (JSON caching), flexible (template variables), and resilient (fallback to English).

---

## Acceptance Criteria

- [ ] `I18nService` is an ES module exported from `/js/services/I18nService.js`.
- [ ] `I18nService.load(langCode)` fetches the translation file from `/data/i18n/{langCode}.json` and caches it in memory (object property). Subsequent calls for the same `langCode` return the cached object without a network fetch.
- [ ] `I18nService.t(key)` returns the translated string for a given dotted key (e.g., `'auth.login.title'`) in the currently active language.
- [ ] `I18nService.t(key, vars)` supports template variable substitution: `I18nService.t('dashboard.welcome', { name: 'Anuradha' })` returns `"Welcome back, Anuradha"` (replaces `{{name}}` in the translation string).
- [ ] If a key is missing in the current language's JSON, `t()` falls back to the English (`en.json`) value for that key.
- [ ] If a key is missing in both the current language and English, `t()` returns the key string itself (e.g., `'dashboard.missingKey'`) as a last-resort fallback, and logs a console warning.
- [ ] `I18nService.setLanguage(code)` loads the new language JSON (if not cached), sets it as the active language, saves `code` to `vb:language` in localStorage, and updates `Alpine.store('ui').language`.
- [ ] On app boot, the language is determined in this priority order: (1) `vb:language` localStorage value, (2) `navigator.language` match to a supported language code, (3) `'en'` default.
- [ ] All static UI strings across every screen are bound to translation keys — no hardcoded English strings in HTML templates (except for proper nouns like "Browns Hotels & Resorts").
- [ ] Alpine.js templates use `$t('key')` magic helper to call `I18nService.t()`.
- [ ] The `data-i18n="key"` attribute approach is supported as an alternative for non-Alpine HTML elements (processed by `I18nService.applyStaticTranslations()`).
- [ ] `I18nService.getSupportedLanguages()` returns the array of supported language objects: `[{ code, name, nativeName, flag, rtl }]`.

---

## UI / UX Notes

- This is primarily a developer-facing service with no direct UI. Its impact is the translated content displayed across all screens.
- Language load time: JSON files should be small (<20KB each); loading should be imperceptible to the user. A subtle loading state on the language selector (spinner for 200ms) is acceptable.
- Template variables should feel natural: `{{name}}`, `{{amount}}`, `{{date}}` placeholders in JSON strings.
- Fallback behaviour must be silent to the guest — they see English text, not a missing-key error or raw key string.

## Technical Notes

- Module structure:
  ```js
  // /js/services/I18nService.js
  const SUPPORTED = [
    { code: 'en', name: 'English',   nativeName: 'English',   flag: '🇬🇧', rtl: false },
    { code: 'si', name: 'Sinhala',   nativeName: 'සිංහල',     flag: '🇱🇰', rtl: false },
    { code: 'ta', name: 'Tamil',     nativeName: 'தமிழ்',     flag: '🇱🇰', rtl: false },
    { code: 'ar', name: 'Arabic',    nativeName: 'العربية',   flag: '🇸🇦', rtl: true  },
    { code: 'de', name: 'German',    nativeName: 'Deutsch',   flag: '🇩🇪', rtl: false },
  ];

  const cache = {};
  let activeCode = 'en';
  let activeLang = {};
  let enLang = {};

  export const I18nService = {
    async load(code) {
      if (!cache[code]) {
        const res = await fetch(`/data/i18n/${code}.json`);
        cache[code] = await res.json();
      }
      return cache[code];
    },

    async setLanguage(code) {
      activeLang = await this.load(code);
      if (code !== 'en') enLang = await this.load('en');
      activeCode = code;
      StorageService.set('vb:language', code);
      if (window.Alpine) Alpine.store('ui').language = code;
      // RTL handling
      const lang = SUPPORTED.find(l => l.code === code);
      document.documentElement.dir = lang?.rtl ? 'rtl' : 'ltr';
      document.documentElement.lang = code;
      document.body.classList.toggle('rtl', !!lang?.rtl);
      this.applyStaticTranslations();
    },

    t(key, vars = {}) {
      const val = this._resolve(activeLang, key)
        ?? this._resolve(enLang, key)
        ?? (console.warn(`[i18n] Missing key: ${key}`), key);
      return Object.entries(vars).reduce(
        (s, [k, v]) => s.replace(new RegExp(`{{${k}}}`, 'g'), v), val
      );
    },

    _resolve(obj, dotKey) {
      return dotKey.split('.').reduce((o, k) => o?.[k], obj) ?? null;
    },

    applyStaticTranslations() {
      document.querySelectorAll('[data-i18n]').forEach(el => {
        el.textContent = this.t(el.dataset.i18n);
      });
    },

    getSupportedLanguages() { return SUPPORTED; },
    getActiveCode() { return activeCode; }
  };
  ```
- Alpine magic helper registration (in `app.js`):
  ```js
  Alpine.magic('t', () => (key, vars) => I18nService.t(key, vars));
  ```
- App boot sequence (in `app.js`):
  ```js
  const savedLang = StorageService.get('vb:language');
  const navLang = navigator.language?.slice(0, 2);
  const supported = I18nService.getSupportedLanguages().map(l => l.code);
  const initLang = supported.includes(savedLang) ? savedLang
                 : supported.includes(navLang)   ? navLang
                 : 'en';
  await I18nService.setLanguage(initLang);
  Alpine.start(); // start Alpine AFTER i18n is loaded
  ```
- Translation JSON key naming: snake_case for leaf values; camelCase for namespace objects — team must pick one convention. **Decision: camelCase throughout** for consistency with JS.
- `fetch` errors (network failure): catch and fall back to `en.json` cache; log warning.

## Test Cases

### Happy Path

1. On app boot with `vb:language = 'de'` in localStorage, verify all UI strings render in German.
2. Call `I18nService.t('auth.login.title')` — verify returns German translation string.
3. Call `I18nService.t('dashboard.welcome', { name: 'Anuradha' })` — verify `{{name}}` is replaced with "Anuradha".
4. Call `I18nService.setLanguage('ar')` — verify `document.documentElement.dir === 'rtl'` and `body.classList.contains('rtl') === true`.
5. Call `I18nService.setLanguage('en')` — verify `dir === 'ltr'` and `body.classList.contains('rtl') === false`.
6. Load German twice (`setLanguage('de')` called twice) — verify only one `fetch` call (cache used second time).

### Edge Cases

- **Missing key in German, present in English:** `t()` returns English value; no error shown to user.
- **Missing key in both German and English:** `t()` returns the key string (e.g., `'profile.unknownField'`); console warning logged.
- **`navigator.language = 'de-DE'`:** `slice(0, 2)` extracts `'de'`; German loaded on boot.
- **`navigator.language = 'fr'` (unsupported):** Falls back to `'en'`.
- **Network error loading `ta.json`:** Catch block falls back to `en.json`; UI still functional in English; console warning logged.
- **Template variable not in string:** `t('key', { name: 'X' })` where key string has no `{{name}}` — returns the string unchanged; no error.
- **Multiple `{{var}}` in one string:** All instances replaced correctly (global regex replace).
- **Alpine not yet started when `setLanguage` called:** Guard `if (window.Alpine)` prevents crash; language still set in localStorage.
