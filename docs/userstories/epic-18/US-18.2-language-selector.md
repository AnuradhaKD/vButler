## User Story

**As a** hotel guest,
**I want** to change the app language from a visible language selector in the header or settings,
**So that** I can use Destinity vButler in my preferred language without needing to navigate deep into settings or restart the app.

---

## Background / Context

International guests arriving at Browns Hotels & Resorts — particularly Arabic-speaking guests from the Middle East, German tourists, and local Sinhala and Tamil speakers — should be able to switch to their native language immediately on first launch or at any point during their stay. Making the selector available on both the authentication screens (pre-login) and throughout the app (post-login) maximises discoverability and removes the language barrier at the earliest possible moment.

---

## Acceptance Criteria

- [ ] A language selector is available in the **Settings screen** under the Language section (see US-16.3).
- [ ] A language selector is also available in the **app header** as a globe icon (🌐) that opens a dropdown on tap — visible on both pre-authentication (login, register) and post-authentication screens.
- [ ] The selector presents five languages, each showing: flag emoji, native script name, and English name in parentheses.
  - 🇬🇧 English
  - 🇱🇰 සිංහල (Sinhala)
  - 🇱🇰 தமிழ் (Tamil)
  - 🇸🇦 العربية (Arabic)
  - 🇩🇪 Deutsch (German)
- [ ] The currently active language is visually highlighted in the dropdown (e.g., bold text, checkmark icon, or brand-colour background row).
- [ ] On selecting a language, `I18nService.setLanguage(code)` is called, which: loads the translation JSON (if not cached), updates all `$t()` bindings, sets `dir` for RTL if Arabic, and saves to `vb:language`.
- [ ] The language switch takes effect immediately across the entire visible UI without a full page reload.
- [ ] The selected language persists across browser sessions (loaded from `vb:language` on boot).
- [ ] The language selector in the header dropdown closes automatically after a language is selected.
- [ ] The selector is available from the login screen, register screen, and all authenticated screens.
- [ ] When Arabic is selected via the header dropdown, the dropdown itself immediately re-renders in RTL layout.

## UI / UX Notes

- **Header globe icon:** 24px globe SVG icon; 44×44px tap target; positioned in the header actions group alongside the bell and avatar.
- **Dropdown panel (header):** Appears below the globe icon; width 220px; border-radius 12px; box shadow; max-height 280px; scroll if needed. On mobile: full-width bottom-sheet drawer preferred over a small dropdown.
- **Language row (in dropdown/drawer):**
  - Height: 52px.
  - Left: flag emoji (24px) + 12px gap.
  - Centre: native script name (16px, regular) + English name below (12px, muted).
  - Right: checkmark icon (brand colour) if this is the active language.
- **Loading state:** If the language JSON takes > 200ms to load, show a subtle loading spinner inside the globe icon or on the selected row; no skeleton screen needed.
- **Settings screen selector:** Can be a `<select>` element styled via Choices.js or a custom Alpine dropdown matching the header dropdown design.
- **Active language in header closed state:** Show the 2-letter language code (e.g., "EN", "AR", "DE") in a small label next to the globe icon to indicate current language.
- **RTL:** When Arabic is active, the globe icon and its dropdown position correctly for RTL layout; dropdown opens to the left.

## Technical Notes

- Header globe component (Alpine.js):
  ```html
  <div x-data="{ open: false }" class="relative">
    <button @click="open = !open" @click.outside="open = false"
            aria-label="Change language" :aria-expanded="open.toString()">
      🌐 <span x-text="$store.ui.language.toUpperCase()" class="text-xs font-semibold ml-1"></span>
    </button>
    <div x-show="open" x-transition @keydown.escape.window="open = false"
         class="absolute right-0 mt-2 w-56 rounded-xl shadow-lg bg-white dark:bg-gray-800 z-50">
      <template x-for="lang in I18nService.getSupportedLanguages()" :key="lang.code">
        <button @click="selectLanguage(lang.code); open = false"
                class="flex items-center w-full px-4 py-3 text-left hover:bg-gray-50 dark:hover:bg-gray-700">
          <span x-text="lang.flag" class="text-xl mr-3"></span>
          <span class="flex-1">
            <span x-text="lang.nativeName" class="block text-sm font-medium"></span>
            <span x-text="lang.name" class="block text-xs text-gray-500"></span>
          </span>
          <span x-show="$store.ui.language === lang.code" class="text-brand ml-2">✓</span>
        </button>
      </template>
    </div>
  </div>
  ```
- `selectLanguage(code)` Alpine method:
  ```js
  async selectLanguage(code) {
    this.loading = true;
    await I18nService.setLanguage(code);
    this.loading = false;
  }
  ```
- `Alpine.store('ui')` must include a `language` property updated by `I18nService.setLanguage()`.
- Choices.js (if used for Settings dropdown): `new Choices(select, { searchEnabled: false, itemSelectText: '' })`. Style override: match app theme via CSS variables.
- On pre-auth screens (login/register): the header globe icon is still rendered (outside the auth form container); language switch works identically.
- When switching to Arabic, the dropdown's own layout direction updates reactively because it inherits `dir="rtl"` from `<html>`.

## Test Cases

### Happy Path

1. Open app — verify globe icon visible in the header on the login screen.
2. Tap globe icon — verify dropdown/drawer opens with all five languages listed.
3. Tap "Deutsch (German)" — verify all visible UI text switches to German immediately without page reload.
4. Verify "DE" label appears next to the globe icon.
5. Close and reopen the browser — verify German is still active.
6. Navigate to Settings → Language — verify "Deutsch" is shown as selected.
7. From Settings, switch to "සිංහල (Sinhala)" — verify UI updates to Sinhala.
8. Tap globe in header — verify "SI" shown; dropdown shows Sinhala with checkmark.

### Edge Cases

- **Select Arabic (RTL):** Globe dropdown closes; `<html dir="rtl">` set; entire UI mirrors; dropdown re-opens correctly for RTL.
- **Click outside dropdown:** Dropdown closes without selecting; language unchanged.
- **Escape key while dropdown open:** Dropdown closes; language unchanged.
- **JSON load fails for Tamil:** Error caught; language remains on previous selection; error toast "Unable to load Tamil translations. Please try again."; console warning logged.
- **Rapid language switching (click 3 languages quickly):** Last selected language wins; no race condition leaves the UI in a mixed-language state (async/await ensures sequential resolution).
- **Pre-auth screen language switch:** Guest selects German on login screen → registers → first authenticated screen is in German.
- **Active language item tapped:** `setLanguage(code)` called with same code; no visible change; no network re-fetch (cached).
