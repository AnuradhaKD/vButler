## User Story

**As a** hotel guest currently checked in,
**I want** to browse the full catalog of available hotel services organised by category,
**So that** I can discover and request any service offered at my Browns Hotels & Resorts property without needing to call the front desk.

---

## Background / Context

The Services Catalog is the primary self-service channel for in-stay guest requests at Browns Hotels & Resorts. By presenting services in a browsable, categorised format — driven by each property's own data configuration — the app replaces the traditional in-room compendium and phone-based request process. Restricting catalog access to active stays ensures guests are not confused by seeing services they cannot yet request, while the category filter system makes it easy to find what they need quickly.

## Acceptance Criteria

- [ ] The Services Catalog screen is accessible only for reservations with `status: "active"`
- [ ] If accessed without an active reservation (upcoming, checked-out, or no reservation), a state message is displayed: "Services are available during your stay" with a back navigation option
- [ ] Services data is loaded from the active reservation's property: `DataService.getProperty(activeReservation.propertyId)` → `property.services.categories[]`
- [ ] A horizontal scrollable filter chip row is displayed at the top of the catalog, containing: "All" chip + one chip per service category (e.g., "Housekeeping", "Dining", "Wellness", "Concierge")
- [ ] "All" chip is selected by default on first load
- [ ] When "All" is selected, all service categories and their cards are displayed
- [ ] When a category chip is selected, only the services in that category are displayed; all other categories are hidden
- [ ] Category chips are mutually exclusive (selecting one deselects the previous)
- [ ] Each category section renders a section header with the category icon and category name
- [ ] Service cards are rendered within each category section (per US-06.2)
- [ ] The catalog scrolls vertically through all categories when "All" is selected
- [ ] The page title is "Services" with a back navigation arrow
- [ ] If `property.services.categories` is empty or undefined, an empty state is shown: "No services available for this property at this time"

## Technical Notes

- **Route**: `#/services`
- **Alpine.js page component**: `x-data="servicesCatalog()"` in `src/js/pages/services.js`
- **Data loading**:
  ```js
  async init() {
    const resv = StorageService.getActiveReservation();
    if (!resv || resv.status !== 'active') { this.noActiveStay = true; return; }
    this.property = await DataService.getProperty(resv.propertyId);
    this.categories = this.property.services?.categories || [];
    this.activeResvId = resv.id;
  }
  ```
- **Property JSON structure**:
  ```json
  {
    "services": {
      "categories": [
        {
          "id": "housekeeping",
          "name": "Housekeeping",
          "icon": "sparkles",
          "services": [ ...service objects... ]
        }
      ]
    }
  }
  ```
- **Filter logic**:
  ```js
  get filteredCategories() {
    if (this.selectedCategory === 'all') return this.categories;
    return this.categories.filter(c => c.id === this.selectedCategory);
  }
  ```
- **Chip rendering**: `x-for="chip in chips"` where `chips = [{ id: 'all', name: 'All' }, ...categories.map(c => ({ id: c.id, name: c.name }))]`
- **Category icon rendering**: map `category.icon` string to Heroicons SVG `<use href="#icon-{category.icon}">`
- **DataService**: `getProperty(propertyId)` fetches from `data/properties/{propertyId}.json` via `fetch()`; cache in memory to avoid re-fetching on filter change

## Test Cases

### Happy Path

1. Navigate to `#/services` with an active reservation at a property with 4 service categories
2. Verify "All" chip is selected; all 4 categories and their service cards are visible
3. Tap the "Wellness" chip; verify only Wellness category section is visible
4. Tap "All"; verify all categories return
5. Scroll vertically through all categories when "All" selected

### Edge Cases

- No active reservation: "Services available during your stay" state shown; no catalog rendered
- Active reservation but property has no services configured: "No services available" empty state
- Property has 8 categories: filter chip row scrolls horizontally; all 8 chips accessible
- Single category selected with 0 services (edge case in data): category header shown with empty card area; or category hidden if no services present
