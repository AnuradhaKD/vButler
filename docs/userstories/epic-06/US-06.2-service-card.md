## User Story

**As a** hotel guest browsing the services catalog,
**I want** each service presented as a clear, informative card showing its name, description, price, availability, and estimated delivery time,
**So that** I can make an informed decision about whether to request a service before tapping through to the request form.

---

## Background / Context

The service card is the primary unit of the services catalog and must communicate enough information for a guest to understand what is being offered, what it will cost, when it is available, and approximately how long it will take. For a luxury hotel app, the card design must feel premium and scannable — guests should be able to assess multiple services at a glance without needing to open each one individually. The availability state prevents guests from submitting requests for services that are currently unavailable.

---

## Acceptance Criteria

- [ ] Each service card displays: service name, description text (2-line truncate with expand), price or availability indicator, estimated delivery time, and available hours pill
- [ ] **Service name**: displayed prominently as the card heading
- [ ] **Description**: truncated to 2 lines by default; a "Show more" inline link expands the full description; "Show less" collapses it
- [ ] **Price display** follows these rules:
  - `service.price === 0` or `service.pricing === "complimentary"` → display "Complimentary"
  - `service.price > 0` → display "LKR [amount]" formatted with comma thousand separator (e.g., "LKR 1,500")
  - `service.pricing === "folio"` → display "Charges apply to folio"
- [ ] **Estimated delivery time**: displayed as a label (e.g., "~20 mins") with a clock icon; sourced from `service.estimatedTime`
- [ ] **Available hours pill**: displays the service's operating window (e.g., "Available 7:00 AM – 10:00 PM"); sourced from `service.availableHours.start` and `.end`
- [ ] **Availability check**: the current time is compared to `service.availableHours.start` and `.end` using Day.js
- [ ] When the current time is **within** available hours: the "Request" button is displayed in enabled state (brand primary colour)
- [ ] When the current time is **outside** available hours: a "Currently Unavailable" overlay label is shown on the card; the "Request" button is disabled (`opacity: 0.5`, `cursor: not-allowed`, `pointer-events: none`)
- [ ] Services not present in the active property's `property.json` are never rendered
- [ ] Tapping the enabled "Request" button navigates to `#/services/:serviceId/request`

## Technical Notes

- **Alpine.js component**: `x-data="serviceCard(service)"` where `service` is the service object passed as a prop via `x-data`
- **Availability check** (runs on init and refreshed every minute via `setInterval`):
  ```js
  isAvailable() {
    const now = dayjs();
    const start = dayjs(`${now.format('YYYY-MM-DD')} ${this.service.availableHours.start}`, 'YYYY-MM-DD HH:mm');
    const end = dayjs(`${now.format('YYYY-MM-DD')} ${this.service.availableHours.end}`, 'YYYY-MM-DD HH:mm');
    return now.isAfter(start) && now.isBefore(end);
  }
  ```
- **24/7 availability**: if `service.availableHours` is `{ "start": "00:00", "end": "23:59" }` or `null`, the service is always available; skip the availability check
- **Price formatting**:
  ```js
  priceDisplay() {
    if (this.service.pricing === 'complimentary' || this.service.price === 0) return 'Complimentary';
    if (this.service.pricing === 'folio') return 'Charges apply to folio';
    return 'LKR ' + this.service.price.toLocaleString('en-LK');
  }
  ```
- **Available hours display**:
  ```js
  hoursDisplay() {
    if (!this.service.availableHours) return 'Available 24/7';
    const fmt = t => dayjs(`2000-01-01 ${t}`).format('h:mm A');
    return `Available ${fmt(this.service.availableHours.start)} – ${fmt(this.service.availableHours.end)}`;
  }
  ```
- **Navigation**: `@click="if (isAvailable()) navigate('#/services/' + service.id + '/request')"` — double-guard with `pointer-events: none` on disabled button
- **Description expand**: Alpine.js `expanded` boolean; `:class="{ 'line-clamp-2': !expanded }"` CSS class toggle

## Test Cases

### Happy Path

1. Render a service card for "Extra Towels" at 14:00, available 07:00–22:00
2. Verify service name displayed; description truncated to 2 lines
3. Tap "Show more"; verify full description visible; tap "Show less"; description truncates again
4. Verify price displays "Complimentary" (for a complimentary service)
5. Verify estimated time "~10 mins" with clock icon
6. Verify available hours pill "Available 7:00 AM – 10:00 PM"
7. Verify "Request" button enabled and brand coloured
8. Tap "Request"; verify navigation to `#/services/extra-towels/request`

### Edge Cases

- Current time is 23:00, service available 07:00–22:00: "Currently Unavailable" overlay shown; Request button disabled
- Service price is LKR 2500: displays "LKR 2,500" (comma formatting)
- Service pricing is "folio": displays "Charges apply to folio" in italic
- Service has no `availableHours` (null): displayed as "Available 24/7"; button always enabled
- Very long service name (>50 chars): wraps to second line; card expands vertically without clipping
- Availability check at exactly start time (07:00): service is available (`isAfter` check means service is available from 07:00:01 — note for QA: clarify if `isSameOrAfter` is required)
