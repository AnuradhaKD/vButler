## User Story

**As a** hotel guest,
**I want** to schedule room makeup or turndown service for a preferred time directly from the app,
**So that** I can plan my day around housekeeping visits and ensure my room is serviced at a convenient time.

---

## Background / Context

Guests at luxury hotels value control over when their room is serviced. "Please service my room after 11 AM" or "I'd like turndown at 7 PM" are common requests that currently require phone calls or front-desk visits. Enabling self-service scheduling through Destinity vButler removes this friction, improves guest satisfaction, and allows the housekeeping team to plan their rounds more efficiently. These are two of the highest-frequency housekeeping interactions and are P0 requirements.

## Acceptance Criteria

- [ ] Tapping "Room Makeup" on the Housekeeping Hub opens a scheduling modal (bottom sheet on mobile)
- [ ] Tapping "Turndown Service" on the Housekeeping Hub opens a separate scheduling modal with "Turndown Service" as the title
- [ ] The modal heading clearly identifies the service type: "Room Makeup" or "Turndown Service"
- [ ] The modal contains a preferred time field using Flatpickr, constrained to the service's available hours from `property.json → housekeeping.roomMakeupHours` or `housekeeping.turndownHours` respectively
- [ ] Time picker shows times in 30-minute increments within the available window; past times for today are disabled
- [ ] A special instructions text field (optional, max 200 chars) is included with placeholder "e.g. Please replace towels, extra pillows needed"
- [ ] A "Submit Request" button is present; it is disabled until a preferred time is selected
- [ ] If DND is currently active (read from `$store.dnd.active`): the service tile is visually greyed, tapping it shows a warning: "Do Not Disturb is currently active. Please deactivate DND before scheduling room servicing." No modal opens
- [ ] If current time is outside the service's available hours, the time picker shows a message "Scheduling is unavailable outside service hours ([open]–[close])" and the Submit button is disabled
- [ ] On submit: a request is created in `vb:requests[]` with `category: "housekeeping"`, `type: "room_makeup"` or `"turndown"`, preferred time, and special instructions
- [ ] Success state displayed inline within the modal: green checkmark, "Request submitted — we'll have your room ready by [time]", and reference number (`HK-{timestamp}`)
- [ ] Screen renders correctly in dark mode

## UI / UX Notes

- Modal / bottom sheet: 90% screen width on mobile (centred), max 480 px; drag handle at top; slides up with CSS transition 300 ms
- Modal header: service type icon left (broom for Room Makeup, moon for Turndown), title 18 px Playfair Display, close "×" right
- Flatpickr time picker: inline or triggered by input; brand primary highlight on selected time; 48 px input height
- Special instructions: 90 px textarea, 14 px placeholder, char counter bottom-right
- Submit button: brand primary filled, full-width, 48 px
- DND warning: amber full-width banner inside modal area (or as a toast if modal does not open); bell-slash icon + message text; "Go to DND Settings" link
- Outside-hours message: muted info card with clock icon; no time picker rendered
- Success state: replaces form content inside modal; green checkmark animation, confirmation text, "Close" button

## Technical Notes

- Alpine.js: shared component `x-data="housekeepingSchedule(serviceType)"` accepting `"room_makeup"` or `"turndown"` as parameter
- `serviceType` determines: modal title, icon, hours source (`housekeeping.roomMakeupHours` vs `housekeeping.turndownHours`), and `type` field in request
- DND check: `$store.dnd.active` read in `openModal()` method; if true, set `showDNDWarning = true` and abort modal open
- Available hours enforcement: `generateTimeSlots(open, close, 30)` filtered to future times for today; if array is empty, set `outsideHours = true`
- Reference number: `"HK-" + Date.now()`
- Request schema:
  ```json
  {
    "id": "HK-1700000000000",
    "category": "housekeeping",
    "type": "room_makeup",
    "preferredTime": "11:30",
    "specialInstructions": "Extra pillows please",
    "status": "pending",
    "createdAt": "2026-03-15T08:00:00Z"
  }
  ```
- Modal close: `closeModal()` resets form state (`selectedTime = null`, `instructions = ''`, `successState = false`)

## Test Cases

### Happy Path

1. Guest opens Housekeeping Hub at 08:00; DND is inactive
2. Guest taps "Room Makeup" tile → modal slides up: "Room Makeup" heading, Flatpickr time picker (09:00–17:00 range), special instructions field
3. Guest selects 11:30; adds note "Extra pillows"; taps "Submit Request"
4. Success state: "Request submitted — we'll have your room ready by 11:30" · Ref HK-1710000000000
5. Guest closes modal; Active Requests section on Hub shows new pending request

6. Guest taps "Turndown Service" tile → separate modal opens: "Turndown Service" heading, time picker (18:00–22:00 range)

### Edge Cases

- **DND active**: Tapping Room Makeup tile shows DND warning; modal does not open
- **Current time past all available slots for today**: Outside-hours message shown in modal; Submit disabled
- **Special instructions left blank**: Request submitted successfully without instructions field
- **Guest submits Room Makeup then immediately submits another**: Both requests appended; no deduplication (operational team manages)
- **Available hours span single slot** (e.g., only 09:00 in range): One pill shown; behaviour correct
- **Dark mode**: Modal background, time picker, DND warning, success state verified
