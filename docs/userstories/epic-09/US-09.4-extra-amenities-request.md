## User Story

**As a** hotel guest,
**I want** to request extra amenities ‚Äî such as towels, toiletries, bedding, and room supplies ‚Äî by selecting items from a categorised list and specifying quantities,
**So that** I can get exactly what I need in my room without waiting on hold or leaving my room.

---

## Background / Context

Extra amenity requests are one of the highest-frequency housekeeping interactions at hotel properties. Guests frequently need additional towels, toiletries, or bedding, particularly on longer stays or when travelling with family. A self-service amenity request screen eliminates the phone call friction entirely and allows guests to specify precise quantities, leading to faster and more accurate fulfilment by the housekeeping team.

## Acceptance Criteria

- [ ] The Extra Amenities screen is accessible from the Housekeeping Hub via the "Extra Amenities" tile and also via `?category=towels` pre-filter from the "Extra Towels" tile
- [ ] Amenity items are grouped into four category sections: Towels, Toiletries, Bedding, and Other
- [ ] Items are rendered as multi-select chips within each category; unselected chips have outlined style; selected chips have brand primary filled style
- [ ] When the `?category=towels` query parameter is present on load, the Towels section is automatically expanded and scrolled into view
- [ ] **Towels category chips**: Bath Towel, Pool Towel, Face Towel
- [ ] **Toiletries category chips**: Shampoo, Conditioner, Body Wash, Toothbrush Kit, Razor Kit, Shower Cap
- [ ] **Bedding category chips**: Extra Pillow, Extra Blanket, Baby Cot
- [ ] **Other category chips**: Hangers, Iron & Board, Voltage Adapter
- [ ] Tapping an unselected chip selects it and reveals a quantity stepper (default 1) adjacent to or below the chip
- [ ] Quantity stepper per selected item: minimum 1, maximum 5; minus button disabled at 1
- [ ] Tapping a selected chip deselects it and removes it (and its quantity) from the request
- [ ] A special instructions text field (optional, max 200 chars) is shown at the bottom of the form
- [ ] A "Submit Request" button is shown; it is disabled if no items are selected
- [ ] On submit: a request is created in `vb:requests[]` with `category: "housekeeping"`, `type: "extra_amenities"`, and an `items` array containing each selected item and its quantity
- [ ] A success banner or inline confirmation is shown: "Your amenity request has been submitted ‚Äî items will be delivered to your room shortly" with a reference number (`HK-{timestamp}`)
- [ ] The form resets after successful submission
- [ ] Screen renders correctly in dark mode

## UI / UX Notes

- Screen route: `#/housekeeping/amenities`
- Page header: "Extra Amenities" in 20 px semi-bold; breadcrumb "Housekeeping > Extra Amenities"
- Category sections: collapsible with section header (16 px semi-bold, category icon, item count selected e.g., "Towels (2 selected)")
- All sections expanded by default; `?category=towels` scroll behaviour uses `$nextTick + scrollIntoView`
- Chip multi-select: flex-wrap row; each chip 36 px height, 12 px horizontal padding, 8 px border radius; selected chips use brand primary background + white text
- Quantity stepper adjacent to chip (inline): appears as a small `[-] [1] [+]` control to the right of the chip label when selected; transitions in with `max-width` CSS animation
- Item count display on section header updates live as chips are selected/deselected
- Submit button: brand primary filled, full-width, 48 px; disabled state: grey/50% opacity
- Success banner: full-width green banner slides down from top, auto-dismisses after 4 seconds; reference number inline

## Technical Notes

- Alpine.js component: `x-data="extraAmenities()"`:
  - `selectedItems: {}` ‚Äî object keyed by item ID, value is quantity: `{ 'bath-towel': 2, 'shampoo': 1 }`
  - `isSelected(itemId)` ‚Äî returns `!!selectedItems[itemId]`
  - `toggleItem(itemId)` ‚Äî adds with qty 1 if not selected; removes if selected
  - `setQty(itemId, qty)` ‚Äî updates quantity; if qty < 1, deselects
  - Computed `selectedCount` ‚Äî total items count; `hasSelections` ‚Äî boolean
- On load: check `URLSearchParams` for `category` param; scroll to matching section anchor via `$nextTick`
- Request schema:
  ```json
  {
    "id": "HK-1700000000001",
    "category": "housekeeping",
    "type": "extra_amenities",
    "items": [
      { "itemId": "bath-towel", "name": "Bath Towel", "quantity": 2 },
      { "itemId": "shampoo", "name": "Shampoo", "quantity": 1 }
    ],
    "specialInstructions": "",
    "status": "pending",
    "createdAt": "2026-03-15T09:00:00Z"
  }
  ```
- Category item lists defined as a static data structure in the component (no JSON lookup required for amenity names)

## Test Cases

### Happy Path

1. Guest taps "Extra Amenities" on Housekeeping Hub
2. Screen loads with 4 collapsible sections; all expanded
3. Guest taps "Bath Towel" chip ‚Üí selected (brand primary fill); quantity stepper `[-] [1] [+]` appears
4. Guest taps "+" ‚Üí quantity becomes 2; section header updates to "Towels (1 selected)"
5. Guest taps "Shampoo" in Toiletries ‚Üí selected; qty 1
6. Submit button becomes active; guest taps "Submit Request"
7. Success banner: "Your amenity request has been submitted" + Ref HK-XXXX; form resets

8. Guest taps "Extra Towels" from Housekeeping Hub ‚Üí navigates to `#/housekeeping/amenities?category=towels`; Towels section scrolled into view on load

### Edge Cases

- **No items selected**: Submit button disabled; cannot submit
- **Quantity at max (5)**: "+" button disabled; further taps ignored
- **Deselecting a chip**: Item removed from `selectedItems`; stepper disappears; section header count decrements
- **Baby Cot selected**: Submits correctly with qty; no special validation
- **All chips in all categories selected**: No layout breakage; submit creates request with all items
- **`?category=towels` on load**: Towels section visible and scrolled into view; other sections still accessible
- **Dark mode**: Chip selected/unselected states, section headers, stepper buttons verified
