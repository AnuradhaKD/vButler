## User Story

**As a** hotel guest,
**I want** to submit a laundry order from the app by selecting garments, service type, and preferred pickup time, with a clear price summary before I confirm,
**So that** I can have my laundry handled by the hotel without using paper laundry slips or making a phone call.

---

## Background / Context

Laundry is a value-added service that guests on multi-night stays rely on heavily, particularly business travellers and long-stay leisure guests. The current paper slip model is prone to errors and offers guests no visibility into status or pricing before commitment. A digital laundry order screen with an itemised price list, service type toggle (standard vs express), and real-time order summary gives guests full transparency and confidence, while generating a structured request the housekeeping team can act on efficiently.

## Acceptance Criteria

- [ ] The Laundry Service screen is accessible from the Housekeeping Hub "Laundry" tile at route `#/housekeeping/laundry`
- [ ] A service type toggle is displayed at the top: "Standard (3–4 hrs)" and "Express (1–2 hrs, +50%)"; Standard is selected by default
- [ ] A preferred pickup time field (Flatpickr) is shown below the service toggle
- [ ] The price list is loaded from `property.json → laundryPriceList[]` and grouped by category (e.g., Tops & Shirts, Trousers & Skirts, Dresses & Formal, Bedding & Linen, Others)
- [ ] Each item row displays: item name, wash price (LKR), iron-only price (LKR), and a quantity stepper (+/-)
- [ ] Per item row, the guest can select service type: "Wash" / "Iron Only" / "Wash + Iron" as a radio or segmented control; default is "Wash + Iron"
- [ ] Quantity stepper: minimum 0 (item not in order); maximum 20; when quantity is 0, service type selector is hidden/disabled for that row
- [ ] An order summary section (sticky or positioned at bottom) shows: total item count, subtotal (LKR), express surcharge (LKR, shown only if Express selected — 50% of subtotal), and total (LKR)
- [ ] The "Submit Order" button is disabled if no items are selected (all quantities = 0)
- [ ] On submit: a request is created in `vb:requests[]` with `category: "housekeeping"`, `type: "laundry"`, service type, preferred pickup time, itemised list, and totals
- [ ] A success screen (replacing the form) shows: reference number (`LDY-{timestamp}`), order summary, and estimated return time (Standard: pickup time + 4 hrs; Express: pickup time + 2 hrs)
- [ ] The order status lifecycle is simulated via request `status` field: "Picked Up" → "In Laundry" → "Ready" → "Delivered"; status visible in My Requests (US-05.x)
- [ ] Screen renders correctly in dark mode

## UI / UX Notes

- Service type toggle: large 2-option pill toggle at top of page; "Express" option has a "+50%" surcharge badge in amber; selected option uses brand primary background
- Preferred pickup time: Flatpickr time input, full-width, 48 px; constrained to today and tomorrow, time range 07:00–20:00
- Category group headers: full-width muted strip, 14 px semi-bold; collapsible (chevron right, expands on tap); all expanded by default

- Success screen: full-page replacement; laundry bag illustration SVG; reference in monospace; status timeline component showing 4 stages (Picked Up currently active)

## Technical Notes

- Route: `#/housekeeping/laundry`
- Alpine.js component: `x-data="laundryService()"`:
  - `serviceType: 'standard'` — toggle state
  - `pickupTime: null` — Flatpickr output
  - `orderItems: {}` — keyed by `item.id`: `{ qty, serviceOption }` (serviceOption: `'wash'|'iron'|'wash_iron'`)
  - Computed `selectedItems` — array of `{item, qty, serviceOption, lineTotal}` where `qty > 0`
  - Computed `subtotal` — sum of lineTotals
  - Computed `expressSurcharge` — `serviceType === 'express' ? subtotal * 0.5 : 0`
  - Computed `total` — `subtotal + expressSurcharge`
  - Computed `hasItems` — `selectedItems.length > 0`
- Line total calculation: based on `serviceOption`:
  - `'wash'` → `item.washPrice * qty`
  - `'iron'` → `item.ironPrice * qty`
  - `'wash_iron'` → `(item.washPrice + item.ironPrice) * qty`
- Estimated return time: `new Date(pickupTime).getTime() + (serviceType === 'express' ? 2 : 4) * 3600000`
- Reference: `"LDY-" + Date.now()`
- Request schema:
  ```json
  {
    "id": "LDY-1700000000000",
    "category": "housekeeping",
    "type": "laundry",
    "serviceType": "express",
    "pickupTime": "14:00",
    "items": [
      { "itemId": "shirt", "name": "Shirt / Top", "qty": 3, "serviceOption": "wash_iron", "lineTotal": 2250 }
    ],
    "subtotal": 2250,
    "expressSurcharge": 1125,
    "total": 3375,
    "estimatedReturn": "16:00",
    "status": "pending",
    "createdAt": "2026-03-15T10:00:00Z"
  }
  ```

## Test Cases

### Happy Path

1. Guest taps "Laundry" on Housekeeping Hub → navigates to Laundry screen
2. Standard selected by default; pickup time field shown
3. Guest selects pickup time 14:00
4. Guest adds 3 × Shirt (Wash + Iron at LKR 750 each) and 1 × Trousers (Wash Only at LKR 550)
5. Order summary: "4 items · Subtotal LKR 2,800 · Total LKR 2,800"
6. Guest switches to Express → order summary updates: "+50% surcharge LKR 1,400 · Total LKR 4,200"
7. Guest taps "Submit Order" → success screen: "LDY-1710000000000 · Estimated return by 16:00" (14:00 + 2 hrs)
8. My Requests shows order with status "Pending"

### Edge Cases

- **No items selected**: "Submit Order" disabled; no quantity selected
- **All items at quantity 0**: Summary shows "0 items · LKR 0"; Submit disabled
- **Quantity at 20**: "+" button disabled
- **Iron-only selected for an item with no iron price (0)**: Line total = 0; still submittable
- **Express surcharge = 0 (no items)**: No surcharge row shown in summary
- **Standard selected after Express**: Surcharge line disappears; total recalculates immediately
- **Dark mode**: Service toggle, category headers, item rows, sticky summary, success screen verified
