## User Story

**As a** hotel guest,
**I want** a single screen where I can manage all housekeeping services ‚Äî including Do Not Disturb, room cleaning, and amenity requests ‚Äî without calling the front desk,
**So that** I have full control over my room's servicing on my terms and at any time of day.

---

## Background / Context

Housekeeping is the most operationally critical guest service in a hotel stay. Guests frequently need to communicate cleaning preferences, request extra towels, or activate DND ‚Äî often in situations where they do not want to make a phone call (e.g., early morning, late night, during a meeting). A dedicated Housekeeping Hub in Destinity vButler makes these interactions instant, silent, and always available. As a P0 feature, this screen must be complete before launch.

## Acceptance Criteria

- [ ] The Housekeeping Hub is accessible from the Home Quick Actions grid via a "Housekeeping" tile
- [ ] The Do Not Disturb (DND) toggle is displayed prominently at the very top of the screen, above all other content (see US-09.2)
- [ ] Below the DND toggle, a 2-column service grid is displayed with the following tiles: Room Makeup, Turndown Service, Extra Amenities, Ironing Service, Laundry (links to dedicated sub-screen), Extra Towels
- [ ] Each service tile shows: an icon (SVG), a service name, and a short descriptor (e.g., "Fresh linen & tidying", "Prepared for bedtime", "Towels, toiletries & more")
- [ ] Tapping Room Makeup or Turndown navigates to the scheduling modal (US-09.3)
- [ ] Tapping Extra Amenities navigates to the amenity request screen (US-09.4)
- [ ] Tapping Ironing Service creates a direct express request (no sub-form): modal with preferred time and special instructions
- [ ] Tapping Laundry navigates to the dedicated Laundry screen `#/housekeeping/laundry` (US-09.5)
- [ ] Tapping Extra Towels is an alias for Extra Amenities pre-filtered to the "Towels" category
- [ ] An "Active Requests" section is displayed at the bottom of the screen, showing any in-progress or pending housekeeping requests from `vb:requests[]` filtered by `category: "housekeeping"`
- [ ] If no active housekeeping requests exist, the Active Requests section is hidden
- [ ] Screen renders correctly in dark mode

## UI / UX Notes

- Page background: property surface background (light grey in light mode, dark surface in dark mode)
- DND toggle: large card, full-width ‚Äî see US-09.2 for detailed spec
- Service grid: 2-column, equal-width cards, 12 px gap; each tile 80 px height minimum with centred icon (32 px) above label (14 px semi-bold) above descriptor (12 px muted); rounded corners 12 px; subtle box shadow
- Icon set: use housekeeping-themed SVGs ‚Äî broom/mop for Room Makeup, moon for Turndown, plus-circle for Extra Amenities, iron for Ironing, washing machine for Laundry, towel roll for Extra Towels
- Active Requests section: section header "Active Requests", then request summary cards (similar to My Requests list items); status badges: Pending (amber), In Progress (blue), Completed (green)
- "Ironing Service" tile: tap opens a simple bottom sheet with time picker (Flatpickr) + notes field + "Request Ironing" button; on submit creates request type: "ironing"

---

## Technical Notes

- Route: `#/housekeeping`
- Alpine.js component: `x-data="housekeepingHub()"`:
  - `dndActive` ‚Äî synced from `StorageService` (see US-09.2)
  - `activeRequests` ‚Äî filtered from `StorageService.getRequests({ category: 'housekeeping', status: ['pending', 'in_progress'] })`
  - `ironingDrawerOpen` ‚Äî controls ironing quick-request drawer
- Service tiles defined as a static array in component: `[{ id, label, icon, descriptor, action }]`; action is a function or route string
- Ironing request: same request object pattern as US-09.3 but with `type: "ironing"`
- "Extra Towels" alias: `router.navigate('#/housekeeping/amenities?category=towels')`
- Active Requests polling: `$watch` on localStorage `vb:requests` key to refresh list on changes

## Test Cases

### Happy Path

1. Guest taps "Housekeeping" in Quick Actions
2. Housekeeping Hub loads; DND toggle at top (inactive state); 2-column service grid below with 6 tiles
3. Guest taps "Room Makeup" ‚Üí US-09.3 modal opens
4. Guest taps "Laundry" ‚Üí navigates to `#/housekeeping/laundry`
5. Guest taps "Extra Towels" ‚Üí navigates to `#/housekeeping/amenities?category=towels`
6. Guest taps "Ironing Service" ‚Üí bottom sheet slides up with time picker and notes; guest submits ‚Üí ironing request created; sheet closes
7. Active Requests section appears at bottom with the newly created ironing request showing "Pending" status

### Edge Cases

- **No active housekeeping requests**: Active Requests section hidden; no layout gap
- **DND is active**: Tapping Room Makeup or Turndown shows warning (see US-09.3 spec) before allowing schedule
- **Multiple active requests**: All listed in Active Requests section with status badges
- **Dark mode**: All tiles, descriptors, active request cards verified
