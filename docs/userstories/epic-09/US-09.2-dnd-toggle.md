## User Story

**As a** hotel guest,
**I want** to activate or deactivate Do Not Disturb directly from the app with a single tap,
**So that** I can ensure my privacy and rest without having to hang a physical sign or contact the front desk.

---

## Background / Context

The Do Not Disturb function is arguably the single most important control a hotel guest has over their immediate environment. Physical DND signs are easily forgotten or misplaced; phone-based DND requests add friction. A prominent, persistent digital DND toggle in Destinity vButler — one that is immediately visible, clearly communicates its state, and persists across app sessions — directly improves guest comfort and reduces unwanted interruptions. As a P0 requirement, this must be the first and most prominent element on the Housekeeping Hub.

## Acceptance Criteria

- [ ] The DND toggle is displayed as a full-width card at the top of the Housekeeping Hub (US-09.1) — above all service tiles
- [ ] DND OFF state: card has a neutral/white background (dark mode: surface-card), a toggle switch in the OFF position, and text "Do Not Disturb is off — Your room is available for servicing"
- [ ] DND ON state: card background changes to a red/amber tint (e.g., `#FEF2F2` in light mode), a "DND Active" badge appears (red pill), the toggle switch is in the ON position, and text reads "Do Not Disturb is active — Your room will not be serviced or disturbed"
- [ ] Toggling DND ON during housekeeping service hours (derived from `property.json → housekeeping.serviceHours`) triggers a confirmation prompt: "Housekeeping is currently active (09:00–17:00). Activating DND will prevent room servicing today. Continue?" with "Yes, activate DND" and "Cancel" options
- [ ] Toggling DND ON outside service hours proceeds without a confirmation prompt
- [ ] Toggling DND OFF never requires a confirmation prompt
- [ ] DND state is persisted to localStorage under `vb:dnd` as `{ active: true/false, activatedAt: ISO-timestamp }`
- [ ] DND state is also written to `vb:requests[]` as a request with `category: "housekeeping"`, `type: "dnd"`, `status: "active"` (or "inactive" on deactivation) for operational visibility
- [ ] A DND indicator (small red "DND" badge) is displayed in the Reservation Context Header (top app bar) whenever DND is active
- [ ] DND state is read on app init and on every navigation to the Housekeeping Hub to ensure the UI reflects the persisted state
- [ ] Screen renders correctly in dark mode

## UI / UX Notes

- DND card: full-width, 16 px padding, 12 px border radius; card transitions from neutral to red-tinted background using CSS `transition: background-color 300ms ease`
- Toggle switch: custom CSS toggle (not native checkbox), 52 px wide × 28 px tall; thumb transitions left/right; OFF: grey track, white thumb; ON: red track, white thumb
- "DND Active" badge: red pill (`background: #EF4444`, white text, 12 px bold), positioned top-right of card; appears/disappears with `x-show` and CSS `opacity + scale` transition
- Status text: 14 px Inter, grey in OFF state; bold in ON state with red text
- Icon: large bell-slash icon (SVG, 32 px) on the left side of the card; bell-slash when OFF, crossed bell when ON; icon colour matches badge colour when active
- Confirmation modal (during service hours): standard modal overlay; red-tinted confirm button "Yes, activate DND"; outlined "Cancel" button
- Context header badge: 18 px × 18 px red circle, "DND" label in 9 px white bold, positioned in the top-right area of the header adjacent to the notification bell

## Technical Notes

- Alpine.js state: `x-data="dndToggle()"` with `dndActive` initialised from `StorageService.get('vb:dnd')?.active ?? false`
- `toggleDND()` method:
  1. If activating (`!dndActive`): check if current time is within `housekeeping.serviceHours`; if yes, set `showConfirm = true` and await user action; else proceed directly
  2. Set `dndActive = !dndActive`
  3. Persist to localStorage: `StorageService.set('vb:dnd', { active: dndActive, activatedAt: new Date().toISOString() })`
  4. Append/update request in `vb:requests[]`
- Context header badge: rendered in the main layout component, driven by a global Alpine store `$store.dnd.active` that is synced from `vb:dnd` on init and on storage events
- Service hours check: parse `housekeeping.serviceHours.open` and `housekeeping.serviceHours.close` (HH:MM) against `new Date()` local time
- Storage event listener on `window` ensures cross-tab sync of DND state if guest has app open in multiple tabs

## Test Cases

### Happy Path

1. Guest opens Housekeeping Hub at 14:00 (within service hours 09:00–17:00); DND card shows OFF state
2. Guest taps DND toggle → confirmation modal: "Housekeeping is currently active (09:00–17:00). Continue?"
3. Guest taps "Yes, activate DND" → card transitions to red-tinted, "DND Active" badge appears, toggle switches ON, text changes to active message
4. Red "DND" badge appears in Reservation Context Header
5. Guest closes app and reopens → DND state persisted; card still shows ON state
6. Guest taps toggle again → no confirmation prompt; DND deactivated; card returns to neutral; header badge disappears

### Edge Cases

- **Toggle activated outside service hours (e.g., 22:00)**: No confirmation prompt; DND activates immediately
- **Property has no serviceHours configured**: Treat as "outside service hours" — no confirmation prompt shown
- **Guest taps "Cancel" on confirmation modal**: DND state unchanged; modal dismissed; toggle visually reverts
- **Multiple tabs open**: DND state synced via storage event; both tabs reflect same state
- **DND active when guest navigates to Room Makeup**: Warning displayed in US-09.3 (cross-feature integration)
- **Dark mode**: Red-tinted card background adjusted to dark-mode equivalent (`#2D0000` or similar); badge and text verified
