## User Story

**As a** hotel guest,
**I want** to see all restaurants, bars, and in-room dining options available at the property in one place,
**So that** I can quickly decide where or how to dine without needing to consult the front desk or printed collateral.

---

## Background / Context

Browns Hotels & Resorts properties may operate multiple dining outlets — signature restaurants, casual eateries, poolside bars, and in-room dining — each with distinct cuisines, hours, and booking mechanisms. Guests expect a single, visually rich entry point that surfaces all options at a glance. The Dining Hub serves as the top-level navigation screen for EPIC-07, feeding into restaurant detail, reservation, and in-room dining flows.

---

## Acceptance Criteria

- [ ] Screen is reachable from the Home Quick Actions grid via a "Dining" tile
- [ ] A brand-gradient hero banner is displayed at the top with the property dining intro text and headline rendered in Playfair Display font
- [ ] All dining outlets from `property.json → dining.restaurants[]` are rendered as individual listing cards
- [ ] Each restaurant card displays: thumbnail image (or brand-gradient placeholder if no image), restaurant name, cuisine type pill, today's operating hours, a "Reserve Table" button, and a "View Menu" button
- [ ] Bars (type: "bar") appear in the same list but are visually distinguishable (e.g., a "Bar" pill instead of cuisine type)
- [ ] In-Room Dining is rendered as a prominent featured card, separate from the restaurant list, showing availability text "Available until [time] tonight" derived from `dining.inRoomDining.closeTime`
- [ ] The In-Room Dining card contains an "Order Now" button that navigates to US-07.4
- [ ] If `dining.restaurants[]` is empty or not configured, an empty state is shown: icon + "No dining outlets configured for this property"
- [ ] If In-Room Dining is not configured (`dining.inRoomDining` absent), the featured IRD card is hidden
- [ ] "Reserve Table" buttons navigate to the Table Reservation flow (US-07.3) pre-populated with the selected restaurant
- [ ] "View Menu" buttons navigate to the Restaurant Detail & Menu screen (US-07.2) for the selected restaurant
- [ ] The screen is fully responsive and renders correctly on 375 px, 390 px, and 430 px viewport widths
- [ ] Dark mode renders correctly with appropriate contrast for all text elements

## UI / UX Notes

- Header: brand gradient background (property primary + secondary colour), overlaid with property dining intro text; headline "Taste & Indulge" (or property override text) in Playfair Display 28 px, white
- Restaurant cards: white background (dark mode: surface-card), 12 px border radius, subtle box-shadow, 16 px padding
- Cuisine pill: small rounded pill, brand-accent background, white text, 12 px font
- Operating hours displayed as "Open: 07:00 – 22:30" in muted grey text, 13 px
- "Reserve Table" button: filled brand primary; "View Menu" button: outlined brand primary — both full-width on mobile, side-by-side on tablet+
- IRD featured card: slightly larger, brand gradient left border (4 px), with a room service icon (SVG) in the top-right corner
- "Available until [time] tonight" in italic Playfair Display 14 px
- Closed outlets show a "Closed" badge overlay on the thumbnail and grey out the "Reserve Table" button
- Sticky "Order In-Room" floating button visible when IRD is available and user has scrolled past the IRD card

---

## Technical Notes

- Page file: `pages/dining.html` or rendered via the router as `#/dining`
- Data source: `DataService.getProperty().dining` — loads `property.json` via the existing `DataService` module
- Alpine.js component: `x-data="diningHub()"` with `restaurants`, `inRoomDining`, `isEmpty` computed properties
- Current time compared against `restaurant.hours.open` / `restaurant.hours.close` (HH:MM strings) to derive "open now" state; use `luxon` or native `Date` for time comparison
- Cuisine pill colour optionally driven by `restaurant.cuisineColour` in JSON; fall back to brand accent
- IRD close time formatted with 12-hour clock for display: "Available until 11:00 PM tonight"
- No network calls; all data from `DataService` (localStorage-backed `property.json`)
- Route params: `?restaurantId=X` passed when navigating to US-07.2 and US-07.3 from this screen

## Test Cases

### Happy Path

1. Guest opens Destinity vButler and taps "Dining" in Quick Actions
2. Dining Hub loads; hero banner displays with property headline
3. Three restaurant cards are visible: "The Verandah" (Sri Lankan), "Pool Bar" (bar), "The Pepper Club" (international)
4. Each card shows thumbnail, cuisine pill, today's hours, and both buttons
5. IRD featured card shows "Available until 11:00 PM tonight" with "Order Now" button
6. Guest taps "View Menu" on The Verandah → navigates to US-07.2 for that restaurant
7. Guest taps "Reserve Table" on The Pepper Club → navigates to US-07.3 pre-filled with that restaurant
8. Guest taps "Order Now" on IRD card → navigates to US-07.4

### Edge Cases

- **No restaurants in JSON**: Empty state shown; IRD card still appears if configured
- **IRD not configured**: IRD featured card absent; no errors thrown
- **Restaurant with no thumbnail**: Brand gradient placeholder renders without broken image
- **Current time past restaurant close time**: "Closed" badge applied; "Reserve Table" button is disabled and greyed
- **Restaurant with hours spanning midnight** (e.g., 18:00 – 02:00): Open/closed logic handles next-day close correctly
- **Dark mode active on load**: All cards, pills, and text render with correct dark-mode colour tokens
- **Single restaurant configured**: List renders with one card; no layout breakage
