## User Story

**As a** hotel guest,
**I want** to browse a restaurant's full menu with item descriptions, prices, and dietary information before I visit or reserve,
**So that** I can make an informed dining decision and plan my experience with confidence.

---

## Background / Context

Guests at Browns Hotels & Resorts properties expect a premium digital dining experience consistent with the luxury boutique positioning. Being able to browse menus in-app — especially with clear dietary tagging for halal, vegetarian, and gluten-free requirements — reduces friction and supports guests with dietary restrictions. This screen also serves as the entry point to the table reservation flow, keeping the guest journey seamless within Destinity vButler.

---

## Acceptance Criteria

- [ ] Screen is reachable from the Dining Hub (US-07.1) via "View Menu" button, with the `restaurantId` passed as a route parameter
- [ ] A hero image banner is displayed if `restaurant.heroImage` is present; if absent, a brand-gradient banner with the restaurant name overlaid is used as fallback
- [ ] Restaurant name is displayed in Playfair Display font on or directly below the hero banner
- [ ] Cuisine type, full operating hours (all days if a weekly schedule exists, otherwise today's hours), and a short description are displayed beneath the name
- [ ] Menu is loaded from `property.json → dining.restaurants[n].menu` and grouped into accordion sections by category (e.g., Starters, Mains, Desserts, Beverages)
- [ ] Each accordion section header shows the category name and item count (e.g., "Mains (8)")
- [ ] Default state: first accordion section is expanded; all others collapsed
- [ ] Each menu item displays: name, description (truncated to 2 lines with "read more" if longer than 100 chars), price in LKR formatted with comma separator (e.g., LKR 1,850), and dietary tag pills
- [ ] Dietary tags rendered as coloured pills: V (Vegetarian) — green, VG (Vegan) — dark green, GF (Gluten-Free) — amber, Halal — teal
- [ ] A "Reserve a Table" section is embedded at the bottom of the page (not a separate screen), surfacing the date/time/party-size inputs from US-07.3 inline
- [ ] A sticky "Reserve a Table" floating action button is visible while scrolling through the menu, scrolling the page to the reservation section on tap
- [ ] If `restaurant.menu` is empty or absent, a message "Menu coming soon — please ask our team for today's offerings" is shown in place of accordions
- [ ] Back navigation returns the guest to the Dining Hub (US-07.1)
- [ ] Screen renders correctly in dark mode

## UI / UX Notes

- Hero image: 100% width, 220 px height on mobile, `object-fit: cover`; brand gradient overlay at bottom 40% for text legibility
- Restaurant name: Playfair Display 26 px, white if on hero image; brand primary if gradient fallback with white text overlay
- Cuisine + hours row: flex row with pill for cuisine and clock icon + text for hours, 14 px Inter
- Accordion section headers: 16 px semi-bold Inter, brand primary colour left border (3 px), chevron icon rotates 180° when expanded, tap target minimum 48 px height
- Accordion expand/collapse: CSS `max-height` transition, 200 ms ease
- Menu item row: item name 15 px semi-bold, description 13 px muted, price 15 px brand primary right-aligned, dietary pills below description in a flex-wrap row
- Dietary pill dimensions: 20 px height, 8 px horizontal padding, 4 px border radius, 11 px bold text
- "Reserve a Table" embedded section: card with brand gradient top border, Playfair Display section heading "Reserve Your Table"
- Sticky FAB: brand primary filled circle button, fork-and-knife icon, bottom-right corner, 72 px from bottom to avoid overlap with nav bar

## Technical Notes

- Route: `#/dining/restaurant?id={restaurantId}`
- Alpine.js component: `x-data="restaurantDetail()"` — on init, reads `restaurantId` from URL params and calls `DataService.getRestaurant(id)`
- Accordion state managed via Alpine array: `openSections: [0]` (first section open by default), toggled by index
- Price formatting: utility function `formatLKR(amount)` → "LKR X,XXX" using `toLocaleString('en-LK')`
- Dietary tags sourced from `item.dietary[]` array in JSON (e.g., `["V", "GF"]`)
- "Read more" truncation: CSS `-webkit-line-clamp: 2` with an Alpine toggle for full text
- Inline reservation section is a lightweight version of US-07.3's form; on submit it follows the same request creation logic
- No external API calls; all data from `DataService`

## Test Cases

### Happy Path

1. Guest taps "View Menu" on "The Verandah" restaurant card in the Dining Hub
2. Restaurant Detail screen loads with hero image, name in Playfair Display, cuisine pill "Sri Lankan", and today's hours
3. Menu accordion shows 4 sections: Starters (4), Mains (8), Desserts (5), Beverages (6)
4. Starters section is open by default; guest reads items with prices (LKR 850 – LKR 1,200)
5. "Pol Sambol Bruschetta" shows V (green) and GF (amber) dietary pills
6. Guest taps "Mains" header → section expands, Starters collapse; chevrons animate
7. Guest taps sticky FAB → page scrolls to embedded reservation section
8. Guest completes inline reservation form → reservation created (same behaviour as US-07.3)

### Edge Cases

- **No hero image**: Brand gradient banner renders with restaurant name; no broken image
- **Menu absent**: "Menu coming soon" message shown; reservation section still visible
- **Item description over 100 chars**: Truncated with "read more" link; tapping expands full description
- **No dietary tags on item**: Item renders correctly without pill row
- **Restaurant with weekly schedule**: All days' hours shown in a compact table format
- **Invalid restaurantId in URL**: Error state shown: "Restaurant not found" with Back button
- **Dark mode**: Hero gradient, accordion borders, pill colours all verified
