## User Story

**As a** hotel guest,
**I want** to browse an in-room dining menu, add items to a cart, and place an order for delivery to my room,
**So that** I can enjoy a meal in the comfort of my room without needing to call the kitchen or front desk.

---

## Background / Context

In-room dining is one of the most-used guest services at Browns Hotels & Resorts properties, particularly for guests arriving on late flights, families with young children, or those seeking a private dining experience. A frictionless, app-based ordering flow — with a visible cart, clear delivery time, and instant confirmation — elevates the experience and reduces phone calls to the kitchen. The flow must gate access by IRD operating hours and integrate with the folio charge system.

## Acceptance Criteria

- [ ] Screen is reachable from: (a) Dining Hub "Order Now" button on the IRD featured card, and (b) deep link `#/dining/in-room`
- [ ] A delivery info bar at the top of the screen shows "Delivering to Room [N] · Est. 30–45 min" using `vb:session.roomNumber`
- [ ] If current time is outside `dining.inRoomDining.hours` (open–close), a "Closed" state is shown with text "In-Room Dining is available from [open] to [close]" and the menu is hidden; the "Place Order" action is inaccessible
- [ ] Menu is organised into horizontal scrollable category tabs: Breakfast, All Day, Mains, Desserts, Beverages (or as configured in `dining.inRoomDining.categories[]`)
- [ ] Switching category tabs filters the displayed items without page reload
- [ ] Each menu item card displays: name, short description, price (LKR), dietary tag pills, and a quantity stepper (+/- buttons with current quantity)
- [ ] Quantity stepper: minimum 0 (item not in cart); maximum 10 per item
- [ ] Adding any item (quantity > 0) causes the sticky bottom order summary bar to appear with a brand-primary background
- [ ] Sticky order summary bar shows: total item count, subtotal (LKR), and "View Order" button; it is hidden when cart is empty
- [ ] Tapping "View Order" opens the order review drawer (bottom sheet on mobile, right-side panel on desktop)
- [ ] Order review drawer displays: itemised list (name, quantity, line total), delivery time selector (ASAP / Scheduled with Flatpickr if Scheduled), special instructions textarea (max 300 chars), and "Confirm Order" button
- [ ] "Confirm Order" is disabled if cart is empty
- [ ] On confirm: a request is written to `vb:requests[]` with `category: "dining"`, `type: "in_room_dining"`, all cart items, delivery time, and room number; charges are added to the guest's folio record
- [ ] A success state replaces the drawer content: green checkmark, "Order Received!", reference number (`IRD-{timestamp}`), estimated delivery time
- [ ] An in-app notification banner appears: "In-room dining order received · Ref: IRD-XXXX"
- [ ] After success, cart is cleared and sticky bar is hidden
- [ ] Screen renders correctly in dark mode

## UI / UX Notes

- Delivery info bar: full-width, brand-primary background, white text, 13 px, room icon left-aligned
- Closed state: full-screen overlay over menu area with a clock illustration, muted text, no interaction possible on menu items
- Category tabs: horizontal scroll row, 40 px height pills, selected tab has brand primary background + white text; non-selected is ghost/outlined
- Item card: white background (dark: surface-card), thumbnail left (64 px × 64 px, rounded 8 px), name + description right, price bottom-left of text column, dietary pills below description, quantity stepper bottom-right of card
- Quantity stepper: 32 px minus (red outline) and plus (brand primary filled) buttons, quantity in 16 px bold between them; when quantity = 0 only the "+" button is visible (minus hidden to reduce clutter)
- Sticky order summary bar: 64 px height, brand-primary bg, white text, "View Order" button is white filled with brand-primary text, slides up from bottom with CSS transition
- Order review drawer: `max-height: 90vh`, scrollable item list, delivery time row (ASAP default selected), special instructions below, Confirm button pinned to bottom of drawer
- Itemised list row: name + qty left, line total right, 14 px; subtotal row bold at bottom
- Success state: animated checkmark SVG, Playfair Display "Order Received!" 22 px, reference in monospace, estimated time in brand primary

## Technical Notes

- Route: `#/dining/in-room`
- Alpine.js component: `x-data="inRoomDining()"` with:
  - `items[]` — full menu loaded from `DataService`
  - `cart: {}` — keyed by `item.id`, value is quantity (`{itemId: qty}`)
  - `activeCategory` — currently selected tab
  - Computed `cartItems` — array of `{item, qty, lineTotal}` filtered from `cart`
  - Computed `subtotal` — sum of all `lineTotal` values
  - Computed `cartCount` — total item quantity
  - `drawerOpen: false` — controls order review drawer
  - `successState: false` — controls post-confirm UI
- IRD hours check: on component init and every 60 seconds, compare `Date.now()` against `inRoomDining.openTime` / `inRoomDining.closeTime`
- Folio charge: `FolioService.addCharge({ description: "In-Room Dining", amount: subtotal, ref: orderId })`
- In-app notification: `NotificationService.show({ message: "In-room dining order received · Ref: " + orderId, duration: 5000 })`
- Request schema:
  ```json
  {
    "id": "IRD-1700000000000",
    "category": "dining",
    "type": "in_room_dining",
    "roomNumber": "304",
    "items": [
      { "itemId": "item-001", "name": "Club Sandwich", "qty": 2, "unitPrice": 1850, "lineTotal": 3700 }
    ],
    "subtotal": 3700,
    "deliveryTime": "ASAP",
    "scheduledTime": null,
    "specialInstructions": "Extra ketchup please",
    "status": "received",
    "createdAt": "2026-03-15T14:30:00Z"
  }
  ```
- Cart persisted to `sessionStorage` (not localStorage) — cart clears on session end / new login

## Test Cases

### Happy Path

1. Guest taps "Order Now" on IRD card in Dining Hub at 14:00 (within hours 06:00–23:00)
2. Screen loads: delivery bar "Delivering to Room 304 · Est. 30–45 min"
3. "All Day" category tab active; 8 items visible with prices
4. Guest taps "+" on "Club Sandwich" (LKR 1,850) → quantity becomes 1; sticky bar slides up showing "1 item · LKR 1,850 · View Order"
5. Guest adds "Mango Smoothie" (LKR 650) × 2 → bar updates: "3 items · LKR 3,150"
6. Guest taps "View Order" → drawer opens; itemised list shows both items with line totals
7. Delivery time: ASAP selected; guest adds special instructions "No ice"
8. Guest taps "Confirm Order" → success state; reference IRD-1710000000000; in-app notification appears
9. Cart cleared; sticky bar hidden; guest taps "Back to Dining"

### Edge Cases

- **Current time outside IRD hours**: Closed state shown; menu hidden; no ordering possible
- **Cart item at quantity 10**: Plus button disabled; further taps do nothing
- **Category with no items**: Tab hidden or shows "No items in this category"
- **Empty cart, drawer open via "View Order"**: "Confirm Order" button disabled; drawer shows "Your cart is empty"
- **Scheduled delivery time selected**: Flatpickr appears; time constrained to remaining IRD hours for today
- **Session room number absent**: Delivery bar shows "Delivering to your room" as fallback
- **Dark mode**: All tabs, cards, drawer, and sticky bar verified
