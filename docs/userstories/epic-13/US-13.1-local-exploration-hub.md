## User Story

**As a** guest,
**I want** to explore a curated list of nearby places and attractions,
**So that** I can discover where to go and what to do in the local area without relying on the concierge.

---

## Background / Context

Browns Hotels & Resorts properties are typically situated in areas of natural beauty, cultural richness, and gastronomic interest. The Local Exploration Hub gives guests a self-service discovery tool to find the best local experiences beyond the property grounds. By leveraging property-curated local data, the hotel's expertise is baked into the recommendations — adding digital concierge value without staff overhead.

---

## Acceptance Criteria

- [ ] The Local Exploration Hub is accessible at route `#/local` via Quick Actions → "Explore"
- [ ] A **tab bar** with three tabs is displayed: **Nearby** (default active), **Popular**, and **Saved**
  - "Nearby" tab: places sorted ascending by `place.distanceKm`
  - "Popular" tab: places in JSON data source order (assumed editorial/popularity order)
  - "Saved" tab: only places whose `id` is in `vb:wishlist.placeIds[]`
- [ ] A **category filter chip row** is displayed below the tab bar: All | Food | Nature | Culture | Shopping | Adventure | Nightlife; "All" is active by default
- [ ] Selecting a category chip filters the visible place cards to those matching `place.category`; the "All" chip removes the category filter
- [ ] The category filter state **persists while navigating between tabs** — switching from Nearby to Popular retains the selected category filter
- [ ] Place data is sourced from `property.json → localPlaces[]`
- [ ] Place cards are rendered in a **list view** (single column, full width) within the content area below the tab bar and filter chips
- [ ] The **Saved** tab shows an empty state when `vb:wishlist.placeIds[]` is empty: a bookmark illustration and text "Bookmark places you want to visit"
- [ ] If a category filter is applied on the Saved tab and no saved places match, the standard empty state for no filter results is shown: "No saved [Category] places yet"

## UI / UX Notes

- Page route: `#/local`
- Page header: "Explore Local" with a map/compass icon; brand top bar
- **Tab bar**: sticky below the page header; 3 tabs with brand underline on active; tab labels in 14px medium; "Saved" tab shows a small badge count if `placeIds` count > 0
- **Category filter chips**: horizontally scrollable row below tabs (no visible scrollbar on mobile); pill style; brand colour when active; gap 8px between chips
- **List view**: cards are full-width; each card is a separate component (see US-13.2); 12px gap between cards; subtle top padding below filters
- Empty Saved state: vertically centred in the tab content area; SVG bookmark illustration; 15px muted text; no "Browse All" CTA needed (tabs themselves allow navigation)
- Empty category filter state (no matches): same style as experiences hub — icon + "No [Category] places found" + "Clear Filter" text link
- Smooth tab switching: content area fades or slides depending on brand motion preferences; `x-transition` Alpine.js
- Pull-to-refresh is not required in Phase 1 (data is static JSON)
- Dark mode: tab bar background, chip backgrounds, list background all use dark surface CSS variables

## Technical Notes

- **Alpine.js** `x-data="localHub()"` manages:
  - `places` — all places loaded from `window.propertyData.localPlaces` on `init()`
  - `activeTab` — string: `'nearby'` | `'popular'` | `'saved'`; default `'nearby'`
  - `selectedCategory` — string: default `'all'`
  - `wishlistPlaceIds` — from `Alpine.store('wishlist').placeIds` (or directly from localStorage)
  - `filteredPlaces` — computed getter applying tab logic then category filter:
    ```js
    get filteredPlaces() {
      let list = this.places;
      if (this.activeTab === 'nearby') list = [...list].sort((a,b) => a.distanceKm - b.distanceKm);
      else if (this.activeTab === 'saved') list = list.filter(p => this.wishlistPlaceIds.includes(p.id));
      if (this.selectedCategory !== 'all') list = list.filter(p => p.category === this.selectedCategory);
      return list;
    }
    ```
  - `isEmpty` — getter: `filteredPlaces.length === 0`
- Tab switching updates `activeTab`; category filter and sort are re-applied reactively
- Category filter chips: only categories present in `places` data should render as chips to avoid dead-end filters; derive distinct categories from `[...new Set(places.map(p => p.category))]` and intersect with the ordered list above
- Saved badge count: `vb:wishlist.placeIds.length` — rendered as a small number in the tab label

## Test Cases

### Happy Path
1. Guest navigates to `#/local` from Quick Actions → "Explore"
2. Hub loads with "Nearby" tab active; category "All" selected
3. 8 place cards render sorted by `distanceKm` ascending
4. Guest taps "Popular" tab → cards reorder to JSON data source order; category filter "All" remains active
5. Guest taps "Food" chip → only Food-category places shown in Popular order
6. Guest taps "Nearby" tab → Food filter persists; only nearby Food places shown, sorted by distance
7. Guest taps "Saved" tab → saved places (bookmarked) shown filtered by "Food" (if any)

### Edge Cases
- **Empty Saved tab**: No places bookmarked → empty state with bookmark illustration shown
- **Category with no places**: Guest taps a category with 0 matches → "No [Category] places found" + "Clear Filter" link
- **All places in one category**: Hub works correctly; other category chips produce empty filter state
- **Saved + Category filter = no results**: "No saved [Category] places yet" message shown
- **Large list (15+ places)**: List scrolls smoothly; no layout issues on mobile
- **`localPlaces` empty in JSON**: Hub shows "No local places available" message; no JS errors
