## User Story

**As a** guest,
**I want** to see a rich summary card for each local place including a hotel recommendation note and directions link,
**So that** I can quickly understand what a place offers and navigate to it easily.

---

## Background / Context

The place card is the primary content unit of the Local Exploration feature. It must convey enough information for a guest to make a visit decision at a glance — distance, travel time, and the hotel's personal recommendation add curated value that differentiates the Browns experience from a generic maps search. The inline expand behaviour keeps the list navigable without requiring a separate detail page in Phase 1.

---

## Acceptance Criteria

- [ ] Each place card displays: a place photo at the top (brand gradient fallback if no photo or load error), the place name, a category badge, distance from the hotel (formatted as "X.X km from hotel"), and travel time (e.g. "~15 min by car")
- [ ] A **hotel recommendation note** sourced from `place.hotelNote` is displayed as a highlighted quote block (visually distinct — e.g. left border in brand accent colour, italic text, subtle background); if `place.hotelNote` is null or empty, the quote block is not rendered
- [ ] A **bookmark icon** (Material Icons `bookmark_add` when not saved, `bookmark` when saved) is displayed in the top-right of the card image area; tapping it toggles the place's save state in `vb:wishlist.placeIds[]` and updates the icon immediately
- [ ] A **"Get Directions"** button is displayed on the card; tapping it opens `place.mapUrl` in a new browser tab (`target="_blank"`, `rel="noopener noreferrer"`)
- [ ] Tapping anywhere on the card **except** the bookmark icon and "Get Directions" button **expands the card inline** to reveal the full description (`place.description`) and a larger version of the photo (or a second image if available); a second tap collapses back to the compact view
- [ ] Only one card may be expanded at a time — expanding a new card collapses any previously expanded card
- [ ] The expanded state is smooth: CSS max-height transition or Alpine.js `x-collapse` for the expand/collapse animation

## UI / UX Notes

- Card structure (compact view, top to bottom):
  - **Image block**: aspect ratio 16:9, `object-fit: cover`, full card width; position relative for badge and bookmark icon overlay
  - **Category badge**: bottom-left of image, same pill style as experiences
  - **Bookmark icon**: top-right of image, 24px; `bookmark_add` (outline) / `bookmark` (filled brand colour); 44×44px touch target
  - **Card body**: 12px padding; white (light) / dark surface (dark mode)
  - **Name**: 16px font-weight 600; single line truncation with ellipsis
  - **Distance + travel time row**: 13px muted text; distance in brand colour; travel time in default muted; separated by a "·" dot
  - **Hotel note quote block**: left border 3px brand accent colour; background `rgba(var(--brand-rgb), 0.06)`; italic 13px text; 8px vertical padding; 12px left padding; margin-top 8px
  - **Action row**: "Get Directions" text button left-aligned; subtle right chevron to indicate expandability (top-right of body or bottom-right)
- **Expanded state** (additional content below action row):
  - Expanded image: same image in taller aspect ratio (4:3) or natural height, `object-fit: cover`
  - Full description text: 14px normal weight, line-height 1.6
  - A "Close" or up-chevron icon to collapse
- Expand chevron: rotates 180° when expanded (CSS `transform: rotate(180deg)` transition)
- Bookmark animation on toggle: same scale pulse as wishlist heart (200ms)
- "Get Directions" button: text link style, brand colour, external link icon suffix
- Card border-radius: 12px; box-shadow: `0 2px 8px rgba(0,0,0,0.10)`
- Mobile: full-width cards (edge-to-edge within content padding); Tablet+: max-width 640px or as part of a wider layout

## Technical Notes

- **Alpine.js** component (`x-data="placeCard(place)"` or managed by parent `x-for`):
  - `expanded` — boolean, default `false`
  - `toggleExpand()` — if global `openCardId !== this.place.id`, close previous (`openCardId = null`), set `openCardId = this.place.id`, set `expanded = true`; else set `expanded = false`; uses `Alpine.store('localHub').openCardId` pattern
  - `isSaved` — computed: `Alpine.store('wishlist').placeIds.includes(place.id)`
  - `toggleBookmark()` — calls StorageService wishlist toggle for `placeIds`; updates `Alpine.store('wishlist').placeIds`
- "Only one card expanded" implementation: store `openCardId` in `Alpine.store('localHub')`; each card watches `$store.localHub.openCardId` to determine its own expanded state
- **Get Directions**: `<a :href="place.mapUrl" target="_blank" rel="noopener noreferrer">Get Directions</a>` — use `@click.stop` to prevent card expand trigger
- Bookmark tap: `@click.stop` to prevent card expand trigger
- Distance formatting: `place.distanceKm.toFixed(1) + ' km from hotel'`
- Travel time: display `place.travelTime` directly from JSON (stored as human-readable string, e.g. "~15 min by car")
- Image fallback: same `onerror` → brand gradient approach as experience cards

## Test Cases

### Happy Path
1. Local Exploration Hub loads 8 place cards in list view
2. First card (nearest): photo, name, "1.2 km from hotel · ~5 min by car", hotel note quote block, "Get Directions" button
3. Guest taps "Get Directions" → `place.mapUrl` opens in new tab; card does not expand
4. Guest taps the card body → card expands inline with full description and larger image; expand animation plays
5. Guest taps a second card → first card collapses; second card expands
6. Guest taps second card again → second card collapses; all cards in compact view
7. Guest taps bookmark icon on a card → icon changes to filled `bookmark`; toast "Place Saved"; `vb:wishlist.placeIds` updated
8. Guest taps bookmark again → icon reverts to `bookmark_add`; toast "Place Removed"

### Edge Cases
- **No hotel note**: `hotelNote` is null → quote block not rendered; card layout adjusts cleanly
- **Image load failure**: Brand gradient fallback renders; all other card elements unaffected
- **`mapUrl` is null or empty**: "Get Directions" button is hidden or disabled (not shown if no map URL)
- **Very long place name**: Truncated to single line with ellipsis; full name visible in expanded state
- **Rapid tap on expand**: Double-tap does not leave card in broken state; toggle logic handles both expand and collapse correctly
- **Bookmark while card is expanded**: Expanded state maintained; only bookmark icon updates
- **Dark mode**: Card surface, quote block background, distance/time text, bookmark icon all render correctly
