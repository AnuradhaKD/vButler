## User Story

**As a** guest,
**I want** to bookmark local places I want to visit and view them in a dedicated saved list,
**So that** I can quickly return to places I have shortlisted without scrolling through the full catalogue.

---

## Background / Context

Guests often browse local exploration options across multiple sessions — discovering a restaurant at breakfast, a nature spot at lunch, and a cultural venue in the evening. The bookmarking feature allows guests to build a personal itinerary of places without committing to a formal plan, in keeping with the relaxed luxury style of Browns Hotels & Resorts.

---

## Acceptance Criteria

- [ ] A **bookmark icon** (Material Icons `bookmark_add` when not saved, `bookmark` when saved) is displayed in the top-right of every place card's image area
- [ ] Tapping the bookmark icon when the place is **not** saved: fills the icon (brand colour), adds `place.id` to `vb:wishlist.placeIds[]` in localStorage, and shows a brief toast "Place Saved"
- [ ] Tapping the bookmark icon when the place **is** saved: reverts the icon to `bookmark_add` (outline), removes `place.id` from `vb:wishlist.placeIds[]`, and shows a brief toast "Removed from Saved"
- [ ] Saved bookmark state persists across page navigation and full session reloads (localStorage-backed)
- [ ] The **"Saved" tab** in the Local Exploration Hub (US-13.1) shows only places whose `id` is in `vb:wishlist.placeIds[]`
- [ ] When a guest removes a bookmark while viewing the **Saved tab**, the corresponding card disappears from the list **immediately** (Alpine.js reactive update — no page refresh required)
- [ ] When the Saved tab is active and all bookmarks have been removed (or none exist), the empty state is shown: a bookmark illustration and text "Bookmark places you want to visit"
- [ ] The `vb:wishlist` localStorage key is shared with EPIC-12 experiences; toggling `placeIds` must not modify `experienceIds`

## UI / UX Notes

- Bookmark icon specs: Material Icons, 24px; positioned top-right of card image, same 44×44px touch target as all icon buttons
- Saved state: filled `bookmark` icon in brand colour (e.g. `var(--brand-primary)`)
- Unsaved state: outline `bookmark_add` icon in white with drop-shadow for visibility over photos; muted grey in dark mode
- Toggle animation: same 200ms scale pulse keyframe as experience wishlist heart
- Toast: bottom-of-screen toast (or top-of-screen if bottom navigation bar is present); dark background, white text, `border-radius: 8px`; auto-dismisses after 2 seconds
- Saved tab badge: small circular number badge on the "Saved" tab label showing the count of `vb:wishlist.placeIds.length`; hidden when count is 0
- Empty Saved state: vertically centred in tab content area; SVG or icon bookmark illustration in muted brand colour; 15px text "Bookmark places you want to visit"; no CTA button required
- Removal animation on Saved tab: card slides out or fades out with `x-transition:leave` before being removed from DOM; smooth, not jarring
- Dark mode: bookmark icon colours adapt; toast background ensures sufficient contrast

## Technical Notes

- **StorageService** wishlist toggle for places (shared utility, mirrors experience toggle):
  ```js
  togglePlace(id) {
    const w = JSON.parse(localStorage.getItem('vb:wishlist') || '{"experienceIds":[],"placeIds":[]}');
    const idx = w.placeIds.indexOf(id);
    if (idx === -1) w.placeIds.push(id);
    else w.placeIds.splice(idx, 1);
    localStorage.setItem('vb:wishlist', JSON.stringify(w));
    return w.placeIds; // return updated array for reactive assignment
  }
  ```
- **Alpine.js global store** (`Alpine.store('wishlist')`) must include `placeIds` array alongside `experienceIds`:
  - On app init: `Alpine.store('wishlist').placeIds = StorageService.getWishlist().placeIds`
  - Bookmark toggle: calls `StorageService.togglePlace(id)` → updates `Alpine.store('wishlist').placeIds`
  - Place card `isSaved` binding: `$store.wishlist.placeIds.includes(place.id)`
- Reactive removal on Saved tab: the `filteredPlaces` computed getter in `localHub()` uses `$store.wishlist.placeIds` reactively; when an ID is removed from the store, Alpine.js re-evaluates the getter, removing the card from the DOM
- `x-transition:leave` on the place card: `leave-active-class="transition-opacity duration-200"` + `leave-to-class="opacity-0"` — TailwindCSS equivalent or custom CSS transition class
- Toast implementation: same shared toast component used across EPIC-12 and EPIC-13; triggered via `Alpine.store('toast').show('Place Saved')`

## Test Cases

### Happy Path
1. Fresh session: all place cards show `bookmark_add` (outline) icons
2. Guest taps bookmark on "Yala National Park" card → icon fills brand colour; toast "Place Saved"; `vb:wishlist.placeIds` contains place id
3. Guest taps "Saved" tab → "Yala National Park" card appears; "Saved" tab badge shows "1"
4. Guest taps bookmark on "Galle Fort" card (also from Nearby tab) → saved; badge shows "2"
5. Guest navigates away (e.g. to Dashboard) and returns to `#/local` → both cards still bookmarked (localStorage persistence)
6. Guest on Saved tab taps "Yala National Park" bookmark → card fades out and disappears immediately; badge shows "1"
7. Guest removes last saved place → empty state rendered

### Edge Cases
- **`vb:wishlist` key absent**: Default object `{ experienceIds: [], placeIds: [] }` used; no error thrown
- **`experienceIds` preservation**: After toggling a place, verify `vb:wishlist.experienceIds` is unchanged (run alongside an experience wishlist save test)
- **Bookmark while on Saved tab**: Tapping `bookmark` on Saved tab → card removed immediately; no need to navigate away and back
- **Offline / storage full**: Catch `QuotaExceededError`; show toast "Unable to save — storage limit reached"; icon does not change to filled state
- **Toast overlap**: Save multiple places rapidly → toasts queue or the last replaces the previous; no visual stacking above the screen
- **Session reload mid-save**: localStorage is synchronous — state is always consistent immediately after `setItem`
- **Saved tab with category filter active**: Only saved places matching the category filter are shown (see US-13.3 integration)
