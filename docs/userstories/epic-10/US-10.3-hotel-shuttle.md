## User Story

**As a** hotel guest,
**I want** to view the hotel shuttle schedule and book a seat for a specific date, time, and destination,
**So that** I can plan excursions or trips to town without worrying about arranging my own transport.

---

## Background / Context

Several Browns Hotels & Resorts properties operate complimentary or low-cost shuttle services to nearby towns, beaches, or attractions on a fixed schedule. These are popular with guests who prefer not to hire a private vehicle for short trips. Surfacing the shuttle timetable clearly in Destinity vButler and allowing guests to reserve seats reduces walk-up demand and allows the hotel to manage vehicle capacity more effectively. The feature is conditionally rendered — if no shuttle is configured for a property, the card is hidden entirely.

## Acceptance Criteria

- [ ] The Hotel Shuttle screen is reachable from the Transport Hub "Hotel Shuttle" card (inline form or navigation to `#/transport/shuttle`)
- [ ] If no shuttle is configured for the property (`transport.options[]` contains no entry with `type: "shuttle"`), the Hotel Shuttle card on the Transport Hub is hidden and this screen is inaccessible
- [ ] A shuttle schedule is displayed as a timetable showing all available shuttle runs, each with: departure time, destination, and available seats
- [ ] The timetable is sourced from `property.json → transport.options[n].schedule[]` for the shuttle option
- [ ] A "Complimentary" badge (green pill) is shown on all shuttle entries where `option.price === 0`; otherwise the price in LKR is shown
- [ ] A date selector (Flatpickr) allows the guest to choose a travel date; the timetable updates to show runs for the selected date
- [ ] Shuttle runs are filtered by `schedule.availableDays` to show only runs operating on the selected day
- [ ] A "Book Seat" button is shown for each available run; if the run is full (`run.bookedSeats >= run.totalSeats`), the button is replaced with a "Full" badge
- [ ] Tapping "Book Seat" opens a booking confirmation bottom sheet with: selected run details (time, destination), a passenger count stepper (1 to available seats), and a "Confirm Booking" button
- [ ] On confirm: a request is written to `vb:requests[]` with `category: "transport"`, `type: "shuttle_booking"`, selected run details, date, and passenger count
- [ ] Success state in the bottom sheet: green checkmark, "Seat Reserved", reference number (`SHT-{timestamp}`), and estimated departure time
- [ ] If the shuttle is complimentary, `FolioService.addCharge` is not called; if priced, the charge is added
- [ ] Screen renders correctly in dark mode

## UI / UX Notes

- Timetable layout: vertical list of run cards; each card 72 px height; departure time bold left (18 px); destination centre (14 px muted); seat count right (e.g., "6 seats left" in green, "2 seats left" in amber, "Full" in grey); "Book Seat" button or "Full" badge far right
- Complimentary badge: small green pill "Complimentary" below the departure time; replaces price display
- Date selector: Flatpickr at top of page, date limited to today + 14 days
- "Full" badge: grey rounded pill "Full", 12 px; replaces "Book Seat" button; row is slightly muted
- Booking bottom sheet: run summary at top (departure time, destination, date); passenger count stepper; "Confirm Booking" button brand primary full-width; close button top-right
- Available seats counter: colour-coded — green if ≥ 50% available, amber if < 50% but > 0, red "Full" if 0

## Technical Notes

- Route: `#/transport/shuttle`
- Alpine.js component: `x-data="hotelShuttle()"`:
  - `shuttleOption` — loaded from `DataService` filtering `transport.options` for `type === "shuttle"`
  - `selectedDate` — Flatpickr value, default today
  - `schedule` — computed: filter `shuttleOption.schedule[]` by `run.availableDays.includes(dayName(selectedDate))`
  - `activeRunId: null` — ID of run for which booking sheet is open
  - `passengerCount: 1`
- `dayName(date)` utility: returns day name string (e.g., "Monday") from a Date object
- Seat availability simulation: `run.bookedSeats` (set in JSON for demo) compared against `run.totalSeats`
- Reference: `"SHT-" + Date.now()`
- Folio charge: skipped if `shuttleOption.price === 0`

## Test Cases

### Happy Path

1. Guest taps "Hotel Shuttle" on Transport Hub → timetable screen loads with today's runs
2. 3 runs shown: 09:00 → Galle (8 seats, Complimentary), 12:00 → Colombo Fort (4 seats, LKR 500), 16:00 → Mirissa Beach (6 seats, Complimentary)
3. Guest taps "Book Seat" on 09:00 Galle run → booking sheet opens: "09:00 → Galle · Today · 1 passenger"
4. Guest taps "Confirm Booking" → success: "Seat Reserved · SHT-XXXX · Departure 09:00"
5. No folio charge (complimentary)

6. Guest changes date to Monday → timetable updates to show Monday-only runs

### Edge Cases

- **No shuttle configured for property**: Shuttle card hidden on Transport Hub; screen inaccessible
- **Selected date has no runs** (e.g., Sunday if no Sunday runs): "No shuttle runs on this day" message shown
- **Run is full**: "Full" badge; no Book Seat button; passenger tap does nothing
- **All runs on selected date are full**: All show "Full"; no bookable option
- **Priced shuttle**: Folio charge created; price shown on timetable entry
- **Passenger count stepper at max (available seats)**: "+" disabled
- **Dark mode**: Timetable cards, complimentary badge, booking sheet verified
