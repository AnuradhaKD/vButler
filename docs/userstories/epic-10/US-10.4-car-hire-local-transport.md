## User Story

**As a** hotel guest,
**I want** to arrange a hire car or book local transport (such as a tuk-tuk or taxi) through the hotel, with clear pricing displayed before I confirm,
**So that** I can explore the local area conveniently without having to negotiate with drivers or use unfamiliar apps.

---

## Background / Context

Sri Lanka's rich landscape of cultural sites, beaches, and towns makes guest mobility a key part of the Browns Hotels & Resorts stay experience. Guests frequently ask the concierge to arrange day trips, driver services, or quick local transport. By providing self-service booking forms for both car hire (full-day, chauffeur-driven or self-drive) and on-demand local transport (destination-based, fixed pricing), Destinity vButler empowers guests and reduces concierge call volume for routine transport requests.

## Acceptance Criteria

- [ ] The Car Hire & Local Transport screen is reachable from the Transport Hub at route `#/transport/car-hire-local`; or the Transport Hub may link to `#/transport/car-hire` and `#/transport/local` as separate sub-screens if preferred
- [ ] The screen contains two distinct booking forms, separated by a tab or section toggle: "Car Hire" and "Local Transport"
- [ ] **Car Hire form fields**:
  - Hire Date (Flatpickr, from today to check-out)
  - Duration: segmented control with options Half Day, Full Day, Multi-Day (if Multi-Day, a date range picker appears for end date)
  - Driver Preference: radio pills — Self-Drive, Chauffeur
  - Vehicle Preference: text input or dropdown sourced from `transport.carHire.vehicleTypes[]`
  - Pickup Time (Flatpickr enableTime)
  - Special Requests: textarea (optional, max 200 chars)
- [ ] **Local Transport form fields**:
  - Destination: free text input with placeholder "e.g. Galle Fort, Unawatuna Beach, Colombo"
  - Travel Date (Flatpickr, from today to check-out + 1 day)
  - Travel Time (Flatpickr enableTime)
  - Number of Passengers: stepper (1–8)
  - Vehicle Type: radio pills or card selection sourced from `transport.localTransport.vehicles[]`; each vehicle shows label (e.g., "Tuk-Tuk", "Car", "Van") and price per trip (LKR)
- [ ] For Local Transport, the price per trip for the selected vehicle type is displayed prominently below the vehicle selection before submission ("This trip is LKR X,XXX — fixed price, no negotiation")
- [ ] "Submit Request" (Car Hire) and "Book Now" (Local Transport) buttons are disabled until all required fields are completed
- [ ] On Car Hire submit: request written to `vb:requests[]` with `category: "transport"`, `type: "car_hire"`, all form values; a note is shown: "Our concierge will confirm vehicle availability within 2 hours"
- [ ] On Local Transport submit: request written to `vb:requests[]` with `category: "transport"`, `type: "local_transport"`, vehicle type, price, and all fields; folio charge added via `FolioService.addCharge`
- [ ] Success states for both forms display a reference number and confirmation note
- [ ] Car Hire does not generate a folio charge at booking (pending concierge confirmation); Local Transport does
- [ ] Screen renders correctly in dark mode

## UI / UX Notes

- Tab toggle at top: "Car Hire" and "Local Transport" as pill tabs; selected tab brand primary; switching reveals the relevant form
- Car Hire form: card-based sections; duration segmented control (3 options, each 33% width); Multi-Day end-date picker appears with CSS animation when "Multi-Day" selected; Self-Drive / Chauffeur pills are prominent as these determine vehicle options
- Local Transport vehicle cards: 2-column grid; each card shows vehicle icon (SVG), label, "LKR X,XXX per trip" price in brand primary; selected card: brand primary border + checkmark
- Price display: full-width amber/brand tinted card below vehicle selection: "Fixed fare: LKR X,XXX — no negotiation, no hidden charges"; updates reactively when vehicle selection changes
- Car Hire success note: amber info card "Our concierge will confirm vehicle availability within 2 hours by phone or WhatsApp"; reference number
- Local Transport success: immediate confirmation (price is fixed); standard success card with reference `LCL-{timestamp}`

## Technical Notes

- Route: `#/transport/car-hire-local` with tab state managed by Alpine (`activeTab: 'car_hire'`)
- Alternatively implemented as separate routes `#/transport/car-hire` and `#/transport/local` sharing the same page component
- Alpine.js component: `x-data="carHireLocal()"`:
  - `carHire: { date, duration, driverPreference, vehiclePreference, pickupTime, specialRequests }`
  - `localTransport: { destination, travelDate, travelTime, passengers, selectedVehicleId }`
  - `vehicleTypes` for Car Hire from `DataService`
  - `localVehicles` for Local Transport from `DataService`
  - `selectedLocalVehicle` — computed lookup
  - `localPrice` — computed `selectedLocalVehicle?.pricePerTrip ?? null`
- Car Hire Multi-Day: `duration === 'multi_day'` triggers `x-show` on end-date Flatpickr input
- Local Transport folio charge: `FolioService.addCharge({ description: "Local Transport - " + destination, amount: localPrice, ref: requestId })`
- Reference numbers: Car Hire — `"CHR-" + Date.now()`; Local Transport — `"LCL-" + Date.now()`
- Car Hire request schema:
  ```json
  {
    "id": "CHR-1700000000000",
    "category": "transport",
    "type": "car_hire",
    "hireDate": "2026-03-17",
    "duration": "full_day",
    "driverPreference": "chauffeur",
    "vehiclePreference": "SUV",
    "pickupTime": "08:00",
    "specialRequests": "Child seat required",
    "status": "pending_confirmation",
    "createdAt": "2026-03-15T10:00:00Z"
  }
  ```
- Local Transport request schema:
  ```json
  {
    "id": "LCL-1700000000000",
    "category": "transport",
    "type": "local_transport",
    "destination": "Galle Fort",
    "travelDate": "2026-03-17",
    "travelTime": "10:00",
    "passengers": 2,
    "vehicleId": "vehicle-car",
    "vehicleLabel": "Car",
    "price": 3500,
    "status": "confirmed",
    "createdAt": "2026-03-15T10:00:00Z"
  }
  ```

## Test Cases

### Happy Path — Car Hire

1. Guest taps "Car Hire" tab → Car Hire form shown
2. Guest selects: 17 March, Full Day, Chauffeur, SUV, Pickup 08:00, "Child seat required"
3. Taps "Submit Request" → success: "CHR-XXXX · Our concierge will confirm within 2 hours"
4. No folio charge at this stage

### Happy Path — Local Transport

1. Guest taps "Local Transport" tab → Local Transport form shown
2. Guest enters: Destination "Galle Fort", 17 March, 10:00, 2 passengers, selects "Car" (LKR 3,500)
3. Fixed fare card appears: "Fixed fare: LKR 3,500 — no negotiation"
4. Taps "Book Now" → success: "LCL-XXXX · Galle Fort · 17 Mar 2026 · 10:00 · Car · LKR 3,500"
5. Folio charge LKR 3,500 added

### Edge Cases

- **Multi-Day selected in Car Hire**: End-date picker appears; end date must be after start date; Flatpickr `minDate` set dynamically
- **Self-Drive selected**: Form submits correctly with `driverPreference: "self_drive"`
- **No local vehicles configured**: Vehicle selection section shows "Contact concierge for local transport options"
- **Destination blank**: "Book Now" disabled; "Destination is required" error shown
- **Passenger count at 8**: "+" disabled
- **Vehicle price = 0**: Fixed fare card shows "Complimentary"; no folio charge
- **Dark mode**: Tab toggle, vehicle cards, fixed fare card, success notes verified
