## User Story

**As a** guest who has checked out,
**I want** to share my overall feedback on my stay through a structured survey,
**So that** Browns Hotels & Resorts can understand what they did well and where they can improve for future guests.

---

## Background / Context

Post-stay feedback is the primary data source for Browns Hotels & Resorts' continuous service improvement programme. Unlike third-party review platforms, in-app feedback is collected immediately after check-out while the experience is fresh, and the structured format provides actionable, categorised data. Making the feedback process frictionless and emotionally positive — thanking the guest warmly upon completion — encourages participation and reinforces the premium brand relationship.

---

## Acceptance Criteria

- [ ] The post-stay feedback form is **triggered** when a reservation's `status` changes to `"checked-out"`:
  - An in-app notification is created: "How was your stay? We'd love to hear from you"
  - A **dashboard CTA card** appears: "Share Your Feedback — [Property Name]" with a star icon and "Leave Feedback" button
- [ ] The feedback form is accessible from:
  1. The in-app notification (tapping navigates to the form)
  2. The Dashboard CTA card's "Leave Feedback" button
  3. The Complaints section — a "Leave Feedback" tab or link when reservation status is "checked-out"
- [ ] The form is available for **up to 30 days post-checkout** (`dayjs().diff(dayjs(reservation.checkOutDate), 'day') <= 30`); after 30 days the form is inaccessible and a message is shown: "Feedback window has closed. Thank you for staying with us."
- [ ] **Star Ratings** (5-star, tap to rate, 1–5) are provided for each of the following dimensions: Overall Stay | Room & Cleanliness | Service | Dining | Facilities; each dimension is labelled and has a row of 5 star icons; tapping a star selects that rating and all stars to its left (filled); at least "Overall Stay" rating is required before submission
- [ ] **Highlight chips** (multi-select) are displayed: Great View | Friendly Staff | Delicious Food | Clean Room | Good Location | Great Value | Quiet & Peaceful | Excellent Amenities; selecting a chip highlights it (brand background); any number of chips may be selected (including zero)
- [ ] An **Open Comments textarea** is provided: placeholder "What could we improve, or what did you love most?"; max 1000 characters; character counter displayed; optional (not required for submission)
- [ ] A full-width **"Submit Feedback" primary button** (brand colour) is displayed; disabled until "Overall Stay" star rating is selected
- [ ] On submission:
  - Feedback is stored in `vb:complaints[]` as `{ id, reservationId, type: "post_stay_feedback", ratings: { overall, room, service, dining, facilities }, highlights: [...], comments, submittedAt }`
  - The reservation record is updated: `reservation.feedbackSubmitted = true`
  - A **thank-you state** is shown on the same page
- [ ] If feedback has already been submitted for a reservation (`reservation.feedbackSubmitted === true`), the form is replaced with: "Feedback Submitted ✓" heading, the date of submission, and a thank-you message — no re-submission is possible
- [ ] The feedback form can only be accessed if `reservation.status === "checked-out"`; for active or upcoming reservations, the route redirects or shows an appropriate message

## Technical Notes

- **Checkout trigger** (status change detection): app checks `vb:reservations` on load; if any reservation transitions to `checked-out` and `feedbackSubmitted !== true`, creates notification and sets a flag to show the dashboard CTA
- **30-day window check**:
  ```js
  const daysSinceCheckout = dayjs().diff(dayjs(reservation.checkOutDate), 'day');
  this.feedbackAvailable = daysSinceCheckout <= 30 && !reservation.feedbackSubmitted;
  ```
- **Star rating interaction** (Alpine.js):
  ```js
  setRating(dimension, value) { this.ratings[dimension] = value; }
  isStarFilled(dimension, starIndex) { return this.ratings[dimension] >= starIndex; }
  // Hover preview:
  hoverRating(dimension, value) { this.hoverStars[dimension] = value; }
  clearHover(dimension) { this.hoverStars[dimension] = 0; }
  ```
- **Highlight chips**: `x-on:click="toggleHighlight(chip)"` → push or splice from `selectedHighlights[]`
- **Feedback storage**:
  ```js
  const feedback = {
    id: generateId(),
    reservationId: reservation.id,
    type: 'post_stay_feedback',
    ratings: this.ratings, // { overall: 5, room: 4, service: 5, dining: 3, facilities: 4 }
    highlights: this.selectedHighlights,
    comments: this.comments,
    submittedAt: dayjs().toISOString()
  };
  const complaints = JSON.parse(localStorage.getItem('vb:complaints') || '[]');
  complaints.push(feedback);
  localStorage.setItem('vb:complaints', JSON.stringify(complaints));
  ```
- **Mark reservation as submitted**:
  ```js
  const reservations = JSON.parse(localStorage.getItem('vb:reservations') || '[]');
  const r = reservations.find(r => r.id === reservation.id);
  if (r) r.feedbackSubmitted = true;
  localStorage.setItem('vb:reservations', JSON.stringify(reservations));
  ```
- **Alpine.js** `x-data="feedbackForm(resvId)"`: `reservation`, `ratings` (object, all default 0), `hoverStars` (object), `selectedHighlights[]`, `comments`, `submitted`, `feedbackAvailable`, `alreadySubmitted`
- Dashboard CTA card: rendered in dashboard Alpine.js component with `x-show="checkedOutReservation && !checkedOutReservation.feedbackSubmitted"`
- Notification: `{ type: 'post_stay_feedback_prompt', message: 'How was your stay? We'd love to hear from you', reservationId, createdAt }`

## Test Cases

### Happy Path
1. Reservation status changes to "checked-out" (simulated by updating localStorage)
2. Dashboard shows CTA card: "Browns Beach Hotel — How was your stay?" + "Leave Feedback" button; notification created
3. Guest taps "Leave Feedback" → navigates to feedback form
4. Introduction text references the property name
5. Guest rates Overall Stay: 5 stars (filled gold); Room & Cleanliness: 4; Service: 5; Dining: 3; Facilities: 4
6. Guest selects highlight chips: "Friendly Staff", "Clean Room", "Great View"
7. Guest types comments: "Absolutely loved the view from room 205. The staff were incredibly attentive."
8. "Submit Feedback" enabled; guest taps → thank-you state appears
9. `vb:complaints[]` contains new `post_stay_feedback` entry; `reservation.feedbackSubmitted = true`
10. Dashboard CTA card disappears; revisiting form shows "Feedback Submitted ✓" state

### Edge Cases
- **No "Overall Stay" rating**: Submit button disabled; all other ratings and comments may be filled
- **Zero highlight chips selected**: Allowed; form submits with empty `highlights[]`
- **Empty comments**: Allowed; optional field; submits with empty string
- **Post-checkout window expired (31+ days)**: Form shows "Feedback window has closed" message; no form inputs rendered
- **Active reservation status**: Route redirects or shows "Available after checkout" message; form not accessible
- **Already-submitted**: `feedbackSubmitted: true` on reservation → immediately shows "Feedback Submitted ✓" without rendering form inputs
- **Rapid star tapping**: Tapping star 3 then star 5 → final rating = 5; no state confusion
- **Desktop hover preview**: Hovering over star 4 highlights stars 1–4; moving away restores to actual rating; clicking locks the rating
- **Multiple checked-out reservations**: Feedback CTA and form correctly associated with the most recent checked-out reservation (or allows selection if multiple)
