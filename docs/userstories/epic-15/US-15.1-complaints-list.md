## User Story

**As a** guest,
**I want** to see all the issues I have reported in one place with their current status,
**So that** I can track whether my concerns are being addressed without repeatedly contacting the front desk.

---

## Background / Context

Complaint tracking is a trust-critical capability for Browns Hotels & Resorts. When guests raise issues, their confidence in the brand depends on visible, timely follow-through. The complaints list gives guests real-time visibility into the status of every reported issue, reducing follow-up calls to the desk and demonstrating the hotel's commitment to service recovery. A well-handled complaint can convert a dissatisfied guest into a loyal advocate.

---

## Acceptance Criteria

- [ ] The Complaints List is accessible at route `#/complaints` and displays all complaints stored in `vb:complaints[]` that belong to the current active reservation, sorted by `submittedAt` descending (most recent first)
- [ ] Each complaint card displays:
  - A **category icon** matching the complaint's category (e.g. `hotel` for Room, `restaurant` for Dining)
  - A **brief description** of the complaint, truncated to ~80 characters with ellipsis if longer
  - A **priority badge**: "Normal" (default style) or "Urgent" (amber/red badge)
  - A **status badge** with the following colour coding: Submitted = yellow | Acknowledged = blue | In Progress = orange | Resolved = green | Closed = grey
  - The **submitted time** formatted using Day.js `fromNow()` (e.g. "2 hours ago", "just now")
- [ ] Tapping a complaint card navigates to the Complaint Detail page at `#/complaints/{id}`
- [ ] A **"Report an Issue" FAB** (Floating Action Button) is displayed in the bottom-right corner of the screen; it uses the brand primary colour and a `+` or `report_problem` icon; tapping it navigates to the complaint submission form at `#/complaints/new`
- [ ] When no complaints exist for the current reservation, an **empty state** is displayed (no FAB hides — it remains visible): a smiley face illustration, the text "No complaints — we hope your stay is perfect!", and the "Report an Issue" FAB still accessible
- [ ] Complaints of type `"post_stay_feedback"` are **excluded** from this list (they appear separately in the feedback section per US-15.4)

## UI / UX Notes

- Page route: `#/complaints`
- Page header: "My Issues" or "Reported Issues" with a `report_problem` icon; brand top bar
- **FAB**: positioned `position: fixed; bottom: 80px; right: 20px` (above bottom navigation bar); circular, 56px diameter; brand primary background; white icon; box-shadow for elevation; `z-index: 100`
- **Complaint card** layout:
  - Left: category icon (24px, brand accent colour) in a small rounded square container
  - Centre: description text (truncated, 14px) on first line; category name in 12px muted below
  - Right: priority badge + status badge stacked vertically; submitted time in 11px muted below badges
  - Full card is tappable (navigates to detail); subtle tap highlight
- **Priority badge styles**: Normal — no badge (or subtle grey label); Urgent — amber background, white text, `priority_high` icon prefix
- **Status badge colour mapping**:
  - Submitted: `background: #FFF3CD; color: #856404` (yellow)
  - Acknowledged: `background: #CCE5FF; color: #004085` (blue)
  - In Progress: `background: #FFE0B2; color: #E65100` (orange)
  - Resolved: `background: #D4EDDA; color: #155724` (green)
  - Closed: `background: #E9ECEF; color: #6C757D` (grey)
- Status and priority badge: 11px uppercase, `border-radius: 4px`, `padding: 2px 6px`
- Day.js `fromNow()` for submitted time; refresh every 60 seconds for time-ago accuracy
- Card gap: 12px; card border-radius: 12px; card shadow: `0 2px 6px rgba(0,0,0,0.08)`
- Empty state: centred vertically in page; smiley illustration (SVG); 16px muted text; FAB still visible
- Dark mode: all badge background/text colours adjusted for dark surface readability

## Technical Notes

- **Data source**: `JSON.parse(localStorage.getItem('vb:complaints') || '[]')`; filter by `c.reservationId === activeReservation.id && c.type !== 'post_stay_feedback'`
- **Sort**: `.sort((a, b) => dayjs(b.submittedAt).diff(dayjs(a.submittedAt)))`
- **Alpine.js** `x-data="complaintsList()"`:
  - `complaints` — filtered and sorted array
  - `init()` — loads from localStorage; starts 60-second refresh interval for `fromNow()` display
  - `fromNow(dt)` — helper: `dayjs(dt).fromNow()`
  - Uses Day.js `relativeTime` plugin: `dayjs.extend(dayjs_plugin_relativeTime)`
- Category icon map:
  ```js
  const categoryIcons = {
    Room: 'hotel', Dining: 'restaurant', Service: 'room_service',
    Facilities: 'business', Staff: 'person', Other: 'report_problem'
  };
  ```
- Description truncation: CSS `-webkit-line-clamp: 2` or JS `desc.length > 80 ? desc.slice(0,80) + '…' : desc`
- **Status simulation** (from US-15.2 and US-15.3) updates `vb:complaints[]` entries; the list auto-reflects updated statuses because it re-reads localStorage on each 60-second interval (or on page focus)
- Complaints from previous stays (different `reservationId`) are not shown; only current active reservation's complaints are listed

## Test Cases

### Happy Path
1. Guest navigates to `#/complaints`
2. Two complaint cards visible: one "Room — Air conditioning not working" (In Progress, 2 hours ago), one "Dining — Wrong dish served" (Resolved, Yesterday)
3. Cards sorted by `submittedAt` descending — most recent at top
4. Air conditioning card: room icon, "Air conditioning not working" (truncated if needed), "Normal" no badge or hidden, "In Progress" orange badge, "2 hours ago"
5. Dining card: restaurant icon, "Wrong dish served", "Resolved" green badge, "Yesterday"
6. Guest taps air conditioning card → navigates to `#/complaints/{id}`
7. FAB visible bottom-right; tapping navigates to `#/complaints/new`

### Edge Cases
- **No complaints**: Empty state with smiley, "No complaints" text; FAB still visible
- **Only post_stay_feedback entries**: These are filtered out; effective empty state rendered
- **Urgent complaint**: Amber/red "Urgent" priority badge renders prominently; stands out from Normal
- **Long description**: Truncated at ~80 characters; ellipsis visible; full description available on detail page
- **Status simulation running**: Complaint transitions from "Submitted" to "Acknowledged" while list is open → after 60-second tick, status badge updates automatically (no manual refresh)
- **Multiple reservations**: Only active reservation's complaints shown; previous stay complaints excluded
- **Complaint from non-active reservation**: Filtered out correctly even if it was submitted during current session
