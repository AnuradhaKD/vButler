## User Story

**As a** guest,
**I want** to see the full details and live status progression of a complaint I have submitted,
**So that** I know exactly where my issue stands and feel confident that the team is working on it.

---

## Background / Context

Status transparency is the most powerful component of service recovery. When guests can see their complaint progressing through a structured workflow, their anxiety about unresolved issues diminishes and trust in Browns Hotels & Resorts is maintained or restored. The vertical timeline stepper, with pulsing current-stage indicator, mirrors the quality of experience guests expect from a luxury boutique brand and provides a compelling, app-native interaction that generic messaging cannot replicate.

## Acceptance Criteria

- [ ] The Complaint Detail page is accessible at `#/complaints/{id}`; it loads the matching complaint from `vb:complaints[]` by `id`
- [ ] The **page header** displays: the reference number (e.g. "CMP-20260315-4821"), the category icon, and a priority badge (Urgent badge shown prominently; Normal badge may be omitted or shown subtly)
- [ ] A **Complaint Summary section** displays: category name, subcategory, full description text (not truncated), and submitted date/time (formatted as "15 Mar 2026, 09:45 AM")
- [ ] An **Attachments section** is displayed if the complaint has `attachments.length > 0`: thumbnail previews of all attached files in a grid; tapping a thumbnail opens the full-size image or video in a lightbox/full-screen overlay; tapping outside the overlay closes it
- [ ] A **Status Timeline** (vertical stepper) is displayed with the following 5 stages in order: Submitted → Acknowledged → In Progress → Resolved → Closed
  - **Reached stages** (current status and all stages before it): filled circle icon (brand colour or green), stage label in full text, timestamp if available (e.g. "Acknowledged — 15 Mar 2026, 09:48 AM")
  - **Current active stage** (the most recent reached stage that is not Resolved or Closed): pulsing animated indicator (CSS keyframe `pulse` on the circle); label in bold brand colour
  - **Future stages** (not yet reached): empty circle (grey outline), label in muted grey text
- [ ] A **Staff Resolution Notes** section is displayed when the complaint status is "resolved" or "closed": presented as a highlighted card with a `check_circle` icon and italic text; in Phase 1, an auto-generated resolution note is stored when the status simulation reaches "resolved" (e.g. "Our team has addressed the issue. We apologise for the inconvenience and hope this has been resolved to your satisfaction.")
- [ ] A **"Mark as Closed" button** appears when the complaint status is "resolved"; tapping it sets `status: "closed"` and `closedAt: dayjs().toISOString()` in `vb:complaints[]`; the timeline updates to show "Closed" as the final reached stage; the button disappears
- [ ] The page **auto-refreshes** complaint data from localStorage every 30 seconds, so status changes made by the simulation (US-15.2) are reflected without manual reload
- [ ] The page displays a **back link** / back chevron returning the guest to `#/complaints`

## Technical Notes

- **Data loading**: `JSON.parse(localStorage.getItem('vb:complaints') || '[]').find(c => c.id === id)`
- **30-second auto-refresh**: `setInterval(() => this.loadComplaint(), 30000)` in Alpine.js `init()`; must be cleared on component destroy to prevent memory leaks
- **Timeline stage logic**:
  ```js
  const stages = ['submitted', 'acknowledged', 'in_progress', 'resolved', 'closed'];
  const currentIdx = stages.indexOf(this.complaint.status);
  stages.forEach((stage, idx) => {
    this.stageState[stage] = idx < currentIdx ? 'reached' :
                              idx === currentIdx ? 'current' : 'future';
    this.stageTimestamp[stage] = this.complaint[stage + 'At'] || null;
  });
  ```
- **"Mark as Closed" action**:
  ```js
  markClosed() {
    const complaints = JSON.parse(localStorage.getItem('vb:complaints') || '[]');
    const c = complaints.find(c => c.id === this.complaint.id);
    if (c) { c.status = 'closed'; c.closedAt = dayjs().toISOString(); }
    localStorage.setItem('vb:complaints', JSON.stringify(complaints));
    this.complaint = { ...this.complaint, status: 'closed', closedAt: dayjs().toISOString() };
  }
  ```
- **Auto-generated resolution note** (set by US-15.2 simulation when reaching "resolved"):
  ```js
  if (status === 'resolved') {
    c.resolutionNote = 'Our team has addressed the issue. We apologise for the inconvenience and hope this has been resolved to your satisfaction.';
    c.resolvedAt = dayjs().toISOString();
  }
  ```
- **Lightbox**: Alpine.js `x-show` + `x-transition`; `lightboxSrc` stores the base64 or URL of the active attachment; clicking thumbnail sets `lightboxSrc = attachment.data`; clicking overlay or close button sets `lightboxSrc = null`
- Route param extraction: SPA router parses `id` from `#/complaints/{id}` and passes to Alpine component
- **Alpine.js** `x-data="complaintDetail(id)"`: `complaint`, `stageState`, `stageTimestamps`, `lightboxSrc`, `refreshInterval`
- Timestamp display for each stage: only shown if the timestamp field exists in the complaint object (e.g. `acknowledgedAt`, `inProgressAt`, `resolvedAt`, `closedAt`)

## Test Cases

### Happy Path
1. Guest taps "Air conditioning" complaint card on the list → navigates to `#/complaints/{id}`
2. Header: "CMP-20260315-4821", room icon, no priority badge (Normal)
3. Summary card: Room, Air Conditioning, full description, submitted time
4. Timeline: "Submitted" filled green, timestamp; "Acknowledged" pulsing brand colour; "In Progress", "Resolved", "Closed" grey outline
5. After 7 minutes (status simulation): page auto-refreshes; "In Progress" becomes pulsing; "Acknowledged" becomes filled with timestamp
6. After 15 minutes: "Resolved" pulsing; Resolution Notes card appears with auto-generated text
7. "Mark as Closed" button appears; guest taps → "Closed" stage fills; button disappears
8. Guest taps back → returns to Complaints List; complaint card shows "Closed" grey badge

### Edge Cases
- **Complaint with 2 attachments**: Thumbnail grid shows 2 images; tapping one opens lightbox; tapping outside closes
- **Video attachment**: Thumbnail shows play icon overlay; lightbox shows `<video>` tag (autoplay false, controls shown)
- **Complaint already at "Resolved" status on page load**: Resolution Notes card immediately visible; "Mark as Closed" button immediately shown
- **Complaint already "Closed"**: "Mark as Closed" button hidden; full timeline filled to Closed; no pulsing indicator
- **Invalid `id` in route**: Complaint not found in localStorage → "Issue not found" message with "Back to Reports" link
- **Auto-refresh during status transition**: Status updates while guest is reading the page → timeline transitions smoothly without jarring re-render
- **Lightbox on small screen**: Attachment fills available screen space; scroll within lightbox if image taller than viewport
- **Dark mode**: All sections (summary, timeline, resolution card, lightbox) readable; green resolution card adapts to dark background
