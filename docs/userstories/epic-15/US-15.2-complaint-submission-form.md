## User Story

**As a** guest experiencing a problem,
**I want** to report an issue with full details including category, description, and optional photos,
**So that** the Browns Hotels & Resorts team is notified promptly and can resolve my concern quickly.

---

## Background / Context

The complaint submission form is the primary service recovery channel in Destinity vButler. Browns Hotels & Resorts has a strong commitment to guest satisfaction, and the speed and ease of issue reporting directly affects the hotel's ability to recover service failures before they escalate. A well-structured form with category classification, priority selection, and attachment support ensures staff receive actionable information without requiring follow-up calls for clarification.

---

## Acceptance Criteria

- [ ] The form is accessible at `#/complaints/new` navigated to from the "Report an Issue" FAB on the Complaints List (US-15.1)
- [ ] A **Category Selector** is displayed as a grid of icon-and-label tap cards; categories are loaded from `property.json → complaintCategories[]` and include at minimum: Room | Dining | Service | Facilities | Staff | Other; selecting a category card highlights it (brand border/background) and populates the Subcategory dropdown
- [ ] A **Subcategory dropdown** is displayed below the category selector; its options are dynamically populated from the selected `category.subcategories[]` array; if no subcategory is selected, form validation blocks submission
- [ ] A **Description textarea** is provided with: `maxlength="500"`, a character counter displayed below the field showing "X / 500 characters remaining", and a placeholder "Please describe the issue in detail…"; at least 10 characters required for submission
- [ ] An **Attachments section** (optional) allows the guest to attach up to **3 files**; supported types: images (JPG, PNG, WEBP) and video (MP4, MOV); maximum file size **10 MB per file**; selected files are stored as **Base64-encoded strings** in localStorage as part of the complaint entry; a thumbnail preview is shown for each attached image; file selection uses `<input type="file" accept="image/*,video/*" multiple>`
- [ ] A **Priority segmented control** provides two options: "Normal" (default) and "Urgent"; selecting "Urgent" displays an inline warning message: "Use Urgent only for immediate safety concerns or emergencies"
- [ ] A **Reservation context bar** is persistently displayed at the top of the form: "Reporting for: Room [roomNumber], [Property Name]" — sourced from the active reservation
- [ ] A **"Submit Report" primary button** (full-width, brand colour) is displayed at the bottom; it is disabled until Category, Subcategory, and Description (≥ 10 characters) are all provided
- [ ] On successful submission:
  - A complaint entry is written to `vb:complaints[]` with: `{ id, referenceNumber, reservationId, category, subcategory, description, priority, attachments: [base64...], status: "submitted", submittedAt, type: "complaint" }`
  - A notification is created in `vb:notifications[]`: "Your issue has been reported. Reference: [CMP-YYYYMMDD-XXXX]"
  - Status simulation is initiated: `setTimeout` chain → 3 minutes: status → "acknowledged" | 7 minutes: status → "in_progress" | 15 minutes: status → "resolved"
- [ ] A **Success state** is displayed on the same page (form replaced): reference number `CMP-YYYYMMDD-XXXX`, heading "We're on it!", subtext "Our team will acknowledge your report within 15 minutes", a summary of the category + description, and a "View My Reports" link navigating to `#/complaints`

## Technical Notes

- **Category data from JSON**: `property.json → complaintCategories[]` structure:
  ```json
  [{ "id": "room", "name": "Room", "icon": "hotel", "subcategories": ["Air Conditioning", "Cleanliness", "Plumbing", "Noise", "Other"] }]
  ```
- **Subcategory population**: Alpine.js reactive — `x-on:click` on category card sets `selectedCategory = cat`; subcategory `<select>` options rendered via `x-for` over `selectedCategory.subcategories`
- **File attachment handling**:
  ```js
  handleFiles(files) {
    Array.from(files).slice(0, 3 - this.attachments.length).forEach(file => {
      if (file.size > 10 * 1024 * 1024) { this.showError('File too large (max 10MB)'); return; }
      const reader = new FileReader();
      reader.onload = e => this.attachments.push({ name: file.name, type: file.type, data: e.target.result });
      reader.readAsDataURL(file);
    });
  }
  ```
- **Character counter**: `x-text="500 - description.length + ' characters remaining'"` bound to description textarea
- **Status simulation** (setTimeout chain):
  ```js
  initiateStatusSimulation(complaintId) {
    setTimeout(() => this.updateStatus(complaintId, 'acknowledged'), 3 * 60 * 1000);
    setTimeout(() => this.updateStatus(complaintId, 'in_progress'), 7 * 60 * 1000);
    setTimeout(() => this.updateStatus(complaintId, 'resolved'), 15 * 60 * 1000);
  }
  updateStatus(id, status) {
    const complaints = JSON.parse(localStorage.getItem('vb:complaints') || '[]');
    const c = complaints.find(c => c.id === id);
    if (c) { c.status = status; c[status + 'At'] = dayjs().toISOString(); }
    localStorage.setItem('vb:complaints', JSON.stringify(complaints));
  }
  ```
- **Reference number**: `CMP-${dayjs().format('YYYYMMDD')}-${Math.floor(1000 + Math.random() * 9000)}`
- **Submit button enable logic**: `isSubmitEnabled = selectedCategory && selectedSubcategory && description.trim().length >= 10`
- **Alpine.js** `x-data="complaintForm()"`: `selectedCategory`, `selectedSubcategory`, `description`, `priority` ('normal'|'urgent'), `attachments[]`, `submitted`, `referenceNumber`
- Base64 attachment storage concern: with 3 × 10MB files, localStorage may be exceeded; warn the user before FileReader if `attachments` would exceed safe threshold; consider compressing images client-side in Phase 2

## Test Cases

### Happy Path
1. Guest taps "Report an Issue" FAB → navigates to `#/complaints/new`
2. Reservation context bar: "Reporting for: Room 205, Browns Beach Hotel"
3. Guest taps "Room" category card → highlights; subcategory dropdown populates with Room subcategories
4. Guest selects "Air Conditioning" subcategory
5. Guest types "The air conditioning in my room has stopped working. Room is very hot." (description ≥ 10 chars)
6. Character counter: "421 / 500 characters remaining"
7. Priority: "Normal" selected; no warning shown
8. Guest attaches 1 photo (JPG, 2MB) → thumbnail preview appears
9. Submit button is enabled; guest taps "Submit Report"
10. Success state: "We're on it!", "CMP-YYYYMMDD-XXXX", category + description summary
11. `vb:complaints[]` contains new entry with `status: "submitted"`; notification created
12. After 3 minutes: status updates to "acknowledged" in localStorage
13. "View My Reports" link → navigates to Complaints List; new complaint card visible

### Edge Cases
- **Category not selected**: Submit button disabled; tapping shows no-op (button disabled)
- **Subcategory not selected**: Submit button disabled even if description and category provided
- **Description < 10 characters**: Submit button disabled; character counter shows near-limit warning
- **File > 10MB**: Error toast "File too large (max 10MB)"; file not added to attachments
- **4th file attachment attempt**: 4th file rejected with toast "Maximum 3 attachments reached"
- **Urgent priority selected**: Warning banner appears; confirmed that "Urgent" badge appears on the resulting complaint card in US-15.1
- **Description at character limit (500)**: Character counter reads "0 characters remaining" in red; typing further characters is blocked by `maxlength`
- **Storage quota warning**: If Base64 attachments would exceed localStorage quota, show error before FileReader
