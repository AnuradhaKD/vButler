## User Story

**As a** guest on an active stay,
**I want** to schedule a wake-up call for a specific time and date,
**So that** I can be reminded to wake up at the right time without relying on a physical bedside phone or front desk call.

---

## Background / Context

Browns Hotels & Resorts guests often require early morning wake-up calls for tours, transfers, or business commitments. The Destinity vButler app provides a self-service digital alternative to the traditional front desk wake-up call, reducing staff load while giving the guest full control. All scheduling is managed client-side via localStorage and surfaced in the requests queue for staff visibility.

## Acceptance Criteria

- [ ] The Wake-Up Call feature is only accessible and functional when `reservation.status === "active"`; guests with upcoming or checked-out reservations see a contextual message ("Available during your stay") and the scheduling form is disabled
- [ ] A large, centred Flatpickr time picker is presented allowing selection in 15-minute increments (00:00, 00:15, 00:30 … 23:45) using HH:MM 12-hour display format (e.g., 06:30 AM)
- [ ] Date selection is provided via three radio pill options: **Today**, **Tomorrow**, and **Specific Date**; selecting "Specific Date" reveals a Flatpickr date picker constrained to the guest's remaining stay dates (check-in date to day before check-out)
- [ ] A **Recurring** toggle is present labelled "Repeat for all remaining nights"; when enabled, one wake-up call entry per remaining night of stay is created in localStorage
- [ ] A live visual preview sentence updates in real time below the form: "Your wake-up call is set for [HH:MM AM/PM] on [Weekday, D MMM]" — updates whenever time or date selection changes
- [ ] A full-width primary "Schedule Wake-Up Call" button using brand background colour is present at the bottom of the form
- [ ] On successful submission, the wake-up call is written to `vb:wakeupCalls:{resvId}` as an array entry AND a corresponding entry is appended to `vb:requests[]` with `type: "wakeup_call"`, `reservationId`, `scheduledAt` (ISO 8601), `isRecurring`, and `status: "scheduled"`
- [ ] A success state is rendered on the same page (no navigation): green checkmark icon, heading "Wake-Up Call Scheduled", a summary line showing the time and date, and a unique reference number (format: `WC-YYYYMMDD-XXXX`)
- [ ] The "Schedule Wake-Up Call" button is disabled and a message is shown ("Wake-up calls must be scheduled at least 30 minutes in advance") when the selected date-time is fewer than 30 minutes from the current time (`dayjs().add(30, 'minute').isAfter(selectedDateTime)`)
- [ ] When Recurring is enabled, the preview updates to: "Your wake-up call is set for [HH:MM AM/PM] every morning for [N] remaining nights"
- [ ] Form resets after successful submission, allowing the guest to schedule another call

## Technical Notes

- **Day.js** is used for all date/time computation:
  - `dayjs().add(30, 'minute')` for the 30-minute constraint check (re-evaluated on every time/date change)
  - Remaining nights = `dayjs(reservation.checkOutDate).diff(dayjs().startOf('day'), 'day')` — ensures accurate count from today
  - For Recurring, iterate from tomorrow's date to `checkOutDate - 1` generating one entry per night with the same time
- **Flatpickr** time picker config: `{ enableTime: true, noCalendar: true, dateFormat: "H:i", minuteIncrement: 15, time_24hr: false }`
- **Flatpickr** date picker config (Specific Date): `{ minDate: "today", maxDate: checkOutDate - 1 day, disable: [dates outside stay] }`
- **localStorage keys**:
  - `vb:wakeupCalls:{resvId}` — array of `{ id, scheduledAt, isRecurring, status, createdAt, referenceNumber }`
  - `vb:requests[]` — append entry with `{ id, type: "wakeup_call", reservationId, scheduledAt, isRecurring, status: "scheduled", createdAt, referenceNumber }`
- Reference number generation: `WC-${dayjs().format('YYYYMMDD')}-${Math.floor(1000 + Math.random() * 9000)}`
- **Alpine.js** `x-data` component manages: `selectedTime`, `selectedDate`, `dateMode` (today/tomorrow/specific), `isRecurring`, `remainingNights`, `isWithin30Min` (computed), `submitted`, `referenceNumber`
- The 30-minute constraint `isWithin30Min` is a computed Alpine getter, re-evaluated via `$watch` on `selectedTime` and `selectedDate`
- Guard check on page load: if `activeReservation.status !== 'active'` render disabled/informational view only

## Test Cases

### Happy Path
1. Guest opens the app with an active reservation (status: "active")
2. Navigate to Quick Actions → Wake-Up Call (`#/wakeup-call`)
3. The scheduling form loads with the time picker defaulting to 07:00 AM
4. Guest selects 06:30 AM using the Flatpickr time picker
5. "Today" radio pill is selected by default; live preview reads "Your wake-up call is set for 06:30 AM on [Today's Weekday, D MMM]"
6. Guest selects "Tomorrow" pill; preview updates to tomorrow's date
7. Guest taps "Schedule Wake-Up Call" button
8. Success state appears: green checkmark, "Wake-Up Call Scheduled", "06:30 AM — [Tomorrow's Date]", reference number WC-XXXXXXXX-XXXX
9. `vb:wakeupCalls:{resvId}` array contains new entry; `vb:requests[]` contains matching entry
10. "Schedule Another" link reloads the form

### Edge Cases
- **Within 30 minutes**: Guest selects a time 20 minutes from now → button is disabled, message shown; adjusting to 35 minutes from now re-enables the button
- **Recurring toggle — 1 night remaining**: Toggle enabled with only 1 night left → info chip reads "Creates 1 wake-up call"; single entry created
- **Recurring toggle — 0 nights remaining** (day of check-out): Toggle is hidden or disabled; only "Today" date option available (if check-out is not today)
- **Specific Date selected then changed to Today**: Flatpickr date picker collapses, preview resets to today
- **Offline / localStorage full**: Catch `QuotaExceededError` on `setItem`; show toast "Unable to save — storage limit reached"
- **Rapid form submission**: Button enters loading state on first click; duplicate submissions prevented
- **Reservation not active**: Navigate directly to `#/wakeup-call` → form is fully disabled; message: "Wake-Up Call scheduling is available during your active stay"
