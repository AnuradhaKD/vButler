## User Story

**As a** hotel guest,
**I want** to see all spa treatments, fitness classes, activities, and wellness packages available at the property in one place,
**So that** I can plan and book my wellness experiences without consulting the spa reception or printed brochures.

---

## Background / Context

Spa and wellness services represent a key revenue stream and guest satisfaction driver at Browns Hotels & Resorts luxury properties. Guests, particularly those on leisure stays, actively seek wellness activities but often do not know what is available unless it is prominently surfaced. The Wellness Hub acts as the top-level navigation screen for EPIC-08, providing a curated, visually inviting overview that leads guests into bookable detail flows.

## Acceptance Criteria

- [ ] The Wellness Hub is accessible from the Home Quick Actions grid via a "Wellness" or "Spa" tile
- [ ] A hero banner is displayed at the top with a brand-gradient background and a spa icon (lotus or leaf SVG); headline "Rejuvenate & Restore" is rendered in Playfair Display
- [ ] The screen is divided into three labelled sections: "Spa Treatments", "Classes & Activities", and "Packages"
- [ ] Each section displays up to 4 items as cards; if more than 4 items exist, a "View All" link is shown at the section footer
- [ ] Tapping "View All" expands the section to show all items (inline expansion, no new screen)
- [ ] If a section has no items configured for the property (`wellness.treatments[]`, `wellness.classes[]`, or `wellness.packages[]` is empty or absent), that section is hidden entirely — no empty-state card
- [ ] Sections are rendered only from `property.json → wellness` data via `DataService`
- [ ] Each section header includes a section icon (e.g., candle for Spa, yoga pose for Classes, gift for Packages)
- [ ] Tapping a Spa Treatment card navigates to the treatment detail drawer (US-08.2)
- [ ] Tapping a Class card navigates to the class/activity schedule view (US-08.3)
- [ ] Tapping a Package card navigates to the package detail (US-08.5)
- [ ] Screen renders correctly in dark mode

## UI / UX Notes

- Hero banner: full-width, 200 px height, brand gradient (primary to secondary), spa/lotus SVG icon centred, headline below icon in Playfair Display 26 px white; subtext "Curated wellness experiences for your stay" in Inter 13 px white/70%
- Section headers: 18 px semi-bold Inter, brand primary colour, section icon (20 px SVG) left-aligned, "View All" link right-aligned in brand primary 13 px
- Cards within each section: horizontal scroll on mobile (snap-scroll), 2-column grid on tablet+
- Treatment card preview: name, duration chip, price (or "Complimentary") — details in US-08.2
- Class card preview: name, next occurrence day+time, "Complimentary" or price — details in US-08.3
- Package card preview: name, session count, total price — details in US-08.5
- Scroll-to-section anchors: if navigated from a deep link (e.g., `#/wellness?section=treatments`), page scrolls to that section smoothly

---

## Technical Notes

- Route: `#/wellness`
- Alpine.js component: `x-data="wellnessHub()"` with:
  - `treatments`, `classes`, `packages` arrays loaded from `DataService.getProperty().wellness`
  - `showAllTreatments`, `showAllClasses`, `showAllPackages` boolean toggles (default `false`)
  - Computed `visibleTreatments` — first 4 if `!showAllTreatments`, else all
  - Same pattern for classes and packages
- Section visibility: `x-show="treatments.length > 0"` pattern
- Navigation to child screens: `router.navigate('#/wellness/treatment?id=X')` etc.
- Optional query param `?section=treatments|classes|packages` scrolls to anchor on load using `$nextTick` + `element.scrollIntoView({ behavior: 'smooth' })`

## Test Cases

### Happy Path

1. Guest taps "Wellness" in Quick Actions on Home screen
2. Wellness Hub loads with hero banner and headline "Rejuvenate & Restore"
3. Three sections visible: Spa Treatments (4 cards shown, "View All" link visible for 7 total), Classes & Activities (3 cards), Packages (2 cards)
4. Guest taps "View All" under Spa Treatments → all 7 treatment cards appear inline
5. Guest taps "Hot Stone Massage" treatment card → US-08.2 detail drawer opens
6. Guest taps "Yoga at Sunrise" class card → US-08.3 schedule view opens
7. Guest taps "Couples Retreat Package" → US-08.5 package detail opens

### Edge Cases

- **No packages configured**: "Packages" section is hidden; no error; other sections unaffected
- **All wellness sections empty**: Screen shows only the hero banner with a message "Wellness services are being configured for this property"
- **Exactly 4 treatments**: "View All" link is hidden (no need to expand)
- **Deep link `#/wellness?section=classes`**: Page scrolls to Classes & Activities section on load
- **Dark mode**: Hero, section headers, cards all verified
