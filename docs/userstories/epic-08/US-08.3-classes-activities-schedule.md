## User Story

**As a** hotel guest,
**I want** to view a schedule of fitness classes and wellness activities available during my stay and book a spot in any class,
**So that** I can plan my days around activities I enjoy and ensure I have a confirmed place in popular sessions.

---

## Background / Context

Luxury guests at Browns Hotels & Resorts properties increasingly expect curated wellness programming — sunrise yoga, aqua aerobics, guided meditation, and beach fitness sessions. A clear schedule filtered to the guest's remaining stay days, with real-time availability indicators, enables guests to engage with these offerings rather than missing them through lack of awareness. The booking confirmation integrates with the central request system for operational visibility.

---

## Acceptance Criteria

- [ ] Classes & Activities are displayed in a schedule view within or navigable from the Wellness Hub (US-08.1)
- [ ] Schedule is filtered to show only today plus the next 7 days (or until check-out, whichever is sooner) based on `class.availableDays` (array of day names, e.g., `["Monday", "Wednesday", "Friday"]`) and `class.startTime`
- [ ] Schedule is grouped by date (e.g., "Today — Sunday, 15 March", "Tomorrow — Monday, 16 March") with date headers
- [ ] Each class entry shows: class name, start time, duration (e.g., "45 min"), price ("Complimentary" if `price: 0`, otherwise LKR X,XXX), instructor name, and maximum participants
- [ ] A "Book" button is shown for each class entry
- [ ] If the simulated participant count for a session has reached `class.maxParticipants`, the "Book" button is replaced with a "Full" badge (grey, disabled); no booking is possible
- [ ] Tapping "Book" opens the Wellness Booking Flow (US-08.4) with `classId` and the specific session date/time pre-populated
- [ ] If no classes are scheduled within the filter window (e.g., all classes run Mon/Wed/Fri and guest is checking out Sunday), a message is shown: "No classes scheduled during your remaining stay — check back or ask our wellness team"
- [ ] All class data is sourced from `property.json → wellness.classes[]`
- [ ] Screen renders correctly in dark mode

## UI / UX Notes

- Schedule layout: vertical timeline-style list grouped by date
- Date header: 14 px semi-bold Inter, muted background strip (full-width), brand primary date text
- Class entry card: white background (dark: surface-card), 12 px radius, 12 px padding; left accent colour bar varies by class type (yoga: green, fitness: orange, meditation: purple, aqua: blue — driven by `class.type` or `class.colour` field)
- Class name: 15 px semi-bold; time + duration: 13 px muted; instructor: 13 px italic muted
- Price pill: right-aligned, small rounded pill; "Complimentary" in green; LKR price in brand accent
- "Book" button: brand primary outline, small (32 px height), right-aligned on the entry card
- "Full" badge: grey pill, "Full" text, 12 px — replaces "Book" button
- Simulated "Full" state: `class.simulatedBookedCount >= class.maxParticipants` (both in JSON for demo purposes)
- Participant count shown as "X / Y spots" (e.g., "4 / 8 spots") below instructor name

## Technical Notes

- Route: `#/wellness/classes` or rendered inline within Wellness Hub
- Alpine.js component: `x-data="classesSchedule()"` with:
  - `classes` — loaded from `DataService`
  - `scheduledSessions` — computed array of `{classObj, sessionDate}` objects generated by expanding `class.availableDays` across today + 7 days
  - `groupedByDate` — computed object keyed by ISO date string, values are arrays of sessions for that date
- Session generation logic: for each day in the next 7 days (capped at check-out), check if that day's name is in `class.availableDays`; if yes, create a session entry
- "Full" simulation: `class.simulatedBookedCount` field compared to `class.maxParticipants`; this simulates a full class without a real backend
- `formatLKR(price)` utility for price display; `price === 0` → "Complimentary"
- Navigation to booking: `router.navigate('#/wellness/book?classId=' + class.id + '&date=' + sessionDate)`

## Test Cases

### Happy Path

1. Guest navigates to Classes & Activities from Wellness Hub
2. Schedule shows grouped entries: "Today — Sunday, 15 March" with "Beach Yoga" at 07:00 (45 min, Complimentary, Instructor: Priya R., 3/8 spots), and "Tomorrow — Monday, 16 March" with "Aqua Aerobics" at 09:00 (LKR 1,500, 5/10 spots)
3. Guest taps "Book" on Beach Yoga → US-08.4 opens with class and date pre-populated
4. "Mindful Meditation" session shows "Full" badge (8/8 spots); no Book button

### Edge Cases

- **No classes on remaining stay days**: Empty schedule message shown
- **Class with `availableDays: ["Daily"]`**: Session generated for every day in the window
- **Check-out is tomorrow**: Only today and tomorrow shown
- **All classes are full**: All entries show "Full" badge; empty-state message not triggered (sessions exist, just full)
- **Class price exactly LKR 0**: "Complimentary" pill shown in green
- **Dark mode**: Date headers, card accent bars, pills verified
