## User Story

**As a** hotel guest,
**I want** to view curated wellness packages that bundle multiple treatments or sessions at a bundled price,
**So that** I can enjoy a comprehensive wellness experience at better value and have everything arranged in one booking.

---

## Background / Context

Wellness packages are a key upsell opportunity at luxury spa properties. Guests who may not seek out individual treatments will often be attracted to a named package such as "Couples Retreat" or "The Full Restore Day" that promises a complete, curated experience. The package card surface within the Wellness Hub presents these offerings attractively, with inclusions and savings clearly communicated to motivate conversion.

---

## Acceptance Criteria

- [ ] The Packages section is displayed in the Wellness Hub (US-08.1) only when `wellness.packages[]` is present and non-empty; section is hidden if not configured
- [ ] Each package is displayed as a card showing: package name, number of sessions/treatments included (e.g., "3 sessions"), inclusions as a bulleted list (up to 4 shown, with "and X more…" if there are more than 4), total price in LKR, and per-session saving ("Save LKR X,XXX vs individual pricing" if `package.individualTotal` is provided)
- [ ] Tapping the package card expands it (inline or bottom drawer) to show the full inclusions list and a longer description
- [ ] A "Book Package" button is present on each card (collapsed) and within the expanded detail view
- [ ] Tapping "Book Package" navigates to the Wellness Booking Flow (US-08.4) with `packageId` pre-populated
- [ ] The saving amount is calculated as: `package.individualTotal - package.price`; if `package.individualTotal` is absent, no saving message is shown
- [ ] All package data is sourced from `property.json → wellness.packages[]`
- [ ] If the section has packages but they all lack a price (e.g., price on request), the price field shows "Price on Request" and the booking flow still proceeds
- [ ] Screen renders correctly in dark mode

## UI / UX Notes

- Package card: wider card format (full-width on mobile), brand gradient top border (4 px), package name in Playfair Display 17 px, session count chip right-aligned (brand accent bg)
- Inclusions list (collapsed): checkmark icons, 13 px, up to 4 items; "and X more…" in muted italic if overflow
- Price row: total price bold 18 px brand primary left; saving badge right ("Save LKR 4,500") in green pill if applicable
- "Book Package" button: brand primary filled, full-width, 44 px
- Expanded state (drawer): full inclusions list, longer description, duration/validity note (e.g., "Valid for 2 guests · Use within your stay"), and "Book Package" button pinned to bottom
- Price on Request: "Contact our spa team to arrange" link → opens WhatsApp or tel: link from `property.spaContact`
- Cards in 1-column layout on mobile; 2-column on tablet+

---

## Technical Notes

- Package card rendered within the Wellness Hub Alpine.js component (or its own sub-component for the expanded drawer, following US-08.2's pattern)
- Saving calculation: `savingAmount = package.individualTotal - package.price`; shown only if `package.individualTotal && savingAmount > 0`
- "Price on Request": `package.price === null || package.price === undefined` → show "Price on Request"; `FolioService.addCharge` not called; booking flow proceeds without a charge line
- Navigation: `router.navigate('#/wellness/book?packageId=' + package.id)`
- `package.sessionCount` used for sessions chip; if absent, derive from `package.inclusions.length`
- Inclusions overflow: `inclusions.slice(0, 4)` for collapsed; full array in expanded

## Test Cases

### Happy Path

1. Guest views Wellness Hub; "Packages" section shows 2 cards: "The Full Restore Day" and "Couples Retreat"
2. "The Full Restore Day" card shows: "3 sessions", 4 inclusions with checkmarks, LKR 24,000, "Save LKR 4,500" green pill
3. Guest taps "Book Package" → US-08.4 opens with packageId pre-populated
4. Guest taps card body → expanded drawer slides up with full inclusions, description, and duration note
5. Guest taps "Book Package" in drawer → same navigation to US-08.4

### Edge Cases

- **No packages configured**: Packages section hidden entirely; other Wellness Hub sections unaffected
- **Package with no `individualTotal`**: No saving badge shown; price still displayed
- **Package with more than 4 inclusions**: "and 2 more…" shown in collapsed card; all inclusions shown in drawer
- **Price on Request**: "Price on Request" shown; "Book Package" leads to a different CTA ("Enquire via WhatsApp")
- **Dark mode**: Gradient border, inclusion checkmarks, saving pill verified
