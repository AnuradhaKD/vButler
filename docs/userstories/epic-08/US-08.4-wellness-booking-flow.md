## User Story

**As a** hotel guest,
**I want** to book a spa treatment, wellness class, or package through a clear and guided booking form,
**So that** I have a confirmed appointment with a reference number and know exactly when to show up and what to expect.

---

## Background / Context

The Wellness Booking Flow is the shared transaction screen for all bookable wellness items — spa treatments, classes, activities, and packages. Centralising the booking logic into a single reusable flow reduces implementation duplication and ensures a consistent guest experience across all wellness booking types. The flow must handle therapist preference (treatment-specific), guest count (class-specific), and health notes, while always surfacing the cancellation policy before confirmation.

## Acceptance Criteria

- [ ] The booking flow is reachable from: (a) "Book Now" on a treatment card (US-08.2), (b) "Book" on a class entry (US-08.3), and (c) "Book Package" on a package card (US-08.5)
- [ ] The booking type (treatment / class / package) and selected item details are pre-populated from the navigation route params (`?treatmentId=X`, `?classId=X`, `?packageId=X`, `?date=YYYY-MM-DD`, `?time=HH:MM`)
- [ ] The screen heading displays the item name in Playfair Display
- [ ] Date picker (Flatpickr) is constrained to the guest's stay dates (`vb:session.checkIn` to `vb:session.checkOut` inclusive)
- [ ] Available time slots are rendered as pill buttons; slots are derived from the treatment/class schedule and available hours; slots in the past for today are disabled
- [ ] Therapist preference field (radio pills: Any / Female / Male) is displayed ONLY when `treatment.therapistPreference === true` for spa treatments; it is hidden for classes and packages
- [ ] Number of guests stepper (1 to `class.maxParticipants`) is displayed for class bookings; hidden for individual treatments
- [ ] Special health notes text field (optional, max 500 chars, with live char counter) is always shown; placeholder: "Please share any health conditions, injuries, or preferences our therapist should know"
- [ ] Cancellation policy info card is displayed above the Confirm button, sourced from `item.cancellationPolicy` or `item.cancellationCutoffHours`
- [ ] "Confirm Booking" button is disabled until date and time are selected
- [ ] On confirm: a request is written to `vb:requests[]` with `category: "wellness"`, appropriate `type` (`spa_treatment` / `wellness_class` / `wellness_package`), all form values, and the charge amount
- [ ] Charge is added to guest's folio via `FolioService.addCharge`; complimentary items (price = 0) do not generate a folio charge
- [ ] A success confirmation card is shown with: item name, date, time, therapist preference (if applicable), cancellation deadline (calculated from `cancellationCutoffHours`), and reference number (`SPA-{timestamp}`)
- [ ] Screen renders correctly in dark mode

## UI / UX Notes

- Screen header: back button (left), item category label (right, e.g., "Spa Treatment", "Wellness Class")
- Item summary card at top: item name (Playfair Display 20 px), duration chip, price (LKR) or "Complimentary", brand gradient left border
- Date picker: Flatpickr input, full-width, 48 px; brand primary highlight on selected date
- Time slots: 2-column pill grid; selected pill filled brand primary; disabled pills muted with strikethrough
- Therapist preference: visible only for eligible treatments; horizontal radio pills (Any / Female / Male); "Any" selected by default
- Number of guests stepper: centred layout with large minus/plus buttons (brand primary); count in 20 px bold
- Health notes textarea: 120 px height, brand primary focus border, char counter bottom-right "0/500"
- Cancellation policy card: amber background (8% opacity), amber icon, 13 px italic text
- Success card: brand gradient top border (4 px), animated checkmark SVG, item name 18 px semi-bold, date/time row, cancellation deadline in amber italic, "SPA-XXXX" in monospace 16 px bold
- Action buttons on success: "View in My Requests" (outlined) + "Back to Wellness" (filled brand primary)

## Technical Notes

- Route: `#/wellness/book`; reads `treatmentId`, `classId`, or `packageId` from URL params plus optional `date` and `time`
- Alpine.js component: `x-data="wellnessBooking()"`:
  - `bookingType` — derived from which param is present
  - `item` — loaded via `DataService` using the appropriate ID
  - `selectedDate`, `selectedTime`, `therapistPreference`, `guestCount`, `healthNotes`
  - `isComplimentary` — computed `item.price === 0`
  - `cancellationDeadline` — computed `selectedDate + selectedTime - item.cancellationCutoffHours`
- Time slot generation reuses `generateTimeSlots(openTime, closeTime, intervalMins)` utility from US-07.3
- Folio charge skipped if `isComplimentary === true`
- Reference number: `"SPA-" + Date.now()`
- Request schema:
  ```json
  {
    "id": "SPA-1700000000000",
    "category": "wellness",
    "type": "spa_treatment",
    "itemId": "treatment-001",
    "itemName": "Ayurvedic Abhyanga",
    "date": "2026-03-18",
    "time": "10:00",
    "durationMinutes": 60,
    "therapistPreference": "Female",
    "guestCount": 1,
    "healthNotes": "Lower back sensitivity",
    "price": 8500,
    "status": "confirmed",
    "cancellationCutoffHours": 24,
    "createdAt": "2026-03-15T10:22:00Z"
  }
  ```

## Test Cases

### Happy Path

1. Guest taps "Book Now" on "Hot Stone Massage" (90 min, LKR 12,500, therapistPreference: true)
2. Booking flow loads: "Hot Stone Massage" in Playfair Display, "90 min" chip, "LKR 12,500" price
3. Guest selects 18 March via date picker; time slots appear: 09:00, 09:30 … 16:00
4. Guest selects 10:30; therapist preference "Female" selected; health notes: "Shoulder injury, avoid deep pressure"
5. Cancellation policy: "Free cancellation up to 24 hours before your appointment"
6. Guest taps "Confirm Booking" → success card: "Hot Stone Massage · 18 Mar 2026 · 10:30 · Female therapist · Cancel by 17 Mar 2026, 10:30 · Ref: SPA-1710000000000"
7. Folio charge LKR 12,500 added

### Edge Cases

- **Complimentary class booking**: Folio charge not created; "Complimentary" shown on success card instead of price
- **Therapist preference hidden for class**: Stepper for guest count shown instead
- **Date pre-populated from class schedule**: Date picker shows pre-selected date; time slot pre-selected from route param
- **No available time slots for selected date**: Message "No availability on this date — please try another day"
- **Health notes at 500 char limit**: Further input blocked; char counter shows "500/500"
- **Cancellation policy absent**: Policy card hidden gracefully
- **Dark mode**: All form elements, success card, amber policy card verified
