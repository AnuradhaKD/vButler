## User Story

**As a** hotel guest,
**I want** a centralised settings screen to control my app appearance, language, notification preferences, and account options,
**So that** I can customise the Destinity vButler experience to match my personal preferences and comfort.

---

## Background / Context

Guests at Browns Hotels & Resorts come from diverse backgrounds — local Sri Lankan guests, diaspora travellers, and international visitors from Germany, the Middle East, and beyond. The Settings screen must accommodate language preferences (including RTL Arabic), appearance choices, and granular notification controls. All settings are persisted locally and take effect immediately without requiring a full page reload.

---

## Acceptance Criteria

- [ ] The Settings screen is accessible from the Profile screen via a "Settings" tab or a gear icon in the profile header.
- [ ] The Settings screen is accessible via a gear icon in the app header or navigation.
- [ ] The **Appearance** section displays three mutually exclusive radio button options: Light, Dark, System. Selecting one applies the theme immediately via `ThemeService` and persists the selection.
- [ ] The **Language** section displays a dropdown with five languages: English, සිංහල (Sinhala), தமிழ் (Tamil), العربية (Arabic), Deutsch (German). Selecting a language triggers an immediate UI language switch via `I18nService` without a page reload.
- [ ] The selected language is saved to `vb:language` in localStorage.
- [ ] The **Notifications** section contains four toggles: Service Updates, Billing Alerts, Promotions, Check-in Reminders. Each toggle state is saved independently to `vb:profile.notificationPrefs`.
- [ ] Toggle states are pre-loaded from `vb:profile.notificationPrefs` on screen mount.
- [ ] The **Account** section contains: "Change Password" (navigates to the change-password screen/modal), "Linked Devices" (placeholder for future; displays count "1 device"), "Delete Account" (navigates to account deletion flow — US-16.5).
- [ ] The **Legal** section contains: "Privacy Policy" and "Terms & Conditions" links that open in a new browser tab via an external URL.
- [ ] The **Help & Support** section contains: "Help Centre" link (external), "Contact Us" (opens `mailto:support@brownshotels.lk`), and a non-interactive "App Version" row displaying the current version string (e.g., "v1.0.0").
- [ ] All sections are rendered as distinct grouped list cards with clear section headings.
- [ ] The Settings screen respects the current theme (light/dark) and updates to RTL layout when Arabic is selected.
- [ ] Navigating away and returning to Settings shows all settings in their last-saved state.

## UI / UX Notes

- Screen layout: scrollable single-column list; each section is a labelled group with a top heading (12px uppercase, muted, letter-spacing).
- Section cards: white (light) / dark-surface (dark) background, rounded-lg, shadow-sm; rows separated by a 1px divider line.
- Row height: 52px minimum; icon on the left (20px, muted colour), label in the centre, right-side control (toggle, chevron, radio, or value text).
- Appearance radio buttons: visually styled as a 3-segment pill selector (not default OS radio); active segment: brand fill.
- Language dropdown: shows flag emoji + language name in its native script; selected language shown in the closed state.
- Notification toggles: iOS-style toggle switch; brand colour (#003c52) when on; grey when off; label describes what the category covers (sub-label 12px muted).
  - Service Updates sub-label: "Requests, complaints, status changes"
  - Billing Alerts sub-label: "Charges and payment confirmations"
  - Promotions sub-label: "Offers and seasonal deals"
  - Check-in Reminders sub-label: "Pre-arrival and check-in nudges"
- "Delete Account" row: destructive red text colour to signal danger.
- "App Version" row: value text right-aligned (e.g., "v1.0.0 (Build 42)"), no chevron, non-tappable.
- External links: show an external-link icon suffix (↗) after the label.
- On language change to Arabic: entire Settings screen (including itself) re-renders RTL immediately.

## Technical Notes

- ThemeService: `ThemeService.set('light' | 'dark' | 'system')` applies `dark` class to `<html>` for dark mode; reads `prefers-color-scheme` media query for "System" mode; persists to `vb:theme`.
- I18nService: `I18nService.setLanguage(code)` loads the language JSON, updates `Alpine.store('ui').language`, triggers all `$t()` bindings to re-evaluate.
- Notification preferences schema in `vb:profile.notificationPrefs`:
  ```json
  {
    "serviceUpdates": true,
    "billingAlerts": true,
    "promotions": false,
    "checkInReminders": true
  }
  ```
- Toggles: bind each to `$store.profile.notificationPrefs.{key}` with `@change` calling `StorageService.set('vb:profile', ...)`.
- App version: read from a `window.APP_VERSION` constant set in a build config or inline script in `index.html`.
- "Linked Devices" row: static display; `vb:devices` key can be scaffolded as `[{ id, name, lastSeen }]` for future use.
- Legal URLs: stored as constants in `/js/config.js` (e.g., `PRIVACY_POLICY_URL`, `TERMS_URL`).
- Settings screen route: `#/settings` — accessible directly via URL hash.

## Test Cases

### Happy Path

1. Log in and navigate to Profile → Settings.
2. Verify all six sections are visible: Appearance, Language, Notifications, Account, Legal, Help & Support.
3. In Appearance, select "Dark" — verify the entire app switches to dark mode immediately.
4. Reload the app — verify dark mode is retained.
5. In Language, select "Deutsch" — verify all UI labels switch to German immediately.
6. Navigate away and return to Settings — verify German is still selected.
7. Toggle "Promotions" off — navigate away — return to Settings — verify "Promotions" toggle is still off.
8. Tap "Change Password" — verify navigation to the Change Password screen/modal.
9. Tap "Privacy Policy" — verify it opens in a new browser tab.
10. Tap "Contact Us" — verify the device's email client opens with `mailto:support@brownshotels.lk`.

### Edge Cases

- **System theme, device switches dark/light:** App appearance changes dynamically if "System" is selected.
- **Language switch to Arabic:** RTL layout applies instantly; Settings screen itself re-renders RTL.
- **All notification toggles off:** Subsequent service events do not create opt-out notification types (validated in US-17.4).
- **`vb:profile.notificationPrefs` missing/null:** Defaults: all four toggles set to `true`.
- **App version key missing:** Display "v1.0.0" as fallback; no crash.
- **External URL tapped offline:** Browser handles gracefully; app does not crash.
- **"Linked Devices" tapped:** Shows a coming-soon message or a static list "This device — last active: now".
