## User Story

**As a** hotel guest,
**I want** to upload or take a profile photo,
**So that** the app feels personalised and staff can visually identify me for VIP or returning-guest recognition.

---

## Background / Context

A personalised avatar adds warmth to the Browns Hotels & Resorts digital experience and makes the app feel less generic. The photo is displayed in the app header on every authenticated screen and in the profile header, reinforcing the guest's identity throughout their stay journey. Because Destinity vButler is a client-side app with no server backend in Phase 1, the photo is stored as a Base64 string in localStorage.

---

## Acceptance Criteria

- [ ] Tapping the avatar circle on the Profile screen opens a bottom drawer with three options: "Take Photo", "Choose from Gallery", and "Remove Photo".
- [ ] "Take Photo" triggers an `<input type="file" accept="image/*" capture="user">` to invoke the device's front camera on mobile.
- [ ] "Choose from Gallery" triggers an `<input type="file" accept="image/*">` to open the device's image gallery/file picker.
- [ ] After a file is selected, the app checks the file size before reading; files larger than 3MB trigger an error toast "Image too large. Please choose a photo under 3MB." and no further processing occurs.
- [ ] Valid files are read using `FileReader.readAsDataURL()` and the resulting Base64 string is stored in `vb:profile.avatarBase64`.
- [ ] The avatar circle in the Profile screen header updates immediately to show the new photo without a page reload.
- [ ] The avatar circle in the app header (visible on all authenticated pages) also updates immediately via the Alpine.js store.
- [ ] "Remove Photo" clears `vb:profile.avatarBase64` (sets to `null`), and the avatar reverts to the initials fallback immediately.
- [ ] The bottom drawer closes automatically after any action is taken (including cancellation via backdrop tap or swipe-down).
- [ ] If no photo has been set, the "Remove Photo" option in the drawer is visually disabled (greyed out) but still displayed.
- [ ] The uploaded photo persists across app sessions (page refresh, browser close and reopen).
- [ ] The avatar image is displayed with `object-fit: cover` to prevent distortion regardless of aspect ratio.

## UI / UX Notes

- Bottom drawer: slides up with a 250ms ease-out transition; handle bar at top; semi-transparent dark backdrop closes drawer on tap.
- Drawer options: full-width list items, 56px height each, left-aligned icon + label text (16px).
  - "Take Photo": camera icon (outline style).
  - "Choose from Gallery": image/photo icon.
  - "Remove Photo": trash icon, colour: destructive red; greyed (opacity-40) when no photo set.
- Avatar circle tap target: minimum 80px diameter; subtle scale animation (scale 0.95) on press.
- After upload: brief pulse/shimmer animation on avatar while `FileReader` processes, then photo fades in.
- Error toast: bottom-centre, red/destructive variant, auto-dismiss after 4 seconds.
- Initials fallback: brand background (#003c52), white text, uses first letter of first name + first letter of last name (e.g., "AP" for Anuradha Perera); uppercase.
- Avatar in app header: 36px diameter circle; same fallback logic.

## Technical Notes

- Two separate hidden `<input type="file">` elements in the DOM (one with `capture="user"`, one without); triggered programmatically via `.click()` from the drawer options.
- File size check: `file.size > 3 * 1024 * 1024` before `FileReader` instantiation.
- `FileReader.onload` callback: sets `Alpine.store('profile').avatarBase64 = event.target.result` and calls `StorageService.set('vb:profile', profile)`.
- Alpine store `profile` must be a global store (not local component data) so that both the profile screen and the app header can reactively bind to `$store.profile.avatarBase64`.
- App header avatar: `<img :src="$store.profile.avatarBase64" x-show="$store.profile.avatarBase64">` with an initials fallback `<span x-show="!$store.profile.avatarBase64">`.
- "Remove Photo": sets `$store.profile.avatarBase64 = null`; calls StorageService to persist.
- Consider storing a compressed/resized version: use `<canvas>` to resize to max 400×400px before Base64 encoding to keep localStorage usage reasonable.
- Canvas resize: `drawImage` with calculated aspect-ratio-preserving dimensions; export via `canvas.toDataURL('image/jpeg', 0.8)`.
- Image MIME type: accept JPEG, PNG, WebP, HEIC (browser-decoded). Output always as JPEG via canvas for consistency.

## Test Cases

### Happy Path

1. Navigate to the Profile screen.
2. Tap the avatar circle.
3. Verify the bottom drawer slides up with all three options visible.
4. Tap "Choose from Gallery".
5. Select a valid JPEG image under 3MB.
6. Verify the avatar circle in the Profile header immediately shows the selected photo.
7. Verify the app header avatar also shows the same photo without navigation.
8. Close the Profile screen and re-open — verify photo persists.
9. Refresh the browser — verify photo is still displayed (loaded from localStorage).
10. Tap avatar → "Remove Photo" → verify avatar reverts to initials "AP".

### Edge Cases

- **File exactly 3MB (3,145,728 bytes):** Accepted and processed.
- **File just over 3MB (3,145,729 bytes):** Error toast shown; avatar unchanged.
- **Non-image file selected (e.g., PDF):** Browser file picker should filter it out via `accept="image/*"`; if OS bypass occurs, `FileReader` reads it but canvas `drawImage` will fail — catch error and show "Unsupported file type" toast.
- **Camera permission denied (mobile):** OS-level denial; browser shows its own error; app does not crash; drawer closes gracefully.
- **Very large resolution image (e.g., 12MP RAW):** Canvas resize reduces to 400×400px; stored Base64 is reasonable size (~50–80KB); no localStorage quota error.
- **No name set (initials fallback):** Avatar shows single initial from email local-part or "?" if email also absent.
- **Dark mode:** Drawer background, backdrop, and option text render correctly.
- **"Remove Photo" when no photo set:** Option appears greyed out (opacity-40); tapping it has no effect and drawer closes.
