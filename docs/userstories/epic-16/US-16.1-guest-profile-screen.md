## User Story

**As a** hotel guest,
**I want** to view and edit my personal details, stay preferences, and dietary requirements in a single profile screen,
**So that** Browns Hotels & Resorts staff can personalise my stay without me having to repeat my preferences at every interaction.

---

## Background / Context

Guests staying at Browns Hotels & Resorts properties (e.g., Browns Beach Hotel, Anantara Kalutara) may visit multiple times or have specific recurring needs. The profile screen centralises all guest-controlled data so that preferences flow through to service requests, dining orders, and housekeeping. All data is persisted in `localStorage` under the `vb:profile` key and pre-populated on every screen load.

## Acceptance Criteria

- [ ] The Profile screen is reachable by tapping the avatar circle in the app header on any authenticated page.
- [ ] The Profile screen is reachable via the bottom navigation bar "Profile" tab on authenticated pages.
- [ ] The profile header displays an avatar circle showing either the guest's uploaded photo or their initials (first + last name) on a brand-coloured background when no photo is set.
- [ ] The profile header displays the guest's full name and email address below the avatar.
- [ ] An "Edit Profile" button is visible in the profile header; tapping it enables inline editing of all editable fields.
- [ ] The Personal Information card is collapsible and contains: Full Name (editable text input), Email (read-only display with a "Change Email" link), Phone (editable), Nationality (dropdown), Date of Birth (date picker).
- [ ] The Stay Preferences card is collapsible and contains: Bed Preference as single-select chips (King / Twin / Double), Pillow Type as single-select chips (Soft / Medium / Firm), Room Preferences as a multi-select list (e.g., High Floor, Away from Elevator, Extra Towels, Baby Cot).
- [ ] The Dietary Preferences card is collapsible and contains multi-select chips: Vegetarian, Vegan, Halal, Gluten-Free, Nut Allergy, Dairy-Free.
- [ ] The ID / Passport card is collapsible and contains a passport number text field and a document upload zone (image or PDF).
- [ ] All fields are pre-populated from `vb:profile` in localStorage on screen load.
- [ ] A "Save Changes" button is present at the bottom of the screen and is only active when at least one field has been modified.
- [ ] Tapping "Save Changes" writes the updated object to `vb:profile` via `StorageService` and displays a "Profile updated successfully" success toast.
- [ ] The success toast auto-dismisses after 3 seconds.
- [ ] Validation errors are shown inline (e.g., phone number format, required full name field).
- [ ] Collapsed cards show a summary line of the current values (e.g., "Halal, Gluten-Free" under Dietary Preferences).
- [ ] The screen is scrollable; the "Save Changes" button is sticky at the bottom or always visible via scroll.
- [ ] The screen respects the active theme (light/dark) and RTL layout when Arabic is selected.

## UI / UX Notes

- Avatar circle: 80px diameter in profile header; brand colour (#003c52) background for initials fallback; white initials text, font-size 28px, font-weight 600.
- Profile header background: subtle gradient or brand-tinted card; full name in 18px semibold; email in 14px muted colour.
- "Edit Profile" button: outlined secondary style; positioned top-right of header card.
- Collapsible cards: use a chevron icon (rotates 180° when open); smooth 200ms CSS transition.
- Collapsed summary text: 13px, muted colour, max 2 lines with ellipsis overflow.
- Chips: rounded-full pill style; selected state uses brand fill (#003c52) with white text; unselected is border-only.
- Nationality dropdown: searchable, includes all ISO countries; flag emoji prefix preferred.
- Date of Birth: native `<input type="date">` on mobile; display formatted as "15 March 1990".
- Upload zone: dashed border, cloud-upload icon, "Tap to upload" label; shows file name + remove icon after upload.
- "Save Changes" button: full-width, brand primary style, disabled state when no changes.
- Toast: bottom-centre position, green success variant, with a checkmark icon.

## Technical Notes

- Alpine.js `x-data` component initialised with a `profile` object mirroring the `vb:profile` schema.
- On mount (`x-init`), call `StorageService.get('vb:profile')` and hydrate the Alpine data object.
- Track changes with a `isDirty` computed boolean comparing current form values against the originally loaded snapshot.
- `StorageService.set('vb:profile', profile)` is called on save; ensure deep-clone to avoid reference mutation.
- Nationality dropdown: use a static JSON array of country objects `{ code, name, flag }` loaded from `/data/countries.json`.
- Passport upload: read as Base64 via `FileReader`; store as `vb:profile.passportDocBase64`; display a thumbnail for images.
- Collapsible card state (open/closed) is NOT persisted — defaults to all closed except the first card on first visit.
- All form inputs must emit `@input` or `@change` handlers updating Alpine data to trigger `isDirty` re-evaluation.
- "Change Email" link: navigates to a dedicated change-email modal/screen (out of scope for this story; link is present but can be a placeholder route).
- Error messages: use `x-show` with `x-transition` for inline error spans below each field.

## Test Cases

### Happy Path

1. Log in as guest "Anuradha Perera" (email: anuradha@example.com).
2. Tap the avatar circle in the app header.
3. Verify Profile screen loads with full name "Anuradha Perera", email "anuradha@example.com" displayed in the header.
4. Verify all cards are collapsed by default with summary values shown.
5. Tap "Personal Information" card — verify it expands to show Full Name, Email, Phone, Nationality, Date of Birth fields all pre-filled.
6. Modify the Phone field to "+94 77 123 4567".
7. Verify the "Save Changes" button becomes active (enabled).
8. Tap "Save Changes" — verify success toast "Profile updated successfully" appears.
9. Navigate away and return to Profile — verify Phone field still shows "+94 77 123 4567".
10. Expand "Dietary Preferences" — select "Halal" and "Nut Allergy" chips.
11. Tap "Save Changes" — verify both dietary preferences are persisted in `vb:profile.dietaryPreferences`.

### Edge Cases

- **No changes made:** "Save Changes" button remains disabled; tapping it (if somehow triggered) does nothing.
- **Empty Full Name:** Inline error "Full name is required" shown; Save button not activated.
- **Invalid phone format:** Inline error "Please enter a valid phone number" shown.
- **Profile not yet set:** Screen shows empty fields; initials avatar shows "?" or first letter of email local-part.
- **Passport upload > 3MB:** Error toast "File too large. Maximum 3MB allowed."
- **Slow storage write:** Button shows loading spinner during async write; disabled to prevent double-submit.
- **Dark mode active:** All cards, chips, inputs render correctly with dark backgrounds and sufficient contrast.
- **Arabic RTL:** Labels align right, chips flow right-to-left, save button text is RTL.
- **150% browser zoom:** All cards remain fully visible without horizontal scroll or content cutoff.
