## User Story

**As a** guest who has submitted valid credentials,
**I want** to verify my identity with a one-time code,
**So that** my account is protected even if my password is compromised.

---

## Background / Context

The OTP step adds a meaningful security layer to the authentication flow without requiring a backend email or SMS service in Phase 1. The simulated OTP approach (any 6-digit code accepted, with the demo code logged to the browser console) allows full end-to-end testing and demonstration while making the architecture ready for a real OTP service in Phase 2. The UX of the OTP screen must feel as polished as a banking app — individual digit boxes, auto-advance, paste support, and clear countdown timer are all standard expectations.

---

## Acceptance Criteria

- [ ] The OTP screen is only accessible after a successful credential submission on the Login screen; direct navigation to `#/otp` without `vb:pendingAuth` in sessionStorage redirects to `#/login`
- [ ] The screen displays 6 individual single-digit `<input>` boxes arranged in a horizontal row
- [ ] Each input accepts only a single digit (0–9); non-numeric keypresses are ignored
- [ ] After entering a digit, focus automatically advances to the next input box
- [ ] Pressing Backspace on an empty input moves focus to the previous input box and clears it
- [ ] Pasting a 6-digit numeric string fills all 6 boxes and triggers verification
- [ ] A countdown timer displays "Expires in M:SS" format, counting down from 10:00 (10 minutes); implemented with Alpine.js `setInterval`
- [ ] "Resend Code" button is disabled and greyed out while the countdown is active; it becomes enabled only when the timer reaches 0:00
- [ ] Clicking "Resend Code" when enabled resets the timer to 10:00, clears all input boxes, resets error count, and logs the demo OTP to console
- [ ] **Phase 1 behaviour:** Any valid 6-digit code is accepted as correct; on screen load, the demo OTP `123456` is logged to the browser console with the message `"⚠️ DEMO: OTP is 123456"`
- [ ] On correct OTP entry: `AuthService.createSession(guest, rememberMe)` is called, `vb:pendingAuth` is cleared from sessionStorage, and the router navigates to `#/dashboard`
- [ ] On incorrect OTP (Phase 1: this applies if a non-6-digit code is submitted, though primary flow accepts any 6 digits): apply a shake CSS animation to the input row and increment an error counter
- [ ] After 3 failed attempts: all input boxes are disabled, a lockout message displays "Too many attempts. Please wait 5 minutes.", a 5-minute lockout countdown is shown; `Alpine.js` manages this state client-side
- [ ] "Back to login" link navigates to `#/login` and clears `vb:pendingAuth` from sessionStorage
- [ ] The screen displays the guest's partially masked email address (e.g., `j***@example.com`) to confirm which account the OTP was sent to

## UI / UX Notes

- **Layout:** Centred card (max-width 400px); full-screen on mobile; padded from edges
- **Heading:** "Verify your identity" or "Enter your code" in Playfair Display; subtitle: "We sent a 6-digit code to [masked email]"
- **OTP input boxes:** Each box is a square (~52×56px on mobile, 56×60px on desktop), `border-2 border-gray-300 dark:border-gray-600 rounded-lg text-center text-2xl font-bold`; focused box border turns brand colour; filled box has light brand tint background
- **Shake animation:** On incorrect OTP, the 6-box row animates with a CSS keyframe horizontal shake lasting ~400ms; boxes briefly flash `border-red-500`
- **Lockout state:** All boxes `opacity-50 cursor-not-allowed`; error message in `text-red-600`; lockout timer shown as `text-sm text-gray-500`
- **Timer:** Right-aligned or centred below inputs; "Expires in 9:45" in `text-sm text-gray-500`; turns `text-red-500` when under 1 minute remaining
- **Resend button:** Text-only button style (`text-brand underline`) when enabled; `text-gray-400 cursor-not-allowed` when disabled

## Technical Notes

- **File:** `js/pages/otp.js`
- **Alpine.js component data:**
  ```js
  {
    digits: ['','','','','',''],
    countdown: 600, // seconds
    timer: null,
    attempts: 0,
    locked: false,
    lockoutTimer: null,
    lockoutRemaining: 300,
    get formattedTime() {
      const m = Math.floor(this.countdown / 60);
      const s = this.countdown % 60;
      return `${m}:${s.toString().padStart(2,'0')}`;
    },
    init() {
      if (!sessionStorage.getItem('vb:pendingAuth')) Router.navigate('#/login');
      console.warn('⚠️ DEMO: OTP is 123456');
      this.startTimer();
    },
    startTimer() {
      this.timer = setInterval(() => {
        if (this.countdown > 0) this.countdown--;
        else clearInterval(this.timer);
      }, 1000);
    },
    // ... handleInput, handleBackspace, handlePaste, verify, resend
  }
  ```
- **Focus management:** Use `this.$refs['digit-0']` through `this.$refs['digit-5']`; call `.focus()` programmatically via `$nextTick`
- **Paste handler:** `@paste.prevent` on first input; extract first 6 numeric chars from clipboard data; distribute to `digits` array; call `verify()`
- **Verification (Phase 1):** `const otp = this.digits.join(''); if (otp.length === 6) { this.completeLogin(); } else { this.handleError(); }`
- **completeLogin:** Read `vb:pendingAuth` from sessionStorage; call `AuthService.createSession(guest, rememberMe)`; `sessionStorage.removeItem('vb:pendingAuth')`; `Router.navigate('#/dashboard')`
- **Lockout:** After 3 failed attempts, start a 5-minute (`300s`) countdown using `setInterval`; when reaches 0, reset attempts, unlock inputs, restart OTP timer

## Test Cases

### Happy Path

1. Complete login on `#/login` → arrive at `#/otp`; masked email shown; console logs `"⚠️ DEMO: OTP is 123456"`; timer starts at 10:00
2. Type `1` in first box → focus moves to second box; type `2` → third; continue to fill all 6 → verification triggers; navigate to `#/dashboard`
3. Paste `123456` into the first box → all 6 boxes fill; verification triggers; navigate to `#/dashboard`
4. Wait for timer to expire → "Resend Code" button becomes enabled; click → timer resets to 10:00; boxes cleared; new demo OTP logged

### Edge Cases

- Navigate directly to `#/otp` without completing login → redirected to `#/login`
- Enter `abc123` via keyboard → only digits `1`, `2`, `3` accepted; letters silently rejected
- Enter an incorrect code (simulate by modifying Phase 1 logic temporarily) → shake animation plays; attempt counter increments
- 3rd failed attempt → lockout state activates; all boxes disabled; 5-minute countdown begins
- During lockout, attempt to type in boxes → input ignored; lockout message visible
- Click "Back to login" → `#/login` route; `vb:pendingAuth` cleared from sessionStorage
- Paste a 7-digit string → only first 6 digits used; 7th ignored
- Paste a non-numeric string → no boxes filled; no error thrown
- Tab key in input boxes → should move focus forward (standard browser behaviour); Shift+Tab moves backward
