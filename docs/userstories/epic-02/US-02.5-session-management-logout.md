## User Story

**As a** guest,
**I want** the app to remember my login and automatically sign me out when my session expires,
**So that** I don't have to log in every time I open the app during my stay, and my account stays secure after I leave.

---

## Background / Context

Session management ties the entire authentication flow together. Without it, guests would need to log in every time they open the app — unacceptable for a hotel stay companion. Equally, a session that never expires leaves the account accessible to anyone who picks up a forgotten device after checkout. The two-tier persistence model ("Remember me" for longer stays vs. same-day sessions) balances convenience and security appropriately for the hospitality context.

## Acceptance Criteria

### Session Creation
- [ ] `AuthService.createSession(guest, rememberMe)` is called after successful OTP verification
- [ ] When `rememberMe` is `true`: `vb:session.expiresAt` = `Date.now() + 30 * 24 * 60 * 60 * 1000` (30 days from now)
- [ ] When `rememberMe` is `false`: `vb:session.expiresAt` = `Date.now() + 24 * 60 * 60 * 1000` (24 hours from now)
- [ ] Session object written to `localStorage` under key `vb:session`:
  ```json
  {
    "guestId": "guest-001",
    "email": "jane.doe@example.com",
    "fullName": "Jane Doe",
    "expiresAt": 1234567890000,
    "createdAt": 1234567890000
  }
  ```
- [ ] After session creation, the Alpine.js auth store is hydrated: `Alpine.store('auth').isLoggedIn = true`, `Alpine.store('auth').guest = { ...guestData }`

### Session Validation
- [ ] `AuthService.checkSession()` is called on every page load (in `js/app.js` initialisation)
- [ ] `checkSession()` reads `vb:session` from localStorage; if absent → returns `null`
- [ ] If `vb:session.expiresAt` < `Date.now()` (session expired) → clears `vb:session` from localStorage, resets Alpine auth store, returns `null`
- [ ] If session is valid → hydrates Alpine auth store with guest data, returns the guest object
- [ ] The router uses `AuthService.checkSession()` return value to enforce route guards (null → redirect to `#/login`)

### Logout
- [ ] A "Log Out" option is accessible from the Settings/Profile screen
- [ ] Clicking "Log Out" opens a confirmation modal (using the shared Modal component from US-01.7) with title "Log out?" and body "You'll need to sign in again to access your reservations."
- [ ] Modal footer has two buttons: "Cancel" (closes modal, no action) and "Log Out" (outlined red or `bg-red-600 text-white`)
- [ ] On confirming "Log Out": `AuthService.logout()` is called
- [ ] `AuthService.logout()` performs: `localStorage.removeItem('vb:session')`, reset Alpine auth store to `{ isLoggedIn: false, guest: null }`, `Router.navigate('#/login')`
- [ ] After logout, the Back button cannot navigate back to authenticated pages (hash history should not allow re-entry)
- [ ] Session cleared after logout: `vb:session` key does not exist in localStorage

## UI / UX Notes

- **Logout confirmation modal:** Use the shared Modal component. On mobile this is a bottom drawer; on desktop a centred dialog. The "Log Out" button should be styled to convey a destructive action — `text-red-600` or `bg-red-600 text-white`; "Cancel" is the secondary/neutral action (displayed first to reduce accidental logout)
- **Session expiry redirect:** If the session expires mid-session (e.g., user leaves the app open for 24+ hours without "Remember me"), the next navigation or route guard check redirects to `#/login` with a toast "Your session has expired. Please sign in again." in the `info` variant
- **No session indicator in UI:** Do not show a "session expires in X minutes" countdown anywhere in the UI — this is an implementation detail. Expiry should be silent until it occurs

## Technical Notes

- **File:** `js/core/auth.service.js`
- **Full AuthService implementation:**
  ```js
  const AuthService = {
    async login(email, password) {
      const guests = await DataService.fetch('/data/mock/guests.json');
      const guest = guests.find(g => g.email === email.toLowerCase());
      if (!guest) return { success: false };
      const hash = await hashPassword(password);
      if (hash !== guest.passwordHash) return { success: false };
      return { success: true, guest };
    },

    createSession(guest, rememberMe = false) {
      const duration = rememberMe ? 30 * 24 * 60 * 60 * 1000 : 24 * 60 * 60 * 1000;
      const session = {
        guestId: guest.id,
        email: guest.email,
        fullName: guest.fullName,
        expiresAt: Date.now() + duration,
        createdAt: Date.now()
      };
      localStorage.setItem('vb:session', JSON.stringify(session));
      this._hydrateStore(guest);
    },

    checkSession() {
      const raw = localStorage.getItem('vb:session');
      if (!raw) return null;
      const session = JSON.parse(raw);
      if (session.expiresAt < Date.now()) {
        this.clearSession();
        return null;
      }
      return session;
    },

    logout() {
      this.clearSession();
      Router.navigate('#/login');
    },

    clearSession() {
      localStorage.removeItem('vb:session');
      if (window.Alpine) {
        Alpine.store('auth').isLoggedIn = false;
        Alpine.store('auth').guest = null;
      }
    },

    _hydrateStore(guest) {
      if (window.Alpine) {
        Alpine.store('auth').isLoggedIn = true;
        Alpine.store('auth').guest = guest;
      }
    }
  };
  ```
- **Alpine auth store initial state (in `js/app.js`):**
  ```js
  Alpine.store('auth', {
    isLoggedIn: false,
    guest: null
  });
  ```
- **Initialisation sequence in `js/app.js`:**
  1. `ThemeService.init()` (must run before DOM paint — see US-01.5)
  2. `Alpine.store('auth', { isLoggedIn: false, guest: null })` registration
  3. `Alpine.start()`
  4. `AuthService.checkSession()` — if valid, hydrate store; else ensure redirect on first protected route
  5. `Router.init()` — reads current hash, runs route guard, renders page
- **Back button after logout:** The router's route guard (`AuthService.checkSession()`) is called on every `hashchange` event. After logout, all protected routes redirect to `#/login`, so even if the user presses Back into a previously cached hash, they land on `#/login`
- **Session expiry toast:** In `Router.navigate()`, when `checkSession()` returns null on a protected route and a previous session existed in this page load, show `ToastService.show('Your session has expired. Please sign in again.', 'info')`

## Test Cases

### Happy Path

1. Login with "Remember me" checked → `vb:session.expiresAt` is ~30 days from now (verify in DevTools → Application → localStorage)
2. Login without "Remember me" → `vb:session.expiresAt` is ~24 hours from now
3. Close and reopen browser (or refresh) with valid session → app loads directly to `#/dashboard`; auth store hydrated with guest data
4. Navigate to Settings → tap "Log Out" → confirmation modal appears
5. Tap "Log Out" in modal → `vb:session` removed from localStorage; redirected to `#/login`; pressing Back does not return to authenticated pages

### Edge Cases

- Manually set `vb:session.expiresAt` to a past timestamp in DevTools → refresh page → redirected to `#/login`; toast "Your session has expired"
- Tap "Cancel" in logout modal → modal closes; session intact; user remains on current page
- Two tabs open: logout in Tab 1 → Tab 2 still shows authenticated content (localStorage change events not cross-tab in Phase 1; document acceptable behaviour)
- `vb:session` in localStorage is malformed JSON (e.g., corrupted) → `checkSession()` wraps `JSON.parse` in try/catch; returns null; redirects to login; no crash
- `rememberMe` flag not passed to `createSession` (undefined) → defaults to `false`; 24-hour session
