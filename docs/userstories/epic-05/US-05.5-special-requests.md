## User Story

**As a** hotel guest completing my pre-arrival checklist,
**I want** to specify my dietary preferences, room preferences, bed type, occasion, and any additional requests,
**So that** Browns Hotels & Resorts can personalise my room setup and dining experience before I arrive.

---

## Background / Context

Personalisation is central to the Browns Hotels & Resorts luxury proposition. By capturing guest preferences in advance, the property team can prepare the room to exact specifications — ensuring the correct bed configuration, noting allergy information for dining, and arranging surprise touches for special occasions such as honeymoons or anniversaries. Saved dietary preferences are synced back to the guest's global profile so they do not need to re-enter them on future stays.

## Acceptance Criteria

- [ ] The Special Requests form is rendered within its expandable accordion card (US-05.1)
- [ ] **Dietary Preferences** section: multi-select chip group with options: Vegetarian, Vegan, Halal, Gluten-Free, Nut Allergy, Dairy-Free, Other
- [ ] Multiple dietary chips can be selected simultaneously
- [ ] **Room Preferences** section: multi-select chip group with options: Ground Floor, High Floor, Away from Elevator, Quiet Room, Connecting Room
- [ ] Multiple room preference chips can be selected simultaneously
- [ ] **Bed Preferences** section: single-select (radio) chip group with options: King, Twin, Double
- [ ] Only one bed preference can be selected at a time; selecting a new one deselects the previous
- [ ] **Occasion** section: single-select (radio) chip group with options: Honeymoon, Anniversary, Birthday, None
- [ ] Only one occasion can be selected; "None" is pre-selected by default
- [ ] **Additional Requests** section: a textarea with placeholder "Anything else we should know?" and a 500-character limit with character counter
- [ ] A "Save" button is present at the bottom of the form
- [ ] On tapping "Save":
  - All selections and text are stored in `vb:preArrival:{resvId}.specialRequests`
  - Dietary preferences are also synced to `vb:profile.dietaryPreferences` (merge, not overwrite)
  - The item is marked complete (`complete: true`) regardless of whether any fields were filled
  - A brief inline summary is stored (e.g., "Vegetarian, Vegan · King · Anniversary")
- [ ] Previously saved values are restored when the accordion is reopened

## Technical Notes

- **Alpine.js component**: `x-data="specialRequestsForm()"` embedded in the US-05.1 accordion card
- **Data model**:
  ```js
  {
    dietaryPreferences: [],   // Array of strings
    roomPreferences: [],      // Array of strings
    bedPreference: null,      // String or null
    occasion: 'None',         // String, default 'None'
    additionalRequests: ''    // String
  }
  ```
- **Multi-select toggle**:
  ```js
  toggleDietary(pref) {
    const i = this.dietaryPreferences.indexOf(pref);
    if (i > -1) this.dietaryPreferences.splice(i, 1);
    else this.dietaryPreferences.push(pref);
  }
  ```
- **Pre-fill on open**: load from `vb:preArrival:{resvId}.specialRequests`; if not found, try `vb:profile.dietaryPreferences` to pre-populate dietary chips
- **Save**:
  ```js
  // Save to pre-arrival
  const preArrival = StorageService.get(`vb:preArrival:${resvId}`) || {};
  preArrival.specialRequests = { complete: true, ...this.formData, summary: this.buildSummary() };
  StorageService.set(`vb:preArrival:${resvId}`, preArrival);
  // Sync dietary preferences to profile (merge)
  const profile = StorageService.get('vb:profile') || {};
  const merged = [...new Set([...(profile.dietaryPreferences || []), ...this.dietaryPreferences])];
  profile.dietaryPreferences = merged;
  StorageService.set('vb:profile', profile);
  this.$dispatch('preArrival:itemComplete', { item: 'specialRequests' });
  ```
- **Summary builder**:
  ```js
  buildSummary() {
    const parts = [];
    if (this.dietaryPreferences.length) parts.push(this.dietaryPreferences.slice(0,2).join(', '));
    if (this.bedPreference) parts.push(this.bedPreference);
    if (this.occasion && this.occasion !== 'None') parts.push(this.occasion);
    return parts.length ? parts.join(' · ') : 'No special requests';
  }
  ```

## Test Cases

### Happy Path

1. Open the Special Requests accordion card
2. Select "Vegetarian" and "Vegan" dietary chips; verify both appear selected
3. Select "High Floor" room preference
4. Select "King" bed preference; then select "Twin"; verify only "Twin" is selected
5. Select "Anniversary" occasion; verify "None" is deselected
6. Enter "Please prepare a room with extra pillows" in additional requests
7. Tap "Save"; verify card collapses with summary "Vegetarian, Vegan · Twin · Anniversary"
8. Verify `vb:profile.dietaryPreferences` now includes Vegetarian and Vegan
9. Reopen the accordion; verify all previous selections restored

### Edge Cases

- Save with no fields selected or filled: item marks complete; summary reads "No special requests"
- Guest profile already has "Halal" in dietary preferences; guest selects "Vegetarian": merged result is ["Halal", "Vegetarian"] in profile — no duplicates, no overwrites of existing
- Additional requests field reaches 500 characters: counter turns red at 480+; no further input accepted
- Guest selects all 7 dietary chips: all 7 saved; summary shows first 2 with implied truncation
- Guest reopens and deselects "Anniversary", re-selects "None": saved correctly; summary updates
