## User Story

**As a** hotel guest completing my pre-arrival checklist,
**I want** to indicate my estimated arrival time and leave any relevant notes for the hotel team,
**So that** Browns Hotels & Resorts can ensure my room and a warm welcome are ready when I arrive.

---

## Background / Context

Knowing when a guest is expected to arrive allows the property team to prioritise room preparation, arrange transport pick-ups, and have welcome amenities in place. This is particularly important for boutique luxury properties where personalised, anticipatory service is a key differentiator. The 30-minute increment time picker is a deliberate UX choice — it simplifies selection while providing sufficient precision for arrival planning.

## Acceptance Criteria

- [ ] The Estimated Arrival Time form is rendered within its expandable accordion card (US-05.1)
- [ ] A Flatpickr time picker is displayed with 30-minute increment steps (e.g., 13:00, 13:30, 14:00)
- [ ] The time picker defaults to 14:00 (2:00 PM) when no value has been previously saved
- [ ] The time picker uses 12-hour format with AM/PM indicator for display, but stores time in 24-hour `HH:MM` format
- [ ] An optional note field is displayed below the time picker with placeholder text: "Any notes for the team? e.g., 'Arriving by train from Colombo'"
- [ ] The note field has a character limit of 300 characters; a character counter is shown: "X / 300"
- [ ] A "Save" button is present at the bottom of the form
- [ ] On tapping "Save": the selected time and note are stored in `vb:preArrival:{resvId}.arrivalTime.time` and `.note`; the item is marked complete (`complete: true`)
- [ ] The inline summary in the collapsed completed state displays the saved time (e.g., "2:00 PM · Arriving by train from Colombo" or "2:00 PM" if no note)
- [ ] The item can be saved with no note (note is optional); the item still marks complete
- [ ] Saving without having changed the time (using the 14:00 default) is valid; item marks complete
- [ ] Previously saved values are restored when the accordion is reopened for editing

## UI / UX Notes

- **Time picker**: Flatpickr rendered in "time mode" (`enableTime: true, noCalendar: true`); `minuteIncrement: 30`; styled to match app input style — `border: 1.5px solid var(--color-border)`, `border-radius: 8px`, `padding: 10px 14px`, `font-size: 0.9375rem`, `width: 100%`
- **Time picker label**: "Estimated Arrival Time" in `font-size: 0.8125rem`, `font-weight: 600`, `margin-bottom: 6px`
- **Note field label**: "Notes for the team" in `font-size: 0.8125rem`, `font-weight: 600`, `margin-top: 16px`; note: "(optional)" appended in `color: var(--color-text-tertiary)`
- **Textarea**: `height: 80px`, `resize: none`, same border/radius style as other inputs; placeholder text in `color: var(--color-text-placeholder)`
- **Character counter**: right-aligned below textarea, `font-size: 0.75rem`, `color: var(--color-text-tertiary)`; turns `color: var(--color-error)` at 290+ characters
- **"Save" button**: full-width brand primary, `border-radius: 8px`, `padding: 12px`, `font-weight: 600`
- **Inline summary** (collapsed completed): formatted time (e.g., "2:00 PM"), then bullet separator "·", then note (truncated to 60 chars with ellipsis); if no note, just the time
- **Visual clarity**: a small clock icon precedes the time picker label to reinforce the purpose of the field

## Technical Notes

- **Flatpickr initialisation**:
  ```js
  flatpickr('#arrival-time-picker', {
    enableTime: true,
    noCalendar: true,
    time_24hr: false,
    minuteIncrement: 30,
    defaultDate: this.savedTime || '14:00',
    onChange: (selectedDates, dateStr) => {
      // Store in 24hr format
      this.arrivalTime = dayjs(`2000-01-01 ${dateStr}`, 'h:mm A').format('HH:mm');
    }
  });
  ```
- **Alpine.js component**: `x-data="arrivalTimeForm()"` embedded in the US-05.1 accordion card; properties: `arrivalTime` (24h string), `note` (string), `noteMaxLength: 300`
- **Pre-fill on open**:
  ```js
  const saved = StorageService.get(`vb:preArrival:${resvId}`)?.arrivalTime || {};
  this.arrivalTime = saved.time || '14:00';
  this.note = saved.note || '';
  ```
- **Save**:
  ```js
  const preArrival = StorageService.get(`vb:preArrival:${resvId}`) || {};
  preArrival.arrivalTime = {
    complete: true,
    time: this.arrivalTime,         // "14:00"
    note: this.note,
    summary: this.buildSummary()    // "2:00 PM · Arriving by train"
  };
  StorageService.set(`vb:preArrival:${resvId}`, preArrival);
  this.$dispatch('preArrival:itemComplete', { item: 'arrivalTime' });
  ```
- **Summary builder**:
  ```js
  buildSummary() {
    const timeDisplay = dayjs(`2000-01-01 ${this.arrivalTime}`).format('h:mm A');
    return this.note ? `${timeDisplay} · ${this.note.slice(0, 60)}` : timeDisplay;
  }
  ```
- **Flatpickr theming**: apply custom CSS class `fp-vbutler` for dark mode compatibility; override Flatpickr's default styles in the app's theme CSS

## Test Cases

### Happy Path

1. Open the Estimated Arrival Time accordion card
2. Verify time picker defaults to 2:00 PM
3. Verify 30-minute increments only available (no 14:15, 14:45 options)
4. Change time to 4:00 PM; enter note "Arriving by taxi from the airport"
5. Tap "Save"; verify card collapses and shows "4:00 PM · Arriving by taxi from the airport"
6. Reopen the accordion; verify time shows 4:00 PM and note is pre-filled
7. Resave with no note change; item remains complete
8. Save with no note; verify inline summary shows only "4:00 PM"

### Edge Cases

- Guest saves without modifying the default 14:00 time: item marks complete with "2:00 PM"
- Note reaches 300 characters: character counter turns red at 290+; no additional characters accepted at 300
- Guest types 301 characters (paste): input truncated to 300; error indicator shown
- Guest returns to this item after previously completing: all saved values restored; re-saving with new time updates storage
- Flatpickr time input cleared (no time selected): default 14:00 applied on save as fallback
