## User Story

**As a** hotel guest completing my pre-arrival checklist,
**I want** my progress to be automatically saved as I complete each item and fully restored when I return to the checklist later,
**So that** I can complete the checklist across multiple sessions without losing any of my submitted information.

---

## Background / Context

Guests may begin the pre-arrival checklist on one occasion and return to complete it later — for example, they might confirm personal details on one day and upload their passport the next day after locating the document. Reliable persistence via localStorage ensures no effort is lost between sessions. The "all complete" celebration state and corresponding notification provide a satisfying sense of closure and confirm to the guest that the hotel has received their pre-arrival information.

---

## Acceptance Criteria

- [ ] Every individual item save/completion writes to `localStorage` under the key `vb:preArrival:{resvId}` immediately upon the guest tapping "Save" or "Confirm" within each sub-form
- [ ] On returning to the checklist screen, all previously completed items are restored to their completed state with inline summaries visible
- [ ] On returning to the checklist screen, all previously saved but not-yet-completed form data is also restored (e.g., a partially entered Personal Details form that was not saved)
- [ ] A "Save Progress" button is displayed at the bottom of the checklist screen (below all accordion cards)
- [ ] Tapping "Save Progress" explicitly triggers a save of any currently open and edited accordion card's data, even if the sub-form "Save" button has not been tapped
- [ ] "Save Progress" displays a brief success indicator ("Progress saved") for 2 seconds after tapping
- [ ] When all mandatory items are complete (Personal Details, ID Upload, Terms & Conditions), the "all set" success state is displayed at the top of the checklist (above the progress bar)
- [ ] The "all set" success state message: "You're all set! Our team is ready to welcome you." with a green check circle icon
- [ ] A CSS confetti animation plays once when the checklist reaches 100% completion for the first time
- [ ] The confetti animation does not replay on subsequent visits to a fully-completed checklist
- [ ] An in-app notification is created when all mandatory items are complete: "Pre-arrival checklist complete for [Property Name]" with a link to `#/pre-arrival/:resvId`
- [ ] The notification is created only once — not duplicated on subsequent visits to a completed checklist

## UI / UX Notes

- **"Save Progress" button**: outlined style (not filled) — `border: 1.5px solid var(--color-brand-primary)`, `color: var(--color-brand-primary)`, `border-radius: 8px`, `padding: 12px`, full-width, `margin-top: 24px`; icon: floppy disk or cloud-upload 16px left of label
- **"Progress saved" indicator**: appears as an inline success message below the button (not a toast) — green check icon + "Progress saved", `font-size: 0.8125rem`, `color: var(--color-success)`, fades out after 2 seconds via CSS `opacity` transition
- **"You're all set!" success block**: displayed in a `var(--color-success)` tinted card (`background: rgba(success, 0.10)`, `border: 1.5px solid var(--color-success)`, `border-radius: 12px`, `padding: 16px 20px`) at the top of the checklist content area, just above the progress bar; large green check circle icon (40px) left of the text column
- **CSS confetti animation**: a lightweight, pure CSS / JavaScript canvas confetti burst (e.g., using `canvas-confetti` library); fires once over the entire viewport for 2.5 seconds; particles in brand colour palette (#D4A853 gold, #1A3B5D navy, #FFFFFF white); does not block user interaction
- **Confetti one-time flag**: `vb:preArrival:{resvId}.confettiFired = true` stored in localStorage; animation only fires if this flag is not yet set
- **Progress bar at 100%**: bar fills completely to brand colour; percentage label shows "100%"; "X of 6 complete" shows "6 of 6 complete"

## Technical Notes

- **Auto-save on item complete**: each sub-form (US-05.2 through US-05.6) dispatches `preArrival:itemComplete` event; the parent `preArrivalChecklist()` component listens and:
  1. Reads the current `vb:preArrival:{resvId}` object
  2. Updates the item's `complete: true` entry
  3. Writes the full object back to `StorageService`
  4. Recomputes `completedCount` and `progressPercent`
  5. Checks if all mandatory items are now complete → triggers success state + confetti if first time

- **"Save Progress" explicit save**: on click, iterate through all open/edited accordion item components and call their internal `save()` method via `$dispatch('preArrival:forceSave')` broadcast; each sub-form listens for this event and writes its current state to storage even if incomplete

- **Completion check**:
  ```js
  checkAllComplete() {
    const data = StorageService.get(`vb:preArrival:${this.resvId}`) || {};
    const mandatoryComplete = ['personalDetails', 'idDocument', 'tcsAccepted'].every(
      key => data[key]?.complete === true
    );
    if (mandatoryComplete && !this.allSetShown) {
      this.allSetShown = true;
      if (!data.confettiFired) {
        this.fireConfetti();
        data.confettiFired = true;
        StorageService.set(`vb:preArrival:${this.resvId}`, data);
      }
      this.createCompletionNotification();
    }
  }
  ```

- **Completion notification (one-time)**:
  ```js
  createCompletionNotification() {
    const data = StorageService.get(`vb:preArrival:${this.resvId}`);
    if (data.notificationSent) return;
    NotificationService.create({
      type: 'pre_arrival_complete',
      title: `Pre-arrival checklist complete for ${this.propertyName}`,
      link: `#/pre-arrival/${this.resvId}`
    });
    data.notificationSent = true;
    StorageService.set(`vb:preArrival:${this.resvId}`, data);
  }
  ```

- **Restore on load**: in `init()`, call `StorageService.get('vb:preArrival:{resvId}')` and pass the saved state to each sub-form's `restoreState(data)` method; if `data.confettiFired`, set `allSetShown = true` without re-firing confetti

- **Library**: `canvas-confetti` (~3KB gzipped); loaded only on pre-arrival page to keep bundle size minimal

## Test Cases

### Happy Path

1. Start pre-arrival checklist; complete Personal Details; close app
2. Reopen app and navigate to checklist; verify Personal Details shows completed state with saved summary
3. Complete ID Upload and Terms & Conditions (all mandatory items)
4. Verify "You're all set!" success block appears at top of checklist
5. Verify CSS confetti animation fires once
6. Verify in-app notification created: "Pre-arrival checklist complete for [Property]"
7. Navigate away and return to checklist; verify success block still shown, confetti does NOT fire again, no duplicate notification

### Edge Cases

- Guest partially fills Personal Details (e.g., name + email only), taps "Save Progress": partial data saved; item remains incomplete but data persists for next session
- Guest taps "Save Progress" with no accordion open: no errors; "Progress saved" indicator still shows
- All optional items incomplete but all 3 mandatory items complete: "You're all set!" shown — only mandatory items gate the success state
- Guest completes checklist on a different device (data not in localStorage): checklist starts fresh; success state and confetti fire normally for that device
- `NotificationService.create()` called twice (e.g., due to rapid tapping): second call silently skipped because `notificationSent` flag is already set
