## User Story

**As a** hotel guest completing my pre-arrival checklist,
**I want** to upload a photo or scan of my ID or passport directly in the app,
**So that** Browns Hotels & Resorts can verify my identity before arrival and expedite the check-in process on the day.

---

## Background / Context

Identity verification is a legal and operational requirement for hotel check-in in Sri Lanka. By allowing guests to submit their ID or passport document digitally through the app before arrival, Browns Hotels & Resorts can process this verification in advance, reducing wait times at the front desk and providing a premium, frictionless arrival experience. The upload is stored locally for this prototype and would integrate with a backend document service in production.

## Acceptance Criteria

- [ ] The ID / Passport Upload form is rendered within its expandable accordion card (US-05.1)
- [ ] The upload zone is displayed with a dashed border, a document upload icon, and the label "Upload ID or Passport"
- [ ] A supporting instruction line beneath the label reads: "Accepted formats: JPG, PNG, PDF — Max size 5 MB"
- [ ] Tapping the upload zone opens the device file picker with `accept="image/jpeg,image/png,application/pdf"`
- [ ] Files larger than 5 MB are rejected immediately after selection; an error message is shown: "File is too large. Maximum size is 5 MB."
- [ ] Files of an invalid type (not JPG, PNG, or PDF) are rejected; error message: "Invalid file type. Please upload a JPG, PNG, or PDF."
- [ ] Valid files are converted to Base64 using `FileReader.readAsDataURL()` and stored in `vb:preArrival:{resvId}.idDocument.base64`
- [ ] The file name and file type are also stored: `vb:preArrival:{resvId}.idDocument.fileName` and `.fileType`
- [ ] After a successful upload:
  - Image files (JPG/PNG): a thumbnail preview of the uploaded image is displayed within the upload zone
  - PDF files: a PDF document icon with the file name label is displayed
- [ ] A "Remove" button is shown after a successful upload, allowing the guest to remove the file and upload a different one
- [ ] Tapping "Remove" clears the stored Base64 data and reverts the zone to the empty upload state
- [ ] Once a file is successfully uploaded, the checklist item is automatically marked complete
- [ ] Marking complete writes `vb:preArrival:{resvId}.idDocument.complete = true` and the inline summary displays the file name

## Technical Notes

- **Hidden file input**: the visible upload zone is a styled `<div>` that triggers a hidden `<input type="file" accept="image/jpeg,image/png,application/pdf">`; `@click="$refs.fileInput.click()"`
- **File validation**:
  ```js
  const MAX_SIZE = 5 * 1024 * 1024; // 5 MB
  const ALLOWED_TYPES = ['image/jpeg', 'image/png', 'application/pdf'];
  if (file.size > MAX_SIZE) { this.error = 'File is too large. Maximum size is 5 MB.'; return; }
  if (!ALLOWED_TYPES.includes(file.type)) { this.error = 'Invalid file type. Please upload a JPG, PNG, or PDF.'; return; }
  ```
- **Base64 conversion**:
  ```js
  const reader = new FileReader();
  reader.onload = (e) => {
    this.base64 = e.target.result;
    this.fileName = file.name;
    this.fileType = file.type;
    this.saveToStorage();
  };
  reader.readAsDataURL(file);
  ```
- **Storage**:
  ```js
  const preArrival = StorageService.get(`vb:preArrival:${resvId}`) || {};
  preArrival.idDocument = {
    complete: true,
    base64: this.base64,
    fileName: this.fileName,
    fileType: this.fileType
  };
  StorageService.set(`vb:preArrival:${resvId}`, preArrival);
  this.$dispatch('preArrival:itemComplete', { item: 'idDocument' });
  ```
- **localStorage size**: Base64 encoding of a 5MB file = ~6.7MB. Note to developers: this approaches the localStorage quota on some browsers. Add a try/catch around `localStorage.setItem` and display a user-friendly error if quota is exceeded: "Unable to save file. Please try a smaller file."
- **Remove**:
  ```js
  this.base64 = null; this.fileName = null; this.fileType = null;
  preArrival.idDocument = { complete: false };
  StorageService.set(`vb:preArrival:${resvId}`, preArrival);
  this.$dispatch('preArrival:itemIncomplete', { item: 'idDocument' });
  ```
- **Alpine.js component**: `x-data="idUploadForm()"` embedded in the US-05.1 accordion card

## Test Cases

### Happy Path

1. Open the ID / Passport Upload accordion card on the pre-arrival checklist
2. Verify the upload zone shows with dashed border, icon, label, and instruction text
3. Tap the upload zone; select a valid JPG file under 5 MB
4. Verify image thumbnail preview renders within the zone
5. Verify file name shown below thumbnail
6. Verify "Remove file" link appears
7. Verify checklist item marks complete (green checkmark, green border)
8. Tap "Remove file"; verify zone reverts to empty upload state; item reverts to incomplete
9. Upload a PDF file; verify PDF icon and file name shown
10. Reload checklist page; verify uploaded file persists and item remains complete

### Edge Cases

- File over 5 MB selected: error message "File is too large. Maximum size is 5 MB."; no storage write; zone remains empty
- File of type `.docx` (unsupported): error message "Invalid file type. Please upload a JPG, PNG, or PDF."
- localStorage quota exceeded during save: user-friendly error message displayed; no silent failure
- Guest uploads file, removes it, uploads a different file: second upload overwrites first in storage; only one file stored at a time
- Very long file name: truncated in display but full name stored in `vb:preArrival`
- Guest returns to checklist after upload: file persists; item shows complete; preview renders from stored Base64
