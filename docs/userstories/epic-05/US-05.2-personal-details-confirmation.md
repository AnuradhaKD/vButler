## User Story

**As a** hotel guest completing my pre-arrival checklist,
**I want** to review and confirm my personal details,
**So that** the Browns Hotels & Resorts team has accurate guest information before my arrival and I do not need to re-enter this data at the front desk.

---

## Background / Context

Personal details are a mandatory component of the pre-arrival checklist and serve as the foundation for guest identity verification and personalised service. The form pre-populates from the guest's existing profile stored in `vb:profile`, minimising input effort while allowing corrections. Saved updates write back to the profile so that the guest's information stays consistent across the app. Auto-completion for returning guests who already have all fields populated removes the burden of re-confirming unchanged data.

## Acceptance Criteria

- [ ] The Personal Details form is rendered within the expandable accordion card for this checklist item (US-05.1)
- [ ] The form pre-fills the following fields from `StorageService.get('vb:profile')`:
  - Full Name (text input, required)
  - Email Address (email input, required)
  - Phone Number (tel input with country code prefix, required)
  - Nationality (text input or select, required)
  - Date of Birth (date input via Flatpickr, required)
- [ ] All 5 fields are editable by the guest inline within the form
- [ ] A "Save & Confirm" button is present at the bottom of the form
- [ ] On tapping "Save & Confirm":
  - All required fields are validated (non-empty; email format valid; phone non-empty; date of birth is a valid past date)
  - If validation passes: updated values are written back to `vb:profile` via `StorageService.set('vb:profile', updatedProfile)`
  - The checklist item is marked complete in `vb:preArrival:{resvId}.personalDetails.complete = true`
  - An inline summary is stored: `vb:preArrival:{resvId}.personalDetails.summary` = "[Full Name] • [Email]"
  - The accordion card collapses and shows the completed state
- [ ] If any required field fails validation, an inline error message appears below the field; the card does not collapse
- [ ] When the checklist loads and all 5 required fields in `vb:profile` are already populated, this item is **automatically** marked complete without requiring the guest to open and re-save the form
- [ ] A validation error state is visually distinct: red border on the field, red error text beneath (`font-size: 0.75rem`, `color: var(--color-error)`)

## UI / UX Notes

- **Form layout**: single-column stacked fields within the accordion card body; `gap: 16px` between fields
- **Field label**: `font-size: 0.8125rem`, `font-weight: 600`, `color: var(--color-text-primary)`, displayed above each input
- **Input style**: `border: 1.5px solid var(--color-border)`, `border-radius: 8px`, `padding: 10px 14px`, `font-size: 0.9375rem`, `width: 100%`; focus state: `border-color: var(--color-brand-primary)`, `box-shadow: 0 0 0 3px var(--color-brand-primary-alpha)`
- **Phone number**: render a small country code dropdown (flag + dial code, e.g., "+94 🇱🇰") prefix before the number input field; default to `+94` (Sri Lanka) unless `vb:profile.countryCode` is set
- **Date of Birth**: Flatpickr date picker; `maxDate: "today"` to prevent future dates; `dateFormat: "d M Y"` display format; value stored as `YYYY-MM-DD`
- **"Save & Confirm" button**: full-width, brand primary filled, `border-radius: 8px`, `padding: 12px`, `font-weight: 600`; loading spinner shown during save (even though save is synchronous to localStorage, a 400ms simulated delay adds quality feel)
- **Auto-complete indicator**: if the item is auto-completed on load, the accordion card immediately shows completed state without ever expanding; a subtle toast notification is not required for auto-completion

## Technical Notes

- **Alpine.js component**: `x-data="personalDetailsForm()"` embedded within the US-05.1 accordion card
- **Pre-fill on init**:
  ```js
  const profile = StorageService.get('vb:profile') || {};
  this.form = {
    fullName: profile.fullName || '',
    email: profile.email || '',
    phone: profile.phone || '',
    countryCode: profile.countryCode || '+94',
    nationality: profile.nationality || '',
    dob: profile.dob || ''
  };
  ```
- **Auto-complete check** (runs in `init()`):
  ```js
  const allFilled = ['fullName','email','phone','nationality','dob'].every(f => !!this.form[f]);
  if (allFilled) this.markComplete();
  ```
- **Validation**:
  ```js
  validateEmail: (v) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(v),
  validateDob: (v) => v && dayjs(v).isBefore(dayjs())
  ```
- **Save**:
  ```js
  const updatedProfile = { ...StorageService.get('vb:profile'), ...this.form };
  StorageService.set('vb:profile', updatedProfile);
  const preArrival = StorageService.get(`vb:preArrival:${resvId}`) || {};
  preArrival.personalDetails = { complete: true, summary: `${this.form.fullName} • ${this.form.email}` };
  StorageService.set(`vb:preArrival:${resvId}`, preArrival);
  this.$dispatch('preArrival:itemComplete', { item: 'personalDetails' });
  ```
- **Event**: dispatch `preArrival:itemComplete` custom event so the parent US-05.1 component updates progress bar reactively

## Test Cases

### Happy Path

1. Guest profile has all 5 required fields populated; open checklist
2. Verify Personal Details item auto-completes and shows completed state without requiring interaction
3. Guest profile has only 3 fields; open checklist; open Personal Details accordion
4. Verify form pre-fills the 3 existing fields; remaining 2 are empty
5. Fill all fields; tap "Save & Confirm"
6. Verify no validation errors; card collapses; green checkmark shown
7. Verify `vb:profile` updated with new values in localStorage
8. Verify progress bar increments by 1

### Edge Cases

- Email field contains invalid format ("notanemail"): inline error "Please enter a valid email address"; save blocked
- Date of Birth set to today or future: inline error "Date of birth must be in the past"; save blocked
- Guest clears the Full Name field and attempts to save: inline error "Full name is required"
- Guest edits already-completed item: accordion re-opens, form shows previously saved values; re-saving updates `vb:profile` and stays complete
- Very long nationality value: input truncates visually but saves full value to storage
- Phone number includes spaces/dashes: accepted and stored as entered
