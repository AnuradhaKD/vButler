## User Story

**As a** hotel guest with an upcoming reservation,
**I want** a structured pre-arrival checklist that guides me through providing necessary information before I arrive,
**So that** my check-in at Browns Hotels & Resorts is faster, more personalised, and free from unnecessary delays at the front desk.

---

## Background / Context

Browns Hotels & Resorts aims to offer a seamless, luxury arrival experience. By completing pre-arrival tasks digitally — confirming personal details, uploading ID, specifying arrival time and special requests, and accepting terms — guests allow the property team to prepare in advance. The checklist provides a clear, step-by-step structure so guests feel guided rather than overwhelmed, and the progress bar rewards completion behaviour.

## Acceptance Criteria

- [ ] The Pre-Arrival Checklist screen is only accessible for reservations with `status: "upcoming"`
- [ ] If accessed for a reservation with any other status (active, checked_out, cancelled), an error state is shown: "Pre-arrival checklist is only available for upcoming reservations"
- [ ] The screen is accessible via: Reservation Detail screen → "Pre-Arrival Checklist" quick link
- [ ] The screen is accessible via: Dashboard hero section upcoming context → direct link
- [ ] A reservation context strip is displayed at the top of the screen showing the property name and formatted check-in date (`DD MMM YYYY`)
- [ ] An overall progress bar is displayed below the context strip showing "X of 6 complete" as text and a percentage-filled bar
- [ ] The progress bar fills proportionally: 0 complete = 0%, 3 complete = 50%, 6 complete = 100%
- [ ] The checklist contains exactly 6 items: Personal Details, ID / Passport Upload, Estimated Arrival Time, Special Requests, Terms & Conditions, (and one property-specific optional item if defined in `property.json → preArrivalItems`)
- [ ] Each checklist item is rendered as a collapsible card (accordion behaviour)
- [ ] Only one accordion card can be expanded at a time; opening one collapses any previously open card
- [ ] **Completed item** appearance: green checkmark icon, green left-border accent (`4px solid var(--color-success)`), label in normal weight, inline summary of submitted data visible below the label
- [ ] **Incomplete item** appearance: empty circle icon, default border, label in normal weight
- [ ] Mandatory checklist items display a red asterisk (`*`) appended to the label
- [ ] Mandatory items: Personal Details, ID / Passport Upload, Terms & Conditions
- [ ] Optional items: Estimated Arrival Time, Special Requests
- [ ] The screen page title is "Pre-Arrival" with a back navigation arrow

## Technical Notes

- **Route**: `#/pre-arrival/:resvId`; the router resolves `resvId` and passes it to the page component
- **Status guard**: on component init, check `reservation.status === 'upcoming'`; if not, set `errorState = true` and return early
- **Pre-arrival data key**: `vb:preArrival:{resvId}` — an object in localStorage storing completion state and data for each checklist item
  ```js
  {
    personalDetails: { complete: true, summary: "Nadia Fernando • nadia@email.com" },
    idDocument: { complete: false },
    arrivalTime: { complete: false },
    specialRequests: { complete: false },
    tcsAccepted: { complete: false }
  }
  ```
- **Progress calculation**:
  ```js
  const completedCount = Object.values(preArrivalData).filter(item => item.complete).length;
  const progressPercent = Math.round((completedCount / 6) * 100);
  ```
- **Accordion state**: Alpine.js `openItem` data property (string, item key); `@click="openItem = openItem === key ? null : key"`; `:class="{ 'is-open': openItem === key }"`
- **Auto-open**: on load, open the first incomplete mandatory item automatically
- **Alpine.js component**: `x-data="preArrivalChecklist()"` in `src/js/pages/preArrival.js`

## Test Cases

### Happy Path

1. Navigate to `#/pre-arrival/:resvId` for an upcoming reservation
2. Verify reservation context strip shows correct property name and check-in date
3. Verify progress bar shows "0 of 6 complete" and 0% fill on first visit
4. Verify 6 accordion cards render with correct labels and mandatory asterisks
5. Verify all cards start in incomplete state (empty circle icon, no green border)
6. Tap Personal Details card header; verify it expands and others remain collapsed
7. Tap another card; verify Personal Details collapses and new card expands
8. Complete Personal Details; verify green checkmark, green left border, inline summary appear
9. Verify progress bar updates to "1 of 6 complete" and ~17% fill

### Edge Cases

- Accessing `#/pre-arrival/:resvId` for an active reservation: error state displayed, no checklist shown
- Accessing for a checked-out reservation: error state displayed
- Returning to checklist after partially completing: previously completed items remain complete with saved data
- All 6 items complete: "You're all set!" success state shown (US-05.7)
- Property has a 6th custom pre-arrival item in `property.json`: renders as 6th accordion card
