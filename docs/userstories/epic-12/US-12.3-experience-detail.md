## User Story

**As a** guest,
**I want** to view the full details of an experience including images, inclusions, pricing, and add-ons,
**So that** I have all the information I need to decide whether to book the experience.

---

## Background / Context

Once a guest is interested in an experience discovered on the hub, they need a rich detail page that communicates everything Browns Hotels & Resorts offers. This page is the conversion point — it must inspire confidence and excitement while providing practical details such as available days, booking cutoff times, and pricing breakdowns. A sticky booking CTA ensures the guest can act at any scroll depth.

---

## Acceptance Criteria

- [ ] The Experience Detail page is accessible at route `#/experiences/{id}` and loads the matching experience from `property.json → experiences[]` by `id`
- [ ] A **full-screen-width hero image** is displayed at the top; if `experience.images[]` contains more than one image, a **Swiper.js carousel** is used with pagination dots; if only one image exists, a static image is shown; if no images exist, brand gradient fallback is displayed
- [ ] The experience **name** is rendered in Playfair Display font below the hero, followed by a **category badge** in the same pill style as the card
- [ ] The full **description** from `experience.description` is displayed as paragraph text below the name
- [ ] A **details row** (horizontal chips or icon-label pairs) displays: Duration (e.g. "3 hours"), Start Time (e.g. "09:00 AM"), and Available Days (formatted as abbreviated weekday names: "Mon, Wed, Fri" from `experience.availableDays[]`)
- [ ] An **Inclusions** section is displayed with a bullet list rendered from `experience.inclusions[]`; each item has a green checkmark icon prefix
- [ ] A **Pricing** section displays: price per person, minimum guests, maximum guests, and any group fee if configured in the experience data
- [ ] A **Booking Cutoff** notice is displayed: "Book at least [N] hours in advance" derived from `experience.bookingCutoffHours`
- [ ] If `experience.addOns[]` contains one or more items, an **Add-Ons section** is displayed; each add-on shows its name and price with a +/– quantity stepper; the selected add-ons and quantities persist in the component state for use by the booking flow (US-12.4)
- [ ] A **wishlist heart icon** is displayed in the page header or top-right area of the hero; it reflects and toggles the experience's wishlist state in `vb:wishlist.experienceIds[]` (same logic as US-12.2)
- [ ] A **sticky bottom CTA bar** is fixed at the bottom of the viewport with text "Book This Experience — From LKR X,XXX" using brand background colour; tapping the CTA opens the booking flow (US-12.4)
- [ ] If `experience.available === false`, the sticky CTA reads "Fully Booked" and is disabled (non-interactive, dimmed)

## UI / UX Notes

- Page route: `#/experiences/{id}`
- Back navigation: standard back chevron in the top bar returning to `#/experiences`
- **Hero image / carousel**:
  - Height: 260px mobile, 340px tablet+
  - `object-fit: cover`, `width: 100%`
  - Swiper.js pagination: white dots, centred at bottom of image area
  - Swiper.js navigation arrows: shown on tablet+ only
  - Image `onerror`: fall back to brand gradient div
- **Name**: Playfair Display, 26px, font-weight 700, dark text (light mode) / white (dark mode), 16px top margin below hero
- **Category badge**: same pill style as hub card; placed immediately below name
- **Details row**: horizontal flex row, wraps if needed; each item is an icon + label chip with a light background; gap: 8px
- **Inclusions section**: heading "What's Included" in 14px uppercase brand colour; items as `<ul>` with custom green checkmark bullets (Material Icons `check_circle_outline`)
- **Pricing section**: heading "Pricing"; layout as a 2-column key-value grid; price per person highlighted in brand colour, larger font (18px)
- **Booking Cutoff**: small amber/warning info chip at bottom of Pricing section; icon `access_time`
- **Add-Ons section**: heading "Enhance Your Experience"; each add-on as a row: name (left), price + stepper (right); stepper: `–` button, quantity display, `+` button; minimum quantity 0
- **Wishlist icon**: positioned in the top-right of the hero image (mobile) or in the sticky header bar (desktop); 44×44px touch target
- **Sticky CTA bar**: `position: sticky; bottom: 0; z-index: 50`; height 64px; brand background; white text and button; safe-area padding for iOS home indicator (`padding-bottom: env(safe-area-inset-bottom)`)
- Content below the hero scrolls naturally; the sticky CTA is always visible
- Dark mode: all sections use dark surface colours from CSS variables

## Technical Notes

- **Swiper.js** config: `{ pagination: { el: '.swiper-pagination', clickable: true }, loop: experience.images.length > 1 }`; only initialised when `images.length > 1`; Swiper.js loaded on-demand (dynamic import or CDN with `defer`)
- **Alpine.js** `x-data="experienceDetail(id)"` manages:
  - `experience` — loaded by `id` from `window.propertyData.experiences` on `init()`
  - `selectedAddOns` — object `{ [addOnId]: quantity }`, default all 0
  - `wishlistIds` — from localStorage, reactive
  - `addOnTotal` — Alpine getter summing `addOn.price * selectedAddOns[addOn.id]`
  - `grandTotalPreview` — `experience.pricePerPerson * 1 (placeholder guests) + addOnTotal` (full calc in booking modal)
  - `toggleWishlist(id)` — same implementation as US-12.2
- Add-on stepper: `@click="selectedAddOns[addOn.id] = Math.max(0, (selectedAddOns[addOn.id]||0) - 1)"` and increment equivalent
- Routing: SPA router extracts `id` from `#/experiences/{id}` hash and passes to component
- Available days formatting: `experience.availableDays.map(d => d.slice(0,3)).join(', ')` where days are stored as full English weekday names
- Booking cutoff display: `experience.bookingCutoffHours + ' hours'` — purely display in this story; enforcement is in US-12.4

## Test Cases

### Happy Path
1. Guest taps an experience card in the hub → navigates to `#/experiences/{id}`
2. Hero carousel renders with 3 images; Swiper pagination dots visible; swipe between images works
3. Experience name in Playfair Display, category badge beneath it
4. Full description paragraph renders
5. Details row: "3 hours", "09:00 AM", "Mon, Wed, Fri"
6. Inclusions section: 5 bullet points with green checkmarks
7. Pricing section: "LKR 7,500 per person", "Min 2 / Max 10 guests"
8. Booking Cutoff chip: "Book at least 24 hours in advance"
9. Add-Ons: 2 items with +/– steppers; incrementing updates display
10. Sticky CTA bar: "Book This Experience — From LKR 7,500" — tapping opens booking flow
11. Wishlist heart reflects saved state; toggling updates localStorage

### Edge Cases
- **Single image**: No Swiper initialised; static image renders
- **No images**: Brand gradient fallback div renders at hero position
- **No add-ons**: Add-Ons section is hidden entirely
- **`available: false`**: Sticky CTA reads "Fully Booked", is disabled and visually dimmed
- **Invalid `id` in route**: Experience not found in data → render "Experience not found" message with link back to hub
- **Very long description**: Page scrolls normally; sticky CTA does not obscure readable text (bottom padding on page content accounts for CTA height)
- **Add-on quantity at 0**: `–` button is disabled (cannot go below 0)
- **Dark mode**: All sections, the sticky CTA, and the Swiper pagination all render correctly
