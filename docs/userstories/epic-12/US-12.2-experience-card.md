## User Story

**As a** guest,
**I want** to see a compact, visually rich card for each experience in the hub,
**So that** I can quickly assess experiences and decide which ones to explore further or save to my wishlist.

---

## Background / Context

The experience card is the primary discovery unit within the Experiences Hub. Browns Hotels & Resorts' brand identity emphasises visual luxury — each card must communicate quality through imagery, clean typography, and clear pricing at a glance. The card is a reusable component used throughout EPIC-12, including the hub grid and any related carousels.

---

## Acceptance Criteria

- [ ] Each experience card displays: a hero image at the top (brand gradient fallback if no image or image load fails), a category badge overlaid on the image (bottom-left), the experience name below the image, the duration (formatted as "X hours" or "X days"), and the price formatted as "From LKR X,XXX per person"
- [ ] A heart/wishlist icon is displayed in the top-right corner of the card image area; when the experience is in `vb:wishlist.experienceIds[]` the icon is filled (brand colour); otherwise it renders as an outline icon
- [ ] Tapping the heart icon toggles the wishlist state: if not saved, add `experience.id` to `vb:wishlist.experienceIds[]` and update icon to filled; if saved, remove the id and update icon to outline — localStorage is updated immediately on each toggle
- [ ] Tapping anywhere on the card body (not the heart icon) navigates the guest to the Experience Detail page at route `#/experiences/{id}`
- [ ] When `experience.available === false`, a "Fully Booked" overlay is displayed over the card image; the card remains tappable (navigates to detail) but the "Fully Booked" label makes the status clear
- [ ] The "Fully Booked" overlay does not prevent wishlist toggling — the heart icon remains functional

## UI / UX Notes

- Card structure (top to bottom):
  - **Image block**: fixed aspect ratio 16:9, `object-fit: cover`; brand gradient fallback via `onerror` or CSS background; position relative to contain badge and heart icon
  - **Category badge**: bottom-left of image, pill shape, 10px uppercase text, semi-transparent dark background with white text
  - **Heart icon**: top-right of image, 24px, white outline (default) / brand-filled (wishlisted); circular hit target (min 44×44px for touch)
  - **Card body**: white (light) / dark surface (dark mode), 12px padding
  - **Experience name**: 15px, font-weight 600, 2-line clamp (`-webkit-line-clamp: 2; overflow: hidden`)
  - **Duration**: 13px, muted colour, with a small clock icon prefix
  - **Price**: 14px, brand colour, "From LKR X,XXX per person"; comma-formatted number
- "Fully Booked" overlay: semi-transparent black overlay (rgba 0,0,0,0.55) over the entire image area; centred white text "Fully Booked" in 12px uppercase bold; a diagonal red/amber ribbon is acceptable alternative
- Card border-radius: 12px; box-shadow: subtle elevation (`0 2px 8px rgba(0,0,0,0.12)`)
- On hover (desktop): card lifts slightly (`transform: translateY(-2px); box-shadow: elevated`)
- Transition on hover: `transition: transform 0.2s ease, box-shadow 0.2s ease`
- Heart icon animation on toggle: quick scale pulse (`transform: scale(1.3)` then back) using CSS keyframe
- Dark mode: card background `var(--surface-dark)`, text colours from CSS variables

## Technical Notes

- The card is an Alpine.js component or a template partial rendered via `x-for` in the hub's grid
- **Wishlist toggle logic** (Alpine method `toggleWishlist(id)`):
  ```js
  const wishlist = JSON.parse(localStorage.getItem('vb:wishlist') || '{"experienceIds":[],"placeIds":[]}');
  const idx = wishlist.experienceIds.indexOf(id);
  if (idx === -1) wishlist.experienceIds.push(id);
  else wishlist.experienceIds.splice(idx, 1);
  localStorage.setItem('vb:wishlist', JSON.stringify(wishlist));
  this.wishlistIds = wishlist.experienceIds; // reactive update
  ```
- Price formatting: `new Intl.NumberFormat('en-LK').format(experience.pricePerPerson)` — produces comma-separated LKR value
- Image fallback: `<img :src="experience.images[0]" @error="$event.target.style.display='none'; $event.target.nextElementSibling.style.display='block'">`; sibling is a `<div class="brand-gradient-fallback">`
- Navigation on card tap: `@click="navigate('#/experiences/' + experience.id)"` — use the app's SPA router method
- Heart icon tap must call `toggleWishlist(experience.id)` and stop event propagation (`@click.stop`) to prevent navigation
- Phase 1: `experience.available` is read directly from the JSON data; no real-time availability check

## Test Cases

### Happy Path
1. Experiences Hub loads with 6 experience cards in the grid
2. Each card shows: image, category badge, name (2-line clamp), duration, "From LKR X,XXX per person"
3. Heart icon is outline on all cards (fresh session, empty wishlist)
4. Guest taps heart on a "Culinary" experience → icon fills with brand colour; `vb:wishlist.experienceIds` now contains the experience id
5. Guest taps heart again → icon reverts to outline; id removed from wishlist
6. Guest taps card body → navigates to `#/experiences/{id}`

### Edge Cases
- **Image fails to load**: Brand gradient fallback div renders seamlessly; category badge and heart still display correctly
- **Experience name exceeds 2 lines**: Text clamped to 2 lines with ellipsis; no layout overflow
- **Fully Booked experience**: Overlay appears over image; card navigates to detail page on tap; heart toggles normally
- **Wishlist localStorage missing**: `vb:wishlist` key absent → default `{"experienceIds":[],"placeIds":[]}` used; no error
- **Very long price number**: `Intl.NumberFormat` correctly formats e.g. `1500000` as `1,500,000`
- **Heart tap propagation**: Tapping heart does NOT navigate to detail page (event.stopPropagation verified)
- **Dark mode**: All card elements readable; gradient fallback uses dark-mode-appropriate brand colours
