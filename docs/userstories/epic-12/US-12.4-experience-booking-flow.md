## User Story

**As a** guest,
**I want** to book an experience directly through the app with date, guests, and add-on selections,
**So that** I can confirm my activity with Browns Hotels & Resorts without visiting the concierge desk.

---

## Background / Context

The booking flow is the transaction layer of EPIC-12. It must be frictionless enough to convert guest interest into a confirmed booking while being comprehensive enough to capture all required information. As a Phase 1 simulation, bookings are stored in localStorage and charges are written to the guest's folio — no external payment gateway is used. An ICS calendar file download provides a post-booking utility that enhances the guest experience.

## Acceptance Criteria

- [ ] The booking flow opens as a **bottom drawer** on mobile (slides up from bottom, covers ~85% of viewport height) and as a **centred modal** on desktop (≥ 768px); triggered by tapping the sticky CTA on the Experience Detail page (US-12.3)
- [ ] A **Date** field is displayed using Flatpickr; available dates are constrained to: (a) within the guest's active stay dates (check-in to day before check-out) AND (b) matching `experience.availableDays`; dates outside these constraints are disabled in the picker
- [ ] A **Number of Guests** stepper is displayed with a minimum of `experience.minGuests` and maximum of `experience.maxGuests`; the stepper cannot go below the minimum or above the maximum
- [ ] If the experience has `addOns[]`, an **Add-Ons** section is displayed with checkbox items (name + price each); the selected add-ons and quantities from the Detail page are pre-populated; selecting/deselecting items updates the Grand Total in real time
- [ ] A **Special Requests** text field is available (placeholder: "Any special requirements or requests?"; max 300 characters)
- [ ] A **Pricing Summary** section is displayed showing: base price × guest count, each selected add-on subtotal, and a **Grand Total** line — all in LKR with comma formatting; the Grand Total updates reactively as guests or add-ons change
- [ ] A full-width "Confirm Booking" primary button is present at the bottom of the form
- [ ] On "Confirm Booking":
  - Booking is written to `vb:requests[]` with `{ id, type: "experience_booking", category: "experience", reservationId, experienceId, experienceName, date, guestCount, addOns[], specialRequests, grandTotal, status: "confirmed", createdAt, referenceNumber }`
  - Charge is written to `vb:reservations[n].folio[]` as `{ date: today, description: experienceName, quantity: guestCount, unitPrice: pricePerPerson, total: grandTotal, category: "Extras" }`
- [ ] A **Success state** is displayed within the same drawer/modal: booking reference number (format `EXP-YYYYMMDD-XXXX`), experience name, selected date, Grand Total, and an "Add to Calendar" button
- [ ] The "Add to Calendar" button generates and downloads/opens an `.ics` file with: event title = experience name, start date-time = selected date + `experience.startTime`, duration = `experience.durationHours`, description = experience description, location = property name and address
- [ ] The **booking cutoff** is enforced: if the selected date's start time is within `experience.bookingCutoffHours` hours of the current time, the "Confirm Booking" button is disabled and an error message is shown: "This experience must be booked at least [N] hours in advance. Please select a later date."
- [ ] The drawer/modal can be dismissed by: swiping down (mobile drawer), tapping the backdrop (desktop modal), or tapping a close (×) button in the header

## UI / UX Notes

- **Bottom drawer** (mobile): slides up using CSS `transform: translateY(0)` transition; handle bar at top; `overflow-y: scroll` within drawer; backdrop overlay behind drawer
- **Modal** (desktop): centred, max-width 560px, `border-radius: 16px`, backdrop overlay; header with "Book Experience" title + close button
- Drawer/modal header: "Book [Experience Name]" in 17px semi-bold; close × button right-aligned
- **Date picker**: Flatpickr inline or input-trigger; disabled dates rendered in strikethrough/muted style; selected date highlighted in brand colour
- **Guest stepper**: same style as other steppers in the app; `–` button / count / `+` button; count shown in brand colour when at min (cannot go lower)
- **Add-Ons section** (if applicable): each add-on as a row with checkbox, name, and price on the right; below each row, if quantity stepper applies (from Detail page), show stepper; add-ons section heading "Add-Ons" in 14px uppercase
- **Pricing summary**: light background card within drawer; rows: "Base (LKR X,XXX × N guests)" + add-on rows + **Grand Total** in larger brand-coloured text
- Booking cutoff error: amber warning banner at top of form (below header) when constraint is violated; replaces form submit CTA with disabled state
- **Success state**: replaces form content; green checkmark (animated scale-in), "Booking Confirmed" heading in Playfair Display, reference badge, summary line, Grand Total, "Add to Calendar" button (outlined brand style) + "Close" button
- iOS safe-area padding at bottom of drawer (`padding-bottom: env(safe-area-inset-bottom)`)

## Technical Notes

- **Flatpickr** config for date selection:
  ```js
  {
    minDate: reservation.checkInDate,
    maxDate: dayjs(reservation.checkOutDate).subtract(1, 'day').format('YYYY-MM-DD'),
    disable: [
      date => !experience.availableDays.includes(
        ['Sunday','Monday','Tuesday','Wednesday','Thursday','Friday','Saturday'][date.getDay()]
      )
    ]
  }
  ```
- **Booking cutoff validation**: on date selection, compute `selectedDateTime = dayjs(selectedDate).set('hour', startHour).set('minute', startMin)`; check `selectedDateTime.diff(dayjs(), 'hour') < experience.bookingCutoffHours`
- **ICS file generation** (Blob approach):
  ```js
  const icsContent = [
    'BEGIN:VCALENDAR', 'VERSION:2.0',
    'BEGIN:VEVENT',
    `DTSTART:${dayjs(selectedDate + ' ' + experience.startTime).format('YYYYMMDDTHHmmss')}`,
    `DURATION:PT${experience.durationHours}H`,
    `SUMMARY:${experience.name}`,
    `DESCRIPTION:${experience.description}`,
    `LOCATION:${property.name}, ${property.address}`,
    'END:VEVENT', 'END:VCALENDAR'
  ].join('\r\n');
  const blob = new Blob([icsContent], { type: 'text/calendar' });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url; a.download = `${experience.name}.ics`; a.click();
  ```
- **Alpine.js** `x-data="experienceBooking(experience, preselectedAddOns)"` manages:
  - `selectedDate`, `guestCount` (init from `experience.minGuests`), `selectedAddOns`, `specialRequests`
  - `grandTotal` — getter: `(experience.pricePerPerson * guestCount) + addOnTotal`
  - `addOnTotal` — getter: sum of `addOn.price * qty` for all selected add-ons
  - `cutoffViolated` — getter: validates cutoff on `selectedDate` change
  - `submitted`, `referenceNumber`
- Reference: `EXP-${dayjs().format('YYYYMMDD')}-${Math.floor(1000 + Math.random() * 9000)}`
- Folio charge: append to `vb:reservations[activeResvIndex].folio` array; persist updated reservations to `vb:reservations`

## Test Cases

### Happy Path
1. Guest taps "Book This Experience" on the Detail page → bottom drawer slides up
2. Flatpickr date picker shows; available days highlighted; non-available days greyed out/disabled
3. Guest selects an available date within their stay period
4. Guest count defaults to `minGuests` (e.g. 2); guest increments to 4
5. Two add-ons selected: total updates in Pricing Summary
6. Grand Total displayed: "LKR 45,000" (4 × 7,500 + add-ons)
7. Guest taps "Confirm Booking"
8. Success state: "Booking Confirmed", reference EXP-XXXXXXXX-XXXX, date, grand total
9. "Add to Calendar" downloads an `.ics` file with correct event details
10. `vb:requests[]` contains new entry; `vb:reservations[n].folio[]` has new charge

### Edge Cases
- **Booking cutoff violated**: Guest selects a date where start time is 10 hours away and cutoff is 24 hours → error banner shown; Confirm button disabled
- **Guest count at minimum**: `–` button disabled at `experience.minGuests`; cannot go lower
- **Guest count at maximum**: `+` button disabled at `experience.maxGuests`
- **No add-ons**: Add-Ons section hidden; Pricing Summary shows only base × guests = Grand Total
- **Date within stay but not available day** (e.g. experience only on Mon/Wed/Fri, guest selects Tuesday): Flatpickr disables Tuesday; guest cannot select it
- **Experience `available: false`**: Booking flow cannot be opened (CTA is disabled on Detail page per US-12.3)
- **ICS download on iOS**: Blob URL + `<a>` click approach tested; `.ics` opens in calendar app
- **Backdrop tap to dismiss**: Drawer/modal closes; no booking submitted
- **Rapid submit click**: Confirm button enters loading state on first click; second click ignored
