## User Story

**As a** guest,
**I want** to browse all curated experiences offered by the property in one place,
**So that** I can discover activities and adventures that enrich my stay at Browns Hotels & Resorts.

---

## Background / Context

Browns Hotels & Resorts positions itself as a luxury boutique brand whose guests expect more than accommodation — they expect curated, memorable experiences. The Experiences Hub is the discovery layer that surfaces all available activities, from cultural tours to wellness retreats, in an engaging visual format. Wishlisting and category filtering allow guests to plan and refine their choices at leisure.

---

## Acceptance Criteria

- [ ] The Experiences Hub is accessible via Quick Actions → "Experiences" and routes to `#/experiences`
- [ ] A **Featured Experience** hero card is displayed at the top of the page: full viewport width, brand gradient fallback if no image, experience title in Playfair Display font, category badge, duration chip, price ("From LKR X,XXX per person"), "Book Now" primary button, and a heart save/wishlist button
- [ ] The featured experience is determined by `experience.isFeatured === true` in the data source; if no experience has `isFeatured: true`, the first experience in the array is used
- [ ] A horizontal scrollable row of **category filter chips** is displayed below the hero: All | Adventure | Cultural | Culinary | Wellness | Romantic | Family; "All" is selected by default
- [ ] Selecting a category chip filters the experience grid to show only experiences matching `experience.category`; "All" removes the filter
- [ ] Below the filter chips, experiences are displayed in a **2-column card grid** on mobile (≤ 767px) and a **3-column card grid** on tablet and desktop (≥ 768px)
- [ ] A **"Saved" chip tab** is available alongside (or as part of) the filter chip row; selecting it filters the grid to show only experiences whose `id` is present in `vb:wishlist.experienceIds[]`
- [ ] Category filter and Saved filter operate independently — selecting a category while in "Saved" mode shows only saved experiences of that category
- [ ] The hub loads experience data from the property's JSON data source (`property.json` → `experiences[]`)
- [ ] If no experiences match the current filter, an empty state is shown: "No experiences found" with a suggestion to clear the filter

## UI / UX Notes

- Page route: `#/experiences`
- Page header: "Experiences" with a compass/star icon; brand top bar
- Featured hero card: minimum height 260px on mobile, 320px on tablet; image fills the card as `object-fit: cover`; text overlaid on a dark gradient scrim (bottom 60% of card); brand gradient (`linear-gradient(135deg, var(--brand-primary), var(--brand-secondary))`) used as fallback when no image
- Experience title in hero: Playfair Display, 24px, white, font-weight 600
- Category badge: uppercase, 10px letter-spacing, semi-transparent white background pill
- "Book Now" button in hero: white background, brand text colour, compact size
- Heart icon in hero: positioned top-right of the card; filled = wishlisted (brand colour), outline = not saved
- Category filter chips: horizontally scrollable (no visible scrollbar on mobile), pill style, brand colour when active, border-only when inactive
- "Saved" chip: visually distinct — uses a bookmark or heart icon prefix; placed at the end of the filter chip row or as a separate tab indicator
- Grid cards: see US-12.2 for card spec
- Empty state: centred illustration or icon, muted text, "Clear Filter" text link
- Smooth fade-in transition on grid re-render after filter change (`x-transition` Alpine.js)

## Technical Notes

- **Alpine.js** `x-data="experiencesHub()"` component manages:
  - `experiences` — all experiences loaded from JSON on `init()`
  - `selectedCategory` — string, default `'all'`
  - `showSavedOnly` — boolean, default `false`
  - `wishlistIds` — loaded from `localStorage.getItem('vb:wishlist')` and parsed
  - `filteredExperiences` — Alpine getter: filters `experiences` by category and wishlist state
  - `featuredExperience` — getter: `experiences.find(e => e.isFeatured) || experiences[0]`
- **Data source**: fetch `property.json` on component `init()` via `fetch()` or use pre-loaded global `window.propertyData`
- **Wishlist integration**: on mount, parse `vb:wishlist` from localStorage; `wishlistIds` is reactive — toggling heart icon on any card updates localStorage and `wishlistIds` array simultaneously
- Category filter: `filteredExperiences = experiences.filter(e => (selectedCategory === 'all' || e.category === selectedCategory) && (!showSavedOnly || wishlistIds.includes(e.id)))`
- When "Saved" chip is active and `wishlistIds` is empty → trigger empty state
- Grid uses CSS Grid: `grid-template-columns: repeat(2, 1fr)` on mobile, `repeat(3, 1fr)` on tablet+ via media query
- Featured hero is excluded from the grid below it (rendered separately, not as part of the filtered grid)

## Test Cases

### Happy Path
1. Guest navigates to `#/experiences` from Quick Actions
2. Featured hero card renders with the experience marked `isFeatured: true` — image loads, title in Playfair Display, category badge, duration, price, Book Now + heart buttons all visible
3. Category chips render: All | Adventure | Cultural | Culinary | Wellness | Romantic | Family; "All" is active
4. 6 experience cards render in a 2-column grid (mobile viewport)
5. Guest taps "Culinary" chip → grid filters to show only Culinary experiences; "All" chip deselects
6. Guest taps "Saved" chip → grid shows only experiences in `vb:wishlist.experienceIds[]`
7. Guest taps "All" chip → full grid restored
8. On tablet viewport, grid changes to 3 columns

### Edge Cases
- **No `isFeatured` in data**: First experience in array is used as hero — no error, no blank hero
- **Empty category**: Guest selects a category with no experiences → empty state shown; "Clear Filter" returns to All
- **Saved tab with empty wishlist**: Empty state shown with heart illustration
- **Saved + Category combo**: Guest is in Saved mode, taps "Wellness" → only saved Wellness experiences shown; if none, empty state
- **Image load failure**: Hero image `onerror` triggers brand gradient fallback class
- **Single experience in data**: Hub renders correctly with 1 hero and 0 grid cards (or hero doubles as grid)
- **Large dataset (20+ experiences)**: Grid renders without layout breakage; scrolling is smooth
