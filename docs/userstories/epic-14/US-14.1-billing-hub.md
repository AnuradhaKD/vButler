## User Story

**As a** guest,
**I want** to see a clear summary of all charges and my current balance in one place,
**So that** I can stay informed about my spending and avoid surprises at check-out.

---

## Background / Context

Billing transparency is a top driver of guest satisfaction and a common pain point in traditional hotel stays. Browns Hotels & Resorts commits to a frictionless, transparent billing experience through Destinity vButler. The Billing Hub serves as the guest's personal financial dashboard — presenting an at-a-glance balance, category-level breakdown, and quick access to itemised details. This is a P0 feature because financial clarity directly affects trust and the guest's willingness to charge services to their room.

## Acceptance Criteria

- [ ] The Billing Hub is accessible at `#/billing` via Quick Actions → "My Bill" and the bottom navigation bar → "Bill"
- [ ] An **Account Balance card** is displayed at the very top of the page:
  - Outstanding balance displayed in large LKR text (e.g. "LKR 45,250")
  - If balance = 0: shows "Paid in Full" with a green checkmark icon instead of the amount
  - If balance > 0: shows a "Pay Now" button in brand colour alongside the balance; tapping "Pay Now" navigates to `#/payment`
- [ ] A **Charge Summary section** displays the total spent per category as individual rows: Room | Dining | Wellness | Services | Extras | Payments; each row shows the category name, icon, and LKR total
- [ ] Each category row in the Charge Summary is **tappable** — tapping it scrolls the page to the corresponding day-group section in the itemised view below (anchor navigation), or navigates to US-14.2
- [ ] Rows with a total of LKR 0 (no charges in that category) are still displayed but show "—" instead of a value, or are hidden — the preference is to hide empty rows for cleanliness
- [ ] A **Date Range Filter** is displayed below the summary: quick filter pills — "All" | "Today" | "Last 3 Days" | "Full Stay" — and a "Custom Range" option that opens a Flatpickr date range picker; filtering affects the itemised view below but not the Account Balance card (which always shows the full outstanding balance)
- [ ] A **debounced text search** input is displayed: filtering the itemised view by `folio.description` as the guest types (300ms debounce); search applies on top of the active date range filter
- [ ] A **Reservation Context bar** is displayed below the page header: property name, reservation number, check-in and check-out dates
- [ ] Billing data is sourced from `vb:reservations[activeIndex].folio[]` in localStorage
- [ ] The page auto-refreshes its data when the user navigates to it (in case a new charge was added by another screen)

## Technical Notes

- **Balance calculation**:
  ```js
  const charges = folio.filter(f => f.type !== 'payment').reduce((sum, f) => sum + f.total, 0);
  const paid = folio.filter(f => f.type === 'payment').reduce((sum, f) => sum + Math.abs(f.total), 0);
  const balance = charges - paid;
  ```
- **Category totals**:
  ```js
  const categories = ['Room', 'Dining', 'Wellness', 'Services', 'Extras'];
  categories.forEach(cat => {
    totals[cat] = folio.filter(f => f.category === cat && f.type !== 'payment').reduce((s, f) => s + f.total, 0);
  });
  totals['Payments'] = folio.filter(f => f.type === 'payment').reduce((s, f) => s + Math.abs(f.total), 0);
  ```
- **Date range filter**: filter folio entries by `dayjs(entry.date).isBetween(startDate, endDate, 'day', '[]')`; "Today" = today only; "Last 3 Days" = `dayjs().subtract(2, 'day')` to today; "Full Stay" = all entries (same as "All")
- **Debounced search**: `setTimeout`-based debounce (300ms) on the `x-on:input` event; filters by `entry.description.toLowerCase().includes(query.toLowerCase())`
- **Alpine.js** `x-data="billingHub()"`:
  - `folio` — from localStorage on `init()`
  - `balance` — computed getter
  - `categoryTotals` — computed object
  - `dateFilter` — default `'all'`; custom range uses `customStart`, `customEnd`
  - `searchQuery` — string, default `''`
  - `filteredFolio` — applies date filter then search filter
- Page re-reads localStorage on `init()` (called each time route is navigated to); use `x-init` with router lifecycle hook
- LKR formatting throughout: `new Intl.NumberFormat('en-LK', { style: 'currency', currency: 'LKR', minimumFractionDigits: 0 }).format(amount)` or simpler `'LKR ' + Intl.NumberFormat('en-LK').format(amount)`

## Test Cases

### Happy Path
1. Guest navigates to `#/billing`
2. Reservation Context bar: "Browns Beach Hotel · RES-20240405-001 · 3 Apr – 7 Apr 2025"
3. Account Balance card: "LKR 45,250" with "Pay Now" button
4. Charge Summary: Room LKR 28,000 | Dining LKR 12,750 | Wellness LKR 4,500 | Payments LKR 0
5. Guest taps "Today" filter pill → only today's charges shown in itemised view; balance card unchanged
6. Guest types "spa" in search → only spa-related charge items visible
7. Guest clears search → all items for active date filter restored
8. Guest taps "Pay Now" → navigates to `#/payment`

### Edge Cases
- **Balance = 0**: "Paid in Full" green state; "Pay Now" button not shown
- **Empty folio**: "No charges yet" empty state; balance shows LKR 0 / "Paid in Full"
- **Custom date range**: Flatpickr range picker opens; selecting 3 Apr – 5 Apr filters correctly
- **Search with no results**: "No charges matching '[query]'" message; clear search link
- **Date filter + search combined**: Both filters applied simultaneously; results are intersection
- **Very large folio (50+ items)**: Rendering and filtering complete without lag; search debounce prevents rapid re-renders
- **Category total = 0**: Empty category rows hidden; only categories with charges shown in summary
- **Page re-navigation**: Navigate away and back; folio data re-loaded; new charges added by booking flow appear
