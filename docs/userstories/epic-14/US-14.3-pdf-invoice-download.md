## User Story

**As a** guest,
**I want** to download a professionally formatted PDF invoice of all my charges,
**So that** I have an official record of my stay expenses for reimbursement, expense reporting, or personal records.

---

## Background / Context

Business travellers and expense-conscious guests require a downloadable invoice as a formal record of hotel charges. Generating this client-side with jsPDF avoids server infrastructure while still delivering a polished, branded document that reflects the quality of Browns Hotels & Resorts. The PDF must open in the browser for print-on-demand as well as download, supporting both use cases without additional interaction.

---

## Acceptance Criteria

- [ ] Tapping the "Download Invoice" button on US-14.2 triggers immediate **client-side PDF generation** using jsPDF and jsPDF-AutoTable — no server request is made
- [ ] The generated PDF contains, in order:
  1. **Browns Hotels & Resorts logo** top-left (embedded as base64 image in the JS; brand mark or wordmark)
  2. **Property details** top-right (or below logo): property name, address, phone, email (from `property.json`)
  3. **"INVOICE" heading** in large Playfair Display-equivalent font (or bold serif via jsPDF standard fonts)
  4. **Guest details block**: guest name, reservation number, room type
  5. **Stay dates block**: check-in date, check-out date, number of nights
  6. **Charges table** (jsPDF-AutoTable): columns — Date | Description | Qty | Unit Price (LKR) | Total (LKR); all folio entries in chronological order; payment rows styled in green text; subtotal and tax rows included as table footer rows per day group
  7. **Tax Summary block**: total charges before tax, total tax amount, total payments, and **Grand Total** (or "Paid in Full" if balance = 0)
  8. **Footer on every page**: property phone and email | "Thank you for staying with us at [Property Name]" | page number "Page X of Y"
- [ ] The generated PDF filename is: `Invoice_{ReservationNumber}_{GuestLastName}.pdf` (e.g. `Invoice_RES-20240405-001_Perera.pdf`)
- [ ] The PDF is **opened in a new browser tab** (not forced as a download) using a Blob URL, enabling the guest to preview and print from the browser's native print dialog
- [ ] An **iOS-compatible fallback** is implemented: if `URL.createObjectURL` is supported, use it to open the tab; otherwise create a hidden `<a>` element with `download` attribute and programmatically click it
- [ ] jsPDF and jsPDF-AutoTable are loaded from CDN; they are loaded **on demand** (only when the button is first tapped) to avoid bloating initial page load — implemented via dynamic `<script>` injection or deferred CDN with `defer` attribute
- [ ] The "Download Invoice" button shows a brief loading spinner while the PDF is being generated (typically < 1 second); the button is disabled during generation to prevent double-triggering
- [ ] All monetary values in the PDF are formatted as LKR with comma separation (e.g. "LKR 12,500")

## UI / UX Notes

- **Button state during generation**: button label changes to a spinner icon or "Generating…" text; button disabled; reverts to "Download Invoice" after tab opens (or on error)
- **PDF visual design**:
  - Page size: A4 (210mm × 297mm), portrait
  - Margins: 15mm all sides
  - Logo: ~40mm wide, top-left corner; if logo base64 unavailable, brand name text in bold as fallback
  - Property details: right-aligned, 10pt, same top row as logo
  - "INVOICE" heading: centred or left-aligned below header row; 20pt bold
  - Guest/stay block: 11pt, labelled key-value pairs (e.g. "Guest Name: Kavindi Perera")
  - **AutoTable**: column widths auto-distributed; header row with brand background colour; alternating row shading; table border: none (clean look)
  - Tax summary: right-aligned block, 11pt; "Grand Total" in 14pt bold brand colour
  - Footer: 9pt, centred, light grey; horizontal rule above footer
- **Charges table styling** (jsPDF-AutoTable `styles` and `columnStyles`):
  - Header fill: brand primary colour (RGB conversion of `var(--brand-primary)`)
  - Header text: white
  - Payment rows: `textColor: [34, 139, 34]` (green)
  - Subtotal/tax rows: `fillColor: [245, 245, 245]`, `fontStyle: 'bold'` or `fontStyle: 'italic'`
- The PDF must not include any sensitive payment card details — only masked transaction references

## Technical Notes

- **jsPDF on-demand loading**:
  ```js
  async function loadJsPDF() {
    if (window.jspdf) return window.jspdf;
    await loadScript('https://cdnjs.cloudflare.com/ajax/libs/jspdf/2.5.1/jspdf.umd.min.js');
    await loadScript('https://cdnjs.cloudflare.com/ajax/libs/jspdf-autotable/3.8.2/jspdf.plugin.autotable.min.js');
    return window.jspdf;
  }
  function loadScript(src) {
    return new Promise((resolve, reject) => {
      const s = document.createElement('script');
      s.src = src; s.onload = resolve; s.onerror = reject;
      document.head.appendChild(s);
    });
  }
  ```
- **PDF generation flow**:
  ```js
  const { jsPDF } = window.jspdf;
  const doc = new jsPDF({ orientation: 'portrait', unit: 'mm', format: 'a4' });
  // Add logo image
  doc.addImage(base64Logo, 'PNG', 15, 12, 40, 14);
  // Add property details
  doc.setFontSize(9); doc.text(property.name, 195, 14, { align: 'right' });
  // ... additional text blocks ...
  // AutoTable for charges
  doc.autoTable({ head: [['Date','Description','Qty','Unit (LKR)','Total (LKR)']], body: tableRows, ... });
  // Open in new tab
  const blob = doc.output('blob');
  const url = URL.createObjectURL(blob);
  window.open(url, '_blank');
  ```
- **iOS fallback**:
  ```js
  if (/iPhone|iPad/.test(navigator.userAgent)) {
    const dataUri = doc.output('datauristring');
    const a = document.createElement('a');
    a.href = dataUri; a.download = filename; a.click();
  } else {
    window.open(URL.createObjectURL(doc.output('blob')), '_blank');
  }
  ```
- **AutoTable body construction**: iterate `folio` chronologically; group by day; for each day push charge rows, then a subtotal row, then a tax row (if applicable); then payment rows; at end push the grand total row
- Logo base64: store a small (<20KB) base64-encoded PNG of Browns logo in `assets/js/logoBase64.js` or inline in the PDF generation function
- Filename construction: `Invoice_${reservation.reservationNumber}_${guest.lastName}.pdf`

## Test Cases

### Happy Path
1. Guest on Billing Hub taps "Download Invoice"
2. Button shows "Generating…" spinner; jsPDF CDN scripts load (first call only)
3. PDF opens in new browser tab within 1 second
4. PDF contains: Browns logo top-left, property details top-right, "INVOICE" heading, guest name, reservation number, room type, check-in/check-out dates
5. AutoTable renders all folio entries in date order; header row has brand background
6. Payment row in green with negative sign
7. Tax summary block shows total charges, total tax, total payments, Grand Total
8. Footer on every page: property contact, "Thank you", page numbers
9. Filename in tab/download: `Invoice_RES-20240405-001_Perera.pdf`

### Edge Cases
- **jsPDF already loaded** (second tap): No duplicate script injection; PDF generates immediately
- **iOS Safari**: Blob URL may not work; `<a download>` fallback triggers; PDF downloads rather than opening in tab — acceptable behaviour
- **Large folio (50+ items)**: AutoTable handles multi-page tables; footer appears on all pages
- **Zero balance**: Grand Total block shows "Paid in Full" in green text instead of an amount
- **No tax**: Tax row omitted from AutoTable and tax summary block
- **Logo base64 missing/fails**: Fallback to property name as bold text in logo area; no error thrown
- **Pop-up blocker**: Browser blocks `window.open()` → catch error; show toast "Please allow pop-ups to view your invoice, or try 'Download' option" + fallback `<a download>` button
- **Special characters in guest name** (e.g. accented characters): jsPDF standard font handles Latin Extended-A characters; no encoding errors
