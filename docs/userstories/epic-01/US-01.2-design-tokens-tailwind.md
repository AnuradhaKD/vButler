## User Story

**As a** developer,
**I want** a single, authoritative Tailwind CSS configuration that encodes all Browns Hotels brand tokens,
**So that** every developer and every UI component references the same colours, typography, and spacing without ad-hoc inline overrides.

---

## Background / Context

Browns Hotels & Resorts has a defined brand identity built around a deep teal palette (`#003c52`). Destinity vButler must reflect this identity consistently across all screens. Because the project uses Tailwind CSS via CDN (no build step in Phase 1), design tokens are declared in a `tailwind.config` object in a `<script>` block placed before the Tailwind CDN `<script>` tag. All custom utility classes generated from these tokens must be available globally to every page module.

---

## Acceptance Criteria

- [ ] A `tailwind.config` JavaScript object is declared in a `<script>` block in `index.html` **before** the Tailwind CDN `<script>` tag
- [ ] `theme.extend.colors.brand` is defined with keys `DEFAULT: '#003c52'`, `light: '#00516e'`, `dark: '#002838'`
- [ ] `theme.extend.colors.surface` is defined with keys `light: '#f5f8f8'`, `dark: '#0f1e23'`
- [ ] `theme.extend.fontFamily.sans` overrides the default sans stack to use `['Inter', 'ui-sans-serif', 'system-ui']`
- [ ] `theme.extend.fontFamily.display` is defined as `['Playfair Display', 'Georgia', 'serif']`
- [ ] `theme.extend.borderRadius` includes `DEFAULT: '4px'`, `lg: '8px'`, `xl: '12px'`, `full: '9999px'`
- [ ] `darkMode` is set to `'class'` (dark mode toggled by presence of `.dark` on `<html>`)
- [ ] Inter (weights 300, 400, 500, 600, 700) is loaded via Google Fonts CDN `<link>` in `<head>`
- [ ] Playfair Display (weights 400, 600, 700) is loaded via Google Fonts CDN `<link>` in `<head>`
- [ ] Material Symbols Outlined icon font is loaded via Google Fonts CDN `<link>` in `<head>`
- [ ] All font `<link>` tags use `rel="preconnect"` and `rel="stylesheet"` pairs for performance
- [ ] Custom Tailwind classes `bg-brand`, `text-brand`, `bg-brand-light`, `bg-surface-light`, `bg-surface-dark`, `font-display`, `font-sans` are usable in any HTML file without additional imports

## UI / UX Notes

- Browns brand colour `#003c52` should feel like a rich, dark teal — not navy and not pure green. Use it for primary buttons, active navigation states, key headings, and the app header/sidebar
- `surface.light` (`#f5f8f8`) is the light mode page background — slightly warm off-white, not pure white
- `surface.dark` (`#0f1e23`) is the dark mode page background — very dark teal-tinged charcoal
- Inter is the body and UI font: clean, neutral, highly legible at small sizes on mobile
- Playfair Display is used exclusively for property names, hero titles, and premium display contexts — never for body copy or UI labels
- Material Symbols Outlined (not Filled or Rounded) is the icon set: consistent stroke weight, outlined style unless the icon represents an "active" state (active nav items use the filled variant via `font-variation-settings: 'FILL' 1`)

## Technical Notes

- **Tailwind CDN URL:** `https://cdn.tailwindcss.com` (v3.x)
- **Config placement:** `<script>tailwind.config = { ... }</script>` placed immediately before `<script src="https://cdn.tailwindcss.com"></script>`
- **No PostCSS / no build pipeline** in Phase 1 — CDN mode supports `tailwind.config` via global assignment
- **Google Fonts CDN links** (place in `<head>` in this order):
  1. `<link rel="preconnect" href="https://fonts.googleapis.com">`
  2. `<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>`
  3. `<link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&family=Playfair+Display:wght@400;600;700&display=swap" rel="stylesheet">`
  4. `<link href="https://fonts.googleapis.com/css2?family=Material+Symbols+Outlined:opsz,wght,FILL,GRAD@20..48,100..700,0..1,-50..200" rel="stylesheet">`
- **Material Symbols variable font:** Use `font-variation-settings: 'FILL' 0` for outlined (default) and `'FILL' 1` for filled (active states); wrap in a CSS utility `.icon-filled { font-variation-settings: 'FILL' 1; }`
- **Colour contrast:** Verify `text-white` on `bg-brand` meets WCAG AA (4.5:1) — `#003c52` on white is approximately 9.7:1 ✓
- **Dark mode surface:** `bg-surface-dark` used on `<html>` or `<body>` in dark mode; component backgrounds use `dark:bg-brand-dark` or semi-transparent overlays

## Test Cases

### Happy Path

1. Open `index.html` in browser — Inter font renders for all body text (verify via DevTools → Computed → font-family)
2. Inspect a button using `bg-brand` class — computed background colour is `#003c52`
3. Toggle dark mode (add `.dark` to `<html>`) — `bg-surface-dark` elements compute to `#0f1e23`
4. Apply `font-display` to a heading — Playfair Display renders in DevTools computed styles
5. Use `text-brand-light` on an element — computed colour is `#00516e`
6. Open Network tab — Google Fonts CSS and font files loaded; Material Symbols font loaded

### Edge Cases

- Load app with browser fonts blocked (offline, no cache) — Inter/Playfair fall back to `ui-sans-serif`/`Georgia` gracefully; layout does not break
- Tailwind `<script>` loads before config `<script>` (wrong order) — Tailwind generates default config without custom tokens; test that correct order is enforced and no class like `bg-brand` works if config is missing
- Material Symbols icon renders as text ("home") rather than icon — indicates font not loaded; verify CDN link is present and correct
