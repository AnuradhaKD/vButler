## User Story

**As a** guest,
**I want** to add Destinity vButler to my phone's home screen and launch it like a native app,
**So that** I can access hotel services instantly during my stay without opening a browser and typing a URL.

---

## Background / Context

A key value proposition of Destinity vButler is that it feels native. The PWA manifest enables guests to install the app from both iOS Safari (Add to Home Screen) and Android Chrome (install prompt or Add to Home Screen). Once installed, the app launches in standalone mode â€” no browser address bar or tabs â€” making it indistinguishable from a downloaded app. This is particularly important for hotel guests who may only interact with the app for a few days and expect a frictionless, native-quality experience.

## UI / UX Notes

- Icons should feature the Destinity vButler monogram or Browns Hotels logo on a `#003c52` (brand teal) background, with adequate safe zone padding to accommodate maskable icon cropping (minimum safe zone: 10% on all sides for maskable icons)
- The splash screen colour `#003c52` will display while the PWA loads â€” this should feel intentional and branded, not a blank white flash
- On iOS, the `apple-mobile-web-app-status-bar-style` should be set to `"black-translucent"` so the status bar overlays the app in brand colour rather than showing a white bar
- The app should not display any browser UI elements (URL bar, back/forward buttons, bookmarks toolbar) when launched from home screen in standalone mode

## Technical Notes

- **manifest.json location:** Project root, referenced with absolute path `/manifest.json`
- **Icon generation:** Create icons at 192Ă—192 and 512Ă—512 using a PNG export of the Destinity vButler/Browns logo on `#003c52` background; use a tool such as Maskable.app to verify maskable safe zone
- **Icon directory:** `assets/icons/icon-192.png`, `assets/icons/icon-512.png`
- **iOS PWA limitations:** iOS Safari does not support the `beforeinstallprompt` event; installability relies entirely on the Apple meta tags and user manually choosing "Add to Home Screen" from the Share menu; display a subtle in-app install hint banner for iOS users (check `navigator.standalone === false && /iphone|ipad/i.test(navigator.userAgent)`)
- **Android install prompt:** Listen for `beforeinstallprompt` event; store the event and show a custom "Install App" banner (dismiss stores preference in localStorage `vb:installDismissed`)
- **Service worker requirement:** Android Chrome will not show the install prompt without a registered service worker (see US-01.4); these two stories must ship together
- **manifest.json full structure:**
  ```json
  {
    "name": "Destinity vButler â€” Browns Hotels",
    "short_name": "Destinity vButler",
    "description": "Your personal hotel concierge by Browns Hotels & Resorts",
    "start_url": "/index.html#/dashboard",
    "display": "standalone",
    "orientation": "portrait",
    "theme_color": "#003c52",
    "background_color": "#003c52",
    "icons": [
      { "src": "/assets/icons/icon-192.png", "sizes": "192x192", "type": "image/png", "purpose": "maskable any" },
      { "src": "/assets/icons/icon-512.png", "sizes": "512x512", "type": "image/png", "purpose": "maskable any" }
    ]
  }
  ```

### Happy Path

1. Open app in Android Chrome over HTTPS â€” after a few seconds of use, an install banner or address bar install icon appears
2. Tap install â†’ home screen icon added; open from home screen â†’ app launches in standalone mode (no browser address bar)
3. Open app in iOS Safari â†’ Share menu â†’ "Add to Home Screen" â†’ confirm; home screen icon added; tap icon â†’ app opens in standalone mode

5. Verify in Chrome DevTools â†’ Application â†’ Manifest that all fields populate correctly and no warnings appear

### Edge Cases

- User dismisses Android install prompt â†’ `vb:installDismissed` set to `true` â†’ banner does not re-appear for the session
- App launched from home screen on iOS: `navigator.standalone` is `true` â†’ in-app install hint not shown
- PWA accessed via HTTP (not HTTPS) â€” manifest still loads, but installability criteria not met; no install prompt (expected); note: localhost over HTTP is exempt
- `manifest.json` returns 404 â€” browser DevTools shows manifest error; AC fails; verify correct path
- Icon file missing â€” manifest valid but icon entry unresolvable; test that app still installs with placeholder icon
